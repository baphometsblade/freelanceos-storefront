A Closer Look at Each Module

FreelanceOS Pro is not a single database with a few columns. It is seven purpose-built modules, each designed for a specific administrative challenge psychiatrists face every week. Here is a detailed breakdown of exactly what you get inside each one.

Module 1 — Patient Intake Workflow

A Kanban-style pipeline database with custom stages: Inquiry Received, Insurance Verification Pending, Insurance Verified, Intake Forms Sent, Intake Forms Received, Appointment Scheduled, and Consult Completed. Each patient record stores referral source, preferred contact method, insurance carrier and plan, primary care physician for coordination, notes about scheduling availability, and a linked appointment once scheduled. A gallery view shows all current intake inquiries by stage at a glance. A list view sorted by days-in-stage shows which inquiries have been sitting longest. New patient intake has never been this visible or this organized.

Module 2 — Appointment Scheduling System

A chronological appointment log database with properties for patient name (linked to intake records), appointment date, appointment type, session format (in-person or telehealth), session length, status, CPT code note, and billing claim link. Multiple views are included: a weekly calendar view of all upcoming appointments, a table view filtered to show only no-shows and late cancellations, a patient-level history view showing every appointment for each patient sorted chronologically, and a monthly summary view for practice volume tracking. Everything you need to understand your schedule history without touching your EHR's reporting module.

Module 3 — Billing and Insurance Tracker

A billing tracking database with properties for patient name, insurance carrier, claim submission date, CPT code, claim status (Pending, Paid, Denied, Appealing, Resubmitted, Written Off), denial reason code, reimbursement amount received, copay amount, copay collection status, and a days-since-submission calculated value. Key views include: Outstanding Claims (all unresolved claims sorted by submission date), Denied Claims Requiring Action (claims in Denied or Appealing status), Claims by Insurer (grouped view to compare payer behavior), and Copay Collection Outstanding (all patients with uncollected copays). The outstanding claims view alone is worth more than $29 to most practices the first time they run it.

Module 4 — CME Credit Planner

A dual-database system with a Completed Activities log and a Planned Activities pipeline. The Completed Activities database stores activity name, sponsor, accreditation organization, activity type, credit category, date completed, credit hours, and any notes. The Planned Activities database stores activity name, date or date range, registration status, expected credit hours, estimated cost, and a registration link. A summary dashboard view shows your running credit total by category versus your target, updated automatically as you add entries. A per-state requirements view lets multi-state practitioners track CME separately per license jurisdiction. Your entire CME history is searchable by any field.

Module 5 — Peer Supervision Notes

A session log database with properties for session date, format (in-person, video, phone), consultation group or supervisor name, participants, topics discussed, de-identified case descriptors, key clinical takeaways, action items with due dates, completion status of action items, and personal reflection notes. A gallery view shows all sessions chronologically with key takeaways previewed. A filtered view shows all open action items from any session, sorted by due date. A search across the entire database returns every session where a specific topic or clinical descriptor was discussed — making consultation history retrieval instantaneous rather than dependent on memory or calendar archaeology.

Module 6 — Licensing and Renewal Calendar

A credentials database with a separate entry for each credential type: state medical license, DEA registration, board certification, malpractice insurance, PDMP enrollment, NPI profile, CAQH ProView, and hospital privileges. Properties include credential type, issuing authority, credential number or registration number, issue date, expiration date, renewal window start date, renewal fee, required documentation checklist, notes, application portal link, and a 90-day priority flag. A filtered view shows every credential expiring within the next 90 days at the top of the list. A state-grouped view for multi-state practitioners shows renewal status per jurisdiction. Never again will you discover a lapsed credential after the fact.

Module 7 — Research and Locum Opportunity Tracker

A pipeline database for professional opportunities with properties including opportunity type, organization name, primary contact name and details, agency name if recruiter-sourced, pipeline stage, application date, interview date, offer date, contract status, start date, end date or duration, geographic location, remote vs. on-site flag, compensation type (hourly, daily, or total contract), compensation amount, follow-up date, and next action notes. A Kanban view groups opportunities by pipeline stage for visual progress tracking. A comparison view shows accepted and pending opportunities side by side with compensation sorted for easy decision-making. A historical archive view stores completed or declined opportunities so you maintain a full record of your professional exploration over time.
