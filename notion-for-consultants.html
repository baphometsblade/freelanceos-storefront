Sales Pipeline

Your Consulting Pipeline in Notion

Build a visual Kanban pipeline for your new business development. Track every prospect from initial contact to signed contract in one board.

To build this pipeline in Notion, create a database called Pipeline or Business Development with a Status property using these exact options as a select field: Lead, Qualified, Proposal Sent, Negotiating, Won, and Lost. Switch to Board view, group by Status, and drag cards between columns as deals progress. Each card is a potential client with properties for Company, Contact Name, Service Type, Deal Size (number), Source, and Next Action Date.

Add a filtered view called This Week that shows only records where Next Action Date is within 7 days and Status is not Won or Lost. Check this view every Monday morning to know exactly which prospects need attention this week. Never let a hot lead go cold because it was buried in a long board.

Lead
Initial contact, referral, or inbound inquiry received
Qualified
Discovery call complete, budget and timeline confirmed
Proposal Sent
Scope of work and pricing sent to prospect
Negotiating
Revisions requested, back-and-forth on scope or price
Won
Contract signed, project kicked off

Add a Lost column for declined proposals so you can analyze your win rate and spot patterns in what types of work you close best.

Client Portal Setup

Client Portal Setup in Notion

Give your clients a professional project dashboard without paying for a separate portal tool. Notion's sharing features make this surprisingly powerful.

Each active client gets a dedicated Notion page that serves as their project portal. You set up the structure once as a template and duplicate it for every new engagement. The client portal page contains everything the client needs to see: project status, upcoming milestones, deliverables with file links, meeting notes, and a shared task list for items that require their input.

📄

Step 1: Create the Client Portal Template

Build a master portal page with sections for Project Overview (status badge, timeline, key contacts), Deliverables (linked database filtered to this client), Meeting Notes (dated sub-pages), and Open Items (tasks requiring client input). This template gets duplicated for every new client.

🌐

Step 2: Enable Public Sharing

In the top-right of the portal page, click Share, then toggle on "Share to web." This generates a public URL. You can also add the client's email address to give them a named login so they can leave comments. Set their permission level to "Can comment" so they cannot accidentally edit your content.

💬

Step 3: Enable Comments on Deliverables

On each deliverable page, enable comments. Clients can leave feedback directly on the document rather than sending a separate email or Slack message. You get structured, contextual feedback and a permanent record of revision requests. No more feedback buried in an email thread from three weeks ago.

📋

Step 4: Update the Portal Weekly

Set a recurring task every Friday to update project status, add any new deliverables, and log that week's meeting notes. Clients who can see live progress ask fewer status-update questions. Proactive transparency is one of the highest-ROI things you can do for client satisfaction and retention.
