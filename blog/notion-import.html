Notion Import: How to Import CSV Files, Markdown, Word Docs, and HTML into Notion

July 11, 2026 · 9 min read · FreelanceOS Blog

Why Import into Notion?

Most people don't start using Notion with a blank slate. You're migrating from somewhere — Trello, Asana, Evernote, Jira, Monday.com, Airtable — or converting existing files that already hold your work: CSV spreadsheets from your accountant, Markdown notes from Obsidian, Word documents full of SOPs, or HTML pages exported from a CMS. Getting that data into Notion without retyping everything is the difference between a smooth transition and a project that stalls out after two hours.

Notion supports several import formats natively and includes dedicated importers for the most common tools people migrate from. The results vary by format and file quality — some imports are nearly perfect out of the box, others require a cleanup pass afterward. This guide covers every import path available in 2026, what works well, what doesn't, and how to fix the issues that come up most often.

How to Start a Notion Import

The import interface is in the same place regardless of what format you're importing. Here's how to reach it:

Step 1

Open Notion on desktop or in a web browser. Mobile does not support imports — you must use the desktop app or the web app at notion.so in a browser.

Step 2

Click Settings & Members in the left sidebar. This opens the settings panel.

Step 3

In the settings sidebar, click Import. You will see all available import options listed on the right side of the panel.

Step 4

Choose your format — CSV, Markdown, Word, HTML, Evernote, Trello, Asana, or Monday.com. Notion's import wizard opens and guides you through uploading your file or connecting your account.

Step 5

After the import completes, Notion places the imported content in your workspace sidebar. Open it, review the result, and make any adjustments needed — property type changes, formatting cleanup, or re-uploading images that didn't transfer.

Imports on mobile are not supported. If you're attempting to import on iOS or Android and the option isn't visible, switch to a desktop browser or the Notion desktop app.

Importing CSV Files as a Notion Database

CSV import is the most versatile import method in Notion and the one you'll use most often when migrating structured data — client lists, project trackers, invoice records, contact databases. When you import a CSV, Notion treats each row as a database entry and each column header as a database property. The result is a full Notion database you can filter, sort, and view in any format.

The Import Process

  1. Go to Settings & Members → Import → CSV
  2. Select your .csv file from your computer
  3. Notion reads the first row as column headers and creates a database — each column becomes a property, each subsequent row becomes a page in the database
  4. After the import completes, open the new database in your workspace sidebar
  5. Click each property header to change property types from the default Text to the appropriate type (Select, Date, Number, Checkbox, etc.)

Best Practices Before You Import

  • Clean column headers — avoid special characters, parentheses, or slashes in column names. Simple names like "Status" or "Due Date" work better than "Status (2026)?"
  • Use YYYY-MM-DD for dates — Notion parses this format reliably. Formats like "07/11/2026" or "July 11" may not convert to a Date property correctly after import
  • UTF-8 encoding — if your CSV has international characters or accented letters, save it as UTF-8 before importing. In Excel: File → Save As → CSV UTF-8 (Comma delimited). In Google Sheets: File → Download → CSV (UTF-8 by default)
  • Remove blank rows — blank rows in your CSV become empty database entries in Notion
  • Split large files — CSVs with 10,000+ rows may be slow to import or fail entirely; split into multiple files of 2,000–5,000 rows for reliability

After a CSV import, all columns default to Text properties. Budget 10–20 minutes after a large import to go through the database and switch property types to Select, Date, Number, etc. This step makes the database actually useful for filtering and sorting.

Importing Markdown Files

Notion imports .md files as Notion pages, converting Markdown syntax into the equivalent Notion blocks. This is the primary migration path from Obsidian, Bear, Roam Research, iA Writer, Logseq, and any other Markdown-first notes tool.

The Import Process

  1. Go to Settings & Members → Import → Markdown & CSV
  2. Select a single .md file or a .zip archive containing multiple Markdown files
  3. Notion creates one page per Markdown file, placed in your workspace sidebar
  4. A ZIP of Markdown files creates a parent page with subpages — the folder structure inside the ZIP maps to nested pages in Notion

What Converts Well

  • Headings — # H1, ## H2, ### H3 become Notion heading blocks at the matching level
  • Bullet and numbered lists — convert to Notion list blocks
  • Bold and italic — **bold** and *italic* format correctly
  • Links — [text](url) becomes a clickable Notion link
  • Code blocks — fenced code blocks (```) become Notion code blocks with syntax highlighting
  • Blockquotes — convert to Notion quote blocks
  • Horizontal rules — convert to Notion dividers

What May Not Transfer

  • Obsidian wiki-links — [[Page Name]] style links appear as plain text in Notion; they do not become Notion page links automatically
  • Dataview queries — Obsidian's Dataview plugin syntax is not recognized by Notion's importer and appears as raw code
  • Custom callout syntax — Obsidian's > [!NOTE] callout format may not convert to Notion callout blocks
  • Embedded images with local paths — if your Markdown references local images (![alt](./image.png)), the images will not transfer; only externally hosted image URLs import correctly
  • YAML front matter — front matter blocks (between --- delimiters) may appear as plain text rather than being parsed as page properties

Importing Word Documents (.docx)

Notion's Word importer converts .docx files to Notion pages. This is the right approach when you're migrating SOPs, documentation, reports, or any content currently living in Microsoft Word or Google Docs (after downloading as .docx).

The Import Process

  1. Go to Settings & Members → Import → Word
  2. Select your .docx file from your computer
  3. Notion creates a new page with the converted content and places it in your sidebar

What Converts Well

  • Headings — Word heading styles (Heading 1, Heading 2, Heading 3) convert to Notion heading blocks at matching levels
  • Paragraphs and body text — standard paragraph text imports cleanly
  • Bold, italic, underline — inline formatting transfers correctly
  • Simple tables — basic single-row-header tables convert to Notion table blocks
  • Bullet and numbered lists — standard list styles convert to Notion list blocks

What May Not Transfer

  • Tracked changes and comments — only the final accepted text imports; tracked changes and Word comment threads do not carry over
  • Footnotes and endnotes — typically dropped or placed at the end as plain text without reference markers
  • Page breaks and section formatting — Notion is not page-based, so these layout elements don't apply and are ignored
  • Complex table styles — merged cells, colored cells, and custom table borders may not convert correctly
  • Embedded images — images in .docx files sometimes transfer and sometimes don't, depending on how they were embedded; plan to re-upload images manually after import if any are missing
  • Headers and footers — page-level headers and footers are not imported into Notion

Word import works best for text-heavy documents like SOPs, guides, and policy documents. If your Word file is primarily a layout or design document with complex tables, charts, or heavy visual formatting, copying and pasting sections manually may produce cleaner results than the importer.

Importing HTML Files

Notion can import HTML files as pages, parsing the HTML structure and converting elements to Notion blocks. This is useful for websites archived as HTML, notes from apps that export HTML, and CMS content exports.

The Import Process

  1. Go to Settings & Members → Import → HTML
  2. Select your .html file
  3. Notion converts the HTML structure to Notion blocks — headings, paragraphs, lists, and links transfer well from semantic HTML

HTML import quality depends heavily on how the HTML is structured. Clean, semantic HTML with proper <h1>, <h2>, <p>, and list tags converts well. HTML relying on CSS-based layout, JavaScript-rendered content, or non-semantic <div> soup may produce inconsistent results. If you only want the main content of a page, strip out navigation, sidebars, and footer HTML before importing — most HTML editors or even a text editor can do this in a few minutes.

Dedicated Tool Importers

Notion has built-in importers for four tools people most commonly migrate from. These handle the export-and-upload process in a more structured way than generic file imports, with better field mapping and structure preservation.

Tool Import Method What Transfers What Doesn't
Evernote Upload .enex export file Notebooks become pages; notes become subpages; text, checklists, and images transfer Evernote tags, inter-note links, and some PDF attachments may not transfer fully
Trello Upload .json board export Cards become database entries; list names become Select property values; card descriptions become page content Checklists within cards, card attachments, and member assignments are dropped
Asana OAuth authentication — no file needed Projects become databases; tasks become entries; task descriptions, due dates, and assignees transfer Custom fields, project timelines, and task dependencies may not fully transfer
Monday.com OAuth authentication — no file needed Boards become databases; items become entries; column values transfer where types match Notion's property types Monday automations, integrations, and complex column types (like mirrors) may not map correctly

Evernote Import Process

  1. In Evernote, go to File → Export Notes → select Evernote XML Format (.enex)
  2. Export each notebook as a separate .enex file (Evernote exports one notebook at a time)
  3. In Notion: Settings → Import → Evernote → upload your .enex file
  4. Notion creates a parent page for each notebook with individual notes as subpages

Trello Import Process

  1. In Trello, open your board → Board menu (three dots, top right) → More → Print and Export → Export as JSON
  2. Download the .json file to your computer
  3. In Notion: Settings → Import → Trello → upload the JSON file
  4. Notion creates a database where each card is an entry; the Trello list name (To Do, In Progress, Done) becomes a Select property value called "List"

Asana and Monday.com Import Process

Both use OAuth rather than file uploads. In Notion's Import menu, select Asana or Monday.com and click Connect. Authorize the connection when prompted in the popup window. Back in Notion, select which projects or boards to import and click Import. Notion pulls the data directly from the connected account — you don't need to export anything from the source tool first.

Importing from Airtable, Jira, and ClickUp (CSV Workaround)

Airtable, Jira, and ClickUp do not have native Notion importers. The workaround is to export from the source tool as CSV, then import the CSV into Notion as a database. This is less automated than a dedicated importer but it works reliably for structured data migration.

Step-by-Step: Airtable to Notion

Step 1 — Export from Airtable

In Airtable, open the table you want to migrate. Click the table name dropdown at the top → Download CSV. Airtable exports the current view's columns and rows. If you want all fields, make sure you're in a view that shows every field before exporting.

Step 2 — Import into Notion

In Notion: Settings → Import → CSV. Select the exported CSV file. Notion creates a database with all columns as Text properties.

Step 3 — Fix Property Types

Open the new database. For each column that should be a Select, Date, Number, Checkbox, or URL, click the column header → Edit property → change the type. Airtable's Single Select fields export as plain text — convert these to Select in Notion and add the dropdown options.

Step 4 — Recreate Views and Filters

Airtable views (Grid, Gallery, Kanban, Calendar) don't export in the CSV. You'll need to create equivalent views in Notion manually using the Add a view button at the top of the database. Kanban views in Notion use a grouped Gallery view based on a Select property.

Jira to Notion

In Jira, go to your project → Issues → use the Export button in the top-right corner of the issue list view → Export to CSV. (The exact location varies by Jira version — in cloud Jira, look for the Export button in the backlog or board's issue list.) Import the resulting CSV file into Notion via Settings → Import → CSV. Jira CSVs tend to have many columns — after import, delete the columns you don't need and set the correct property types for Status, Priority, and date fields.

ClickUp to Notion

In ClickUp, open a List or Space → click the three-dot menu in the upper right → Export → CSV. Import into Notion via the CSV importer. ClickUp exports include task names, statuses, assignees, due dates, and custom fields — all arrive as text properties that you then reconfigure to the appropriate property format in Notion.

What to Do When Imports Fail or Look Wrong

Notion imports rarely produce a perfect result on the first attempt, especially for large or complex files. Here are the most common issues and how to fix them.

Large File Fails or Hangs

If your CSV is 10MB or larger, or contains more than 10,000 rows, the import may time out or silently fail. Split the file into smaller chunks first. Open the CSV in Excel or Google Sheets, cut the rows into separate files of 2,000–5,000 rows each, and import them one at a time. You can combine multiple CSV imports into one database later using the Merge with DB option that appears when importing additional CSVs into an existing Notion database.

Encoding Errors — Characters Display as Garbled Text

If you see strange character sequences like â€™ instead of apostrophes, your file is not UTF-8 encoded. Fix in Excel: File → Save As → choose CSV UTF-8 (Comma delimited) from the format dropdown. Fix in Google Sheets: File → Download → Comma-separated values — Google Sheets always exports as UTF-8. Re-import the corrected file after saving.

Images Missing After Import

Notion's importers handle text content and convert formatting, but embedded images — particularly locally referenced ones in Markdown or Word files — often don't transfer. After any import that should have included images, plan to re-upload them manually by clicking the relevant block in Notion and using the Upload option. For large volumes of images, this is consistently the most time-consuming part of a migration.

Formatting Looks Wrong or Inconsistent

Non-standard Markdown syntax, unusual HTML markup, or complex Word styles may not convert correctly. The fastest fix is to clean up the source file before re-importing — simplify the Markdown, strip non-semantic HTML, or apply standard heading styles in Word before saving and importing again. For minor formatting issues on already-imported content, editing directly in Notion is usually faster than re-running the import.

Import Appears to Do Nothing

Notion occasionally fails silently — the import dialog closes but no new content appears in the sidebar. Check your workspace sidebar carefully; imported pages sometimes land with auto-generated names that are easy to miss. If nothing appears, try a smaller test file first to confirm the format is valid, then re-attempt the full import.

Import Is Running Slowly

Large databases with hundreds or thousands of rows may take several minutes to process after you click Import. Notion processes these in the background. Leave the Notion tab open and don't refresh the page — refreshing during import may interrupt the process. For very large CSVs, it's normal for the progress indicator to appear stuck for 3–10 minutes before the import completes.

Notion import is best suited for text content migration. Expect to spend some time cleaning up after any large import — rebuilding property types, re-uploading images, and fixing formatting edge cases. A structured cleanup pass is still much faster than rebuilding the content from scratch in Notion.

Frequently Asked Questions

How do I import a CSV file into Notion?

Go to Settings & Members → Import → CSV in Notion. Select your CSV file. Notion reads the first row as column headers and creates a database with each column as a Text property. After the import completes, open the database and change property types as needed — for example, converting a "Status" column to a Select property or a "Due Date" column to a Date property. For best results, clean your CSV first: ensure UTF-8 encoding, clean column headers with no special characters, and use YYYY-MM-DD for date values.

Can I import from Trello into Notion?

Yes — Notion has a dedicated Trello importer. First, export your Trello board as a JSON file (from Trello: Board menu → More → Print and Export → Export as JSON). Then in Notion, go to Settings → Import → Trello and upload the JSON file. Notion creates a database where each card becomes a database entry and each Trello list name becomes a value in a Select property.

Does Notion support importing from Obsidian?

Notion does not have a dedicated Obsidian importer, but since Obsidian stores notes as Markdown files (.md), you can use the Markdown import option. Export or copy your Obsidian vault's .md files (or zip the folder), then go to Settings → Import → Markdown in Notion and upload. Most standard Markdown formatting transfers well. Obsidian-specific features like wiki-links ([[Page Name]]), Dataview queries, and custom callout syntax will not convert — you will need to clean those up after import.

What file formats can Notion import?

Notion can import the following formats: CSV (converts to a database), Markdown (.md files — converts to pages), Word documents (.docx — converts to pages), HTML files (converts to pages), plain text (.txt), Evernote exports (.enex files), Trello board exports (.json), Asana projects (via OAuth), and Monday.com boards (via OAuth). Notion does not have native importers for Airtable, Jira, ClickUp, or Google Docs — for those, export to CSV and use the CSV importer.
