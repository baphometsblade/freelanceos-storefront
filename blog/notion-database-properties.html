Notion Database Properties: The Complete Guide to Every Property Type

Updated July 2026 · By FreelanceOS Team
Home › Blog › Notion Database Properties

Notion databases are only as powerful as their properties. Properties are the fields that define what data each row can store — the difference between a database that just holds names and one that tracks status, due dates, assigned people, budget, and linked client records. Understanding every property type is what separates a beginner Notion user from someone who builds systems that actually work.

This guide covers all Notion database property types, when to use each one, and how to use the advanced types — Formula, Relation, and Rollup — to make your databases work together.

What Are Notion Database Properties?

Properties are the columns in a Notion database. Every row (page) in a database can store data in each property. Properties appear as columns in Table view, as metadata on cards in Board and Gallery views, and as fields at the top of each database item when you open it as a full page.

To add a property: in Table view, click the + button at the right end of the column headers. In any other view, open a database item and click + Add a property.

Properties are shared across all views of the same database. Adding a property in Table view makes it available in Board, Calendar, Gallery, and List views of the same database too.

The Core Property Types (Use These Daily)

Text (default)
Basic

Free-form text. The default "Name" property is a special text property that serves as the item's title. Additional Text properties store notes, descriptions, or any unstructured text.

Number
Basic

Stores numeric values. Supports formatting as currency, percent, or with custom number formats. Use for prices, quantities, hours worked, word counts, scores. Can be used in Formulas.

Select
Basic

Choose one option from a predefined list of labeled, colored options. Use for mutually exclusive categories: Content Type, Project Stage, Client Tier, Priority Level. Each item can have only one value.

Multi-select
Basic

Choose multiple options from a predefined list. Use for tags, labels, skills, or anything where a row might belong to more than one category. Filterable and groupable like Select.

Status
Basic

A specialized Select property with three built-in groups: Not started, In progress, Done. Notion uses these groups to auto-organize Board views. Customize option names within each group.

Date
Basic

Stores a date and optional time. Supports date ranges with an "End date" toggle. Used by Calendar view for display. Add reminders directly on date properties.

Person
Collaboration

Assign workspace members to a database item. Used for task ownership, project leads, or client contacts who are Notion members. Each item can have multiple people assigned.

Files & Media
Basic

Upload or link files directly to a database item. Supports images, PDFs, videos, and other attachments. Useful for storing deliverables, contracts, or reference assets alongside project records.

Checkbox
Basic

A simple true/false toggle. Perfect for "Done?" fields, "Invoiced?" flags, or "Approved?" checklists. Filter your database by checked/unchecked to see only completed or pending items.

URL
Basic

Stores a web link that renders as a clickable URL. Use for client website links, source article URLs, reference links, or any external resource associated with the row.

Email
Basic

Stores an email address with mailto: link rendering. Use in CRM or client databases for quick email access directly from the database view without opening the item.

Phone
Basic

Stores a phone number with tel: link rendering for click-to-call on mobile. Use in contact or client databases.

The Advanced Property Types

Formula

Formula properties calculate a value based on other properties in the same database row. They use Notion's formula language — a function-based syntax similar to spreadsheet formulas.

What you can do with formulas:

  • Calculate total project value: prop("Rate") * prop("Hours")
  • Calculate days until deadline: dateBetween(prop("Due Date"), now(), "days")
  • Concatenate text: prop("First Name") + " " + prop("Last Name")
  • Flag overdue items: if(prop("Due Date") < now(), "OVERDUE", "On track")
  • Calculate percentage complete from a number: format(round(prop("Done") / prop("Total") * 100)) + "%"

Formula results can be text, numbers, dates, or checkboxes depending on what the formula returns. Formula properties are read-only — you cannot edit the result directly.

Relation

A Relation property creates a two-way link between two databases in your Notion workspace. Instead of typing a client's name as text, a Relation connects each project row to the actual client record in your Clients database.

How to set up a Relation:

  • Add a new property to your database and choose "Relation"
  • Select the database you want to link to
  • Choose whether to show the relation on the other database too (two-way) or only on this database (one-way)
  • On each row, click the Relation property to search and select items from the linked database

Relation properties are what enable Notion to function as a relational database system, not just a collection of isolated spreadsheets.

Rollup

A Rollup property reads values from a linked (related) database and performs an aggregation calculation across all linked items. Rollups require an existing Relation property to work.

Rollup examples:

  • On a Clients database: sum all Invoice amounts from a linked Invoices database to show total revenue per client
  • On a Projects database: count the number of linked Tasks to show how many tasks each project has
  • On a Clients database: show the latest date from linked Projects to see the most recent project date per client
  • On a Content database: sum word counts from linked Draft pages to see total words per content cluster

Available aggregations: count, count values, count unique values, count empty, count not empty, percent empty, percent not empty, sum, average, median, min, max, range, earliest date, latest date, date range, show original, show unique values.

Auto-Generated Properties (Read-Only)

These properties are automatically maintained by Notion and cannot be manually edited:

  • Created time — the exact timestamp when the database item was created
  • Created by — the workspace member who created the item
  • Last edited time — the timestamp of the most recent edit to the item
  • Last edited by — who made the most recent edit
  • ID — a unique auto-incrementing number for each database item. Useful for invoice numbers, ticket IDs, or any system that needs a stable numeric identifier per row.
Use the ID Property for Invoice Numbers

Add an ID property to your invoice database and Notion will auto-assign sequential numbers (INV-001, INV-002, etc.) to every invoice you create. No more manual numbering or risk of duplicates.

Specialized Properties

Button
Automation

A clickable button that triggers an automation. Configure it to add a page to another database, edit properties, create a new item, send a notification, or run a Notion AI action — all with one click from the database row.

AI Summary
AI (Notion Plus+)

Uses Notion AI to auto-generate a summary of the page content in a property field. Requires Notion AI add-on. Useful for automatically summarizing meeting notes or research pages in a database.

Select vs. Multi-Select vs. Status: When to Use Which

These three properties are often confused because they all involve choosing from a list of options. Here is the decision framework:

  • Use Select when each item belongs to exactly one category and categories are mutually exclusive. Examples: Project Type (Design / Development / Writing), Client Tier (A / B / C), Priority (High / Medium / Low).
  • Use Multi-select when each item can belong to multiple categories simultaneously. Examples: Tags (Notion / Freelance / Tutorial), Skills Required (Design / Copywriting / Strategy), Platforms (Instagram / LinkedIn / YouTube).
  • Use Status when you want Notion to auto-group items into workflow columns in Board view, and when your workflow fits the Not started / In progress / Done structure. Status works best for task and project workflows.

Tips for Designing Notion Database Properties

  • Start minimal — add only the properties you will actually fill in. Empty properties add visual clutter without value. You can always add more properties later.
  • Use descriptive property names — "Client (Relation)" instead of just "Client" so you remember at a glance that it is a linked relation, not just a text field.
  • Hide properties you rarely need — in view settings, hide properties that are rarely used from the table view. They still exist and can be revealed per-row, but they keep your views clean.
  • Do not use Text properties where Select would work better — if you find yourself typing the same values repeatedly (like "Instagram" or "In progress"), replace the Text property with a Select or Status property for consistency and filterability.
  • Add a Created time property to every important database — it automatically records when each item was created, which becomes invaluable for auditing and filtering later.

Frequently Asked Questions

What are Notion database properties?
Notion database properties are the columns (fields) in a Notion database. Each property stores a specific type of data — text, numbers, dates, selections, checkboxes, and more. Properties appear as columns in Table view, as metadata on item cards in Gallery and Board views, and as fields when you open any database item as a full page.
What is the difference between Select and Multi-select in Notion?
Select lets you choose one option from a predefined list — like a status or category where each item belongs to exactly one group. Multi-select lets you choose multiple options from a list — like tags, where an item can have several labels at once. Use Select for mutually exclusive categories and Multi-select for tags or labels that can overlap.
What is a Notion Relation property?
A Relation property links one database to another in your Notion workspace. For example, a Projects database can have a Relation to a Clients database, letting you connect each project to its client. Once linked, you can use Rollup properties to pull data from the related database — like summing all invoice amounts linked to a client.
What is a Notion Rollup property?
A Rollup property reads data from a linked (related) database and performs a calculation on it. For example: if a Clients database has a Relation to an Invoices database, a Rollup on the Clients database can sum all invoice amounts for each client, count the number of projects, or show the latest project date. Rollups require a Relation property to exist first.
What is the Status property in Notion?
The Status property is a specialized Select-style property with three built-in groups: Not started, In progress, and Done. These groups are color-coded and used by Notion's Board view to organize items into status columns automatically. You can customize the option names and add options within each group, but the three-group structure is fixed.
