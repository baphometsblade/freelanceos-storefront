Notion Gallery View: The Complete Guide to Visual Databases in Notion

10 min read • Updated July 2026 • FreelanceOS Team

Gallery view transforms any Notion database into a visual card grid. Here is how to set it up, customize it, and use it for portfolios, product catalogs, mood boards, and more.

What Is Notion Gallery View?

Notion Gallery View is one of Notion's six database layout types. Where table view shows data as rows and columns and board view groups cards by status, gallery view presents every database entry as a visual card in a responsive grid — with a large image or thumbnail at the top of each card.

Gallery view is the go-to choice for any database where images matter more than text: portfolios, product catalogs, recipe books, movie watchlists, inspiration boards, and client mood boards. Instead of scanning a list of names, you browse a wall of visual cards.

The core idea: Every page in a Notion database can have a cover image. Gallery view puts that cover image front and center on each card. The result feels more like a visual magazine spread or Pinterest board than a spreadsheet.

Gallery view has evolved significantly since its introduction. In 2026, gallery cards can display page covers, file attachments, property values, and page content previews — giving you fine-grained control over what each card shows and how each card looks.

When to Use Gallery View

Gallery view is the right choice when:

  • Your database entries have meaningful images (photos, designs, screenshots, thumbnails)
  • You want to browse content visually rather than scan a text list
  • You are building a portfolio, catalog, or inspiration board meant to be shared publicly
  • You are organizing content that users should be able to identify at a glance

Gallery view is not the right choice for data-heavy databases without images (use table), workflow management (use board or timeline), or databases where most entries lack images — blank cards look incomplete and undermine the gallery aesthetic.

How to Create a Notion Gallery View

Adding a gallery view to any existing Notion database takes under 30 seconds. Here is the step-by-step process:

Step 1 — Open your database

Navigate to any Notion database — either a full-page database or an inline database embedded in a page. If you do not have one yet, create a new page and type /database to add an inline database, or use /full page database for a standalone database.

Step 2 — Click the view menu

At the top left of your database, you will see the current view name (usually "Table" or "Default"). Click it, then click + Add a view at the top of the dropdown panel that appears.

Step 3 — Select Gallery

In the view type selector, click Gallery. A name field appears — you can rename this view (e.g., "Visual View" or "Portfolio Grid") or keep the default name. Click Done or press Enter to confirm.

Step 4 — Your gallery appears

Your database immediately switches to the gallery card grid. Each entry becomes a card. If your pages already have cover images, they appear on the cards automatically. If not, the cards show a gray placeholder at the top — we cover how to add images in a later section.

Step 5 — Access gallery view settings

Click the ... menu (or gear icon) next to the gallery view name to open view-specific settings. This is where you customize card size, card preview source, and which properties appear on each card. These settings only affect this gallery view — your other views remain unchanged.

Customizing Notion Gallery Cards

Gallery view has four main customization settings, all accessible from the view settings panel. Understanding each one gives you full control over how your gallery looks and feels.

Card Size

Notion offers three card size options in gallery view:

SizeCards Per Row (desktop)Best For
Small4–5Large catalogs, Pinterest-style boards, seeing many items at once
Medium3–4Most use cases — balanced image size and text visibility. The default.
Large2–3Portfolio showcases, hero product images, when image quality matters most

The right card size depends on your image content and how many entries you have. For a recipe database with clear food photos, medium or small works well. For a design portfolio where detail matters, large cards let your work breathe and make a stronger impression.

Card Preview

The card preview setting controls what image appears at the top of each gallery card. This is the single most impactful setting for gallery appearance.

Preview OptionWhat It ShowsWhen to Use
NoneNo image — cards show only text propertiesText-heavy databases where images are not relevant
Page coverThe page's cover image (set via the page header)When you use Notion's built-in Unsplash cover images
Page contentA preview of the first content block on the pageWhen pages have a relevant image or text at the very top
[Files property]The first image from a Files & Media propertyWhen you upload images directly to the database — the most reliable option

Best practice: For most visual databases, create a dedicated Files & Media property named "Image" or "Cover" and upload your images there. Then set Card Preview to that property. This gives you the most consistent, reliable image display across all gallery cards.

Image Fitting: Contain vs. Cover

When you have a card preview image, Notion offers two fitting modes:

  • Contain — the full image is visible within the card thumbnail area, with letterboxing if the aspect ratio does not match. No cropping. Best when image content near the edges matters (logos, diagrams, infographics).
  • Cover — the image fills the entire card thumbnail area, cropping edges to fit. No letterboxing. Produces a cleaner, more uniform look. Best for photos and design screenshots.

For most visual galleries, Cover mode produces a more professional appearance. Use Contain only when your images are logos or diagrams where edge content must not be cropped.

Properties Shown on Cards

Below the image on each gallery card, Notion shows a selection of database properties. You control which properties appear and their order via the Properties section of view settings.

Best practice: show 2–3 properties maximum on gallery cards. More than that clutters the card and defeats the visual purpose of the gallery layout. The most useful properties to surface on gallery cards:

  • A Select or Multi-select tag (category, type, status)
  • A Date (published date, creation date, deadline)
  • A Number (price, rating, duration)

Keep card text minimal. The image does the communicating; the properties provide just enough context for filtering and quick identification. Full details live inside the page, which opens with a single click.

Gallery View vs. Board View: Which Should You Use?

Gallery view and board view both display database entries as cards, but the difference is fundamental to how and when to use each:

FeatureGallery ViewBoard View
Card arrangementUniform grid, all cards equal sizeColumns grouped by a Select or Status property
Best forVisual browsing, image-first contentWorkflow management, status tracking
Primary emphasisImages and visual identityStage and status grouping (Kanban)
Moving cardsCannot drag to change propertiesDrag cards between columns to update status
Typical use casesPortfolio, catalog, mood board, media libraryProject tasks, editorial calendar, sales pipeline

The simple decision rule: if you need to see your content visually, use Gallery. If you need to move items through stages, use Board. Many advanced Notion users add both views to the same database for different purposes.

How to Add Cover Images to Gallery Cards (3 Methods)

A gallery without images is just a clunky grid. Here are the three ways to get images onto your gallery cards, from easiest to most powerful:

Method 1 — Upload to a Files Property (Most Reliable)

  1. Add a Files & Media property to your database (click + in the properties header, select Files & Media)
  2. For each database entry, click the Files property cell and upload your image (drag from desktop, paste from clipboard, or use the file picker)
  3. In gallery view settings, set Card preview to that Files property

Every card now shows its uploaded image. Consistent, clean, and fully under your control.

Method 2 — Set a Page Cover (Quickest for New Databases)

  1. Open any database entry (click the page title to open it)
  2. Hover at the top of the page — an Add cover prompt appears
  3. Click it and choose: upload an image, paste a URL, or pick from Unsplash (free)
  4. In gallery view settings, set Card preview to Page cover

Unsplash covers are free, high-quality, and take five seconds to set. A strong choice for databases like reading lists or movie watchlists where visual consistency matters more than custom photography.

Method 3 — Paste an Image URL

For databases pulling content from the web (products, articles, films), you can paste image URLs into a URL or Text property. Some Notion automation tools (Zapier, Make, Notion AI) can auto-populate cover images from a URL field, creating an automated image pipeline. This is the most advanced method but produces the most scalable image management for large databases.

Quick tip for recipe databases: Search the dish name on Google Images, right-click a food photo, copy the image address, and paste it into a Notion Files property. About 10 seconds per recipe. For 50 recipes, roughly 8 minutes total — a worthwhile investment for a beautiful visual cookbook.

Gallery View Tips and Best Practices

  • Use cover images consistently — a gallery with 80% images and 20% blank cards looks worse than a gallery with no images at all. Either commit to adding images to every entry or switch to a different view layout.
  • Limit visible properties to 2–3 per card — the gallery card is a teaser, not an information panel. Show the name, one tag, and one date. Let the full page hold the detailed data.
  • Create multiple views on the same database — add a gallery view AND a table view. Use the gallery for visual browsing and the table for bulk property editing. They share identical underlying data.
  • Use filters to curate focused galleries — filter by Status = Published for a public portfolio. Filter by Category = "Web Design" for a niche showcase. Filters transform a messy database into a curated visual experience with no data duplication.
  • Sort by Date to keep galleries fresh — sort by Created Time (descending) or a custom Date property to always show the newest items first. Essential for portfolios and content libraries.
  • Use Large card size when sharing publicly — when you share a gallery page via Notion's "Share to web" toggle, large card size gives your work the strongest visual impact on desktop screens.
  • Group by Select property for sectioned galleries — Notion allows optional grouping in gallery view. Group a portfolio by Project Type, a recipe database by Cuisine, or a product catalog by Category. This creates horizontal visual sections that act as a built-in navigation index.

Frequently Asked Questions

What is Notion Gallery View?
Notion Gallery View is a database layout that displays each entry as a visual card in a responsive grid. Unlike table or list views, gallery view emphasizes images — each card can show a cover image, file attachment, or page content thumbnail. It is ideal for portfolios, product catalogs, recipe collections, and mood boards where visual browsing matters more than data analysis.
How do I switch to Gallery View in Notion?
Open any Notion database and click the view name at the top left (e.g., "Table"). Click "+ Add a view," then select "Gallery" from the layout options. Your database immediately switches to the card grid layout. Customize card size, image source, and visible properties in the view settings panel accessible from the "..." icon next to the view name.
How do I add images to Notion Gallery View?
Add a Files and Media property to your database and upload images there, then set the gallery card preview to that property. Alternatively, set a page cover image on each entry and set card preview to "Page cover." Notion supports free Unsplash images as page covers — available in seconds and high quality.
What is the difference between Gallery View and Board View?
Gallery view shows all entries as equal visual cards in a uniform grid — best for image browsing. Board view groups entries into columns based on a Status or Select property (Kanban-style) — best for tracking workflow stages. Use gallery for visual content like portfolios and catalogs; use board for process management like project tasks or editorial calendars.
Can I use Notion Gallery View as a portfolio?
Yes — gallery view is one of the most popular ways to build a visual portfolio in Notion. Create a database with a Files and Media property for project screenshots, upload your work samples, set card preview to that property, add Select tags for project type and year, and share the page publicly via Notion's share settings.
