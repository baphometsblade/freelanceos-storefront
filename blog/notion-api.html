Notion API: The Complete Beginner's Guide to the Notion API (2026)

The Notion API unlocks a completely different way to use Notion — not just as a note-taking app, but as a database you can read from and write to programmatically. Whether you want to push new Stripe payments into a Notion income tracker, sync GitHub issues to your project board, or build a public website powered by Notion data, the API makes it possible.

This guide walks you through everything: setting up your first integration, authenticating, querying databases, creating pages, and 8 practical things you can build. No prior API experience needed.

1. What Is the Notion API?

The Notion API is an official REST API published at developers.notion.com. It allows you — or any authorized tool — to interact with your Notion workspace over HTTP. Instead of clicking in the Notion UI, you send requests and receive responses in JSON.

What the Notion API lets you do:

  • Query databases — pull all rows (pages) from a Notion database with filters and sorts
  • Create pages — add new entries to databases or new pages inside other pages
  • Update page properties — change the value of any property on an existing page
  • Append block content — add paragraphs, code blocks, bullet lists, images to a page
  • Search your workspace — find pages and databases by keyword
  • Retrieve users — get info about workspace members

Common use cases for the Notion API:

  • Notion as a CMS: Store blog posts or product data in Notion, pull them via API to render on a website
  • Automation hubs: When something happens in another app (new Stripe payment, new GitHub issue, new form submission), automatically create a record in Notion
  • Data sync: Keep Notion databases in sync with external data sources like Google Sheets, Airtable, or SQL databases
  • Custom dashboards: Read from multiple Notion databases and combine data in ways the Notion UI doesn't support
  • Scheduled updates: Run a script every night to update properties like "Days Since Last Contact" or "Revenue This Month"
Who this guide is for: Freelancers, solopreneurs, and developers who use Notion and want to connect it to other tools or automate repetitive database updates. You need basic familiarity with Notion databases. Coding examples use Python — the concepts apply to any language.

2. Getting Started with the Notion API

Before you can make any API call, you need to create a Notion integration and share your pages with it. Here's the exact process:

Step 1
Create a Notion Integration

Go to notion.com/my-integrations. Click "+ New integration". Give it a name (e.g., "My Automation" or "Invoice Logger"). Select the workspace it should access. Leave capabilities at the default (Read content, Update content, Insert content). Click Submit.

Step 2
Copy Your Integration Token

After creating the integration, you'll see an Internal Integration Secret (sometimes called the API key or token). It starts with ntn_ or secret_. Click "Show" then copy it. Store it securely — treat it like a password. Never put it in client-side JavaScript or commit it to a public GitHub repo.

Step 3
Share Pages/Databases with Your Integration

This is the most commonly missed step. By default, your integration cannot access anything. For each Notion page or database you want to access via API: open it in Notion → click the "..." (three dots) in the top right → click "Connections" → search for and select your integration. The integration now has access to that page and everything nested inside it.

Step 4
Make Your First API Call

Test that everything is working by fetching your own user info. In a terminal, run this curl command (replace YOUR_TOKEN with your actual token):

curl -X GET 'https://api.notion.com/v1/users/me' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Notion-Version: 2022-06-28'

If successful, you'll get back a JSON object with your Notion user information. If you see a 401 error, double-check your token. If you see a 403 error, the integration hasn't been shared with any pages yet.

3. Notion API Authentication

Every Notion API request requires two headers:

HeaderValuePurpose
AuthorizationBearer YOUR_INTEGRATION_TOKENAuthenticates your integration
Notion-Version2022-06-28Specifies which API version to use
Content-Typeapplication/jsonRequired for POST/PATCH requests with a body

The Notion API uses Bearer token authentication. Your integration token is the bearer token. It grants the API the same access level as your integration — which is limited to only the pages you've explicitly shared with it.

Security note: Store your token in an environment variable, not hardcoded in your script. In Python: import os; token = os.environ.get("NOTION_TOKEN"). Set the variable in your shell with export NOTION_TOKEN="your-token-here" or use a .env file with the python-dotenv package.

In Python, a reusable headers dictionary looks like this:

import os

NOTION_TOKEN = os.environ.get("NOTION_TOKEN")

headers = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Notion-Version": "2022-06-28",
    "Content-Type": "application/json"
}

4. Reading a Notion Database

The most common Notion API operation is querying a database — pulling all its rows so you can read or display them. The endpoint is POST /v1/databases/{database_id}/query.

Finding Your Database ID

Open your Notion database in a browser. The URL looks like:

https://www.notion.so/yourworkspace/XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX?v=YYYYYYYYYYYY

The database ID is the 32-character string after the last slash and before the ?v=. It often looks like 1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d (though it may include hyphens in some URL formats).

Basic Database Query in Python

import requests
import os

NOTION_TOKEN = os.environ.get("NOTION_TOKEN")
DATABASE_ID = "your-database-id-here"

url = f"https://api.notion.com/v1/databases/{DATABASE_ID}/query"
headers = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Notion-Version": "2022-06-28",
    "Content-Type": "application/json"
}

response = requests.post(url, headers=headers)
data = response.json()

# Print the title of each page in the database
for page in data["results"]:
    # The title property is usually called "Name" or "Title"
    title_prop = page["properties"].get("Name", {})
    title_parts = title_prop.get("title", [])
    title = title_parts[0]["text"]["content"] if title_parts else "(no title)"
    print(f"- {title} (ID: {page['id']})")

Adding Filters to Your Query

To filter results, add a filter object to your request body. Filters are powerful — you can filter by any property type with conditions specific to that type.

# Filter for pages where Status is "In Progress"
body = {
    "filter": {
        "property": "Status",
        "status": {
            "equals": "In Progress"
        }
    }
}

response = requests.post(url, headers=headers, json=body)
data = response.json()

Sorting Results

# Sort by Due Date ascending
body = {
    "sorts": [
        {
            "property": "Due Date",
            "direction": "ascending"
        }
    ]
}

# Combine filter + sort
body = {
    "filter": {
        "property": "Status",
        "status": {"equals": "In Progress"}
    },
    "sorts": [
        {"property": "Due Date", "direction": "ascending"}
    ]
}

5. Creating a Page in Notion via API

To create a new row in a Notion database (or a new page nested inside another page), use POST /v1/pages. You specify the parent (the database) and the properties to set.

import requests
import os

NOTION_TOKEN = os.environ.get("NOTION_TOKEN")
DATABASE_ID = "your-database-id-here"

url = "https://api.notion.com/v1/pages"
headers = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Notion-Version": "2022-06-28",
    "Content-Type": "application/json"
}

# Create a new task in a Tasks database with Name, Status, and Due Date
new_page = {
    "parent": {"database_id": DATABASE_ID},
    "properties": {
        "Name": {
            "title": [
                {
                    "text": {"content": "Review Q3 invoices"}
                }
            ]
        },
        "Status": {
            "status": {"name": "In Progress"}
        },
        "Due Date": {
            "date": {"start": "2026-07-15"}
        }
    }
}

response = requests.post(url, headers=headers, json=new_page)
result = response.json()
print(f"Created page: {result['id']}")

6. Updating and Deleting Pages via API

To update an existing page's properties, use PATCH /v1/pages/{page_id}. You only need to include the properties you want to change — other properties are left untouched.

import requests
import os

NOTION_TOKEN = os.environ.get("NOTION_TOKEN")
PAGE_ID = "the-page-id-to-update"  # Get this from a query response

url = f"https://api.notion.com/v1/pages/{PAGE_ID}"
headers = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Notion-Version": "2022-06-28",
    "Content-Type": "application/json"
}

# Mark a task as Done
update = {
    "properties": {
        "Status": {
            "status": {"name": "Done"}
        },
        "Done": {
            "checkbox": True
        }
    }
}

response = requests.patch(url, headers=headers, json=update)
print(f"Updated: {response.status_code}")

Archiving (Deleting) a Page

The Notion API doesn't permanently delete pages — it archives them (moves them to Trash). To archive a page:

update = {"archived": True}
response = requests.patch(url, headers=headers, json=update)

To restore an archived page, set "archived": false.

Practical Automation: Mark Task Done When External Event Fires

A common pattern is closing out a Notion task when something happens externally — for example, when a client pays an invoice in Stripe, automatically mark the corresponding Notion invoice row as "Paid":

  1. Stripe sends a webhook when a payment succeeds
  2. Your webhook handler (Cloudflare Worker, AWS Lambda, or simple Python Flask) receives it
  3. It queries your Notion Invoices database, filtering for the matching invoice number
  4. It patches that page's Status to "Paid" and sets a "Paid Date" property to today

7. Using the Official Notion Python SDK

The official Notion Python client (notion-client) wraps the raw HTTP API with a cleaner, more Pythonic interface. Install it:

pip install notion-client

Here's a complete working script that queries a database, prints all rows, creates a new row, and updates a property — all using the SDK:

from notion_client import Client
import os

notion = Client(auth=os.environ.get("NOTION_TOKEN"))
DATABASE_ID = "your-database-id"

# 1. Query all pages in the database
results = notion.databases.query(database_id=DATABASE_ID)
pages = results["results"]

print(f"Found {len(pages)} pages:")
for page in pages:
    title_parts = page["properties"]["Name"]["title"]
    title = title_parts[0]["text"]["content"] if title_parts else "(no title)"
    print(f"  - {title}")

# 2. Create a new page (row)
new_page = notion.pages.create(
    parent={"database_id": DATABASE_ID},
    properties={
        "Name": {
            "title": [{"text": {"content": "New task from Python"}}]
        },
        "Status": {
            "status": {"name": "Not Started"}
        }
    }
)
print(f"Created: {new_page['id']}")

# 3. Update a property on the new page
notion.pages.update(
    page_id=new_page["id"],
    properties={
        "Status": {"status": {"name": "In Progress"}}
    }
)
print("Updated status to In Progress")
SDK vs raw requests: For most projects, the SDK is cleaner and handles edge cases better. Use raw requests calls when you're working in a language without an official SDK, writing a quick one-liner, or deploying to an environment where you can't install packages.

Handling Pagination

Notion API queries return at most 100 results per page. For larger databases, you need to paginate using the start_cursor parameter:

all_pages = []
cursor = None

while True:
    kwargs = {"database_id": DATABASE_ID}
    if cursor:
        kwargs["start_cursor"] = cursor
    
    response = notion.databases.query(**kwargs)
    all_pages.extend(response["results"])
    
    if not response["has_more"]:
        break
    cursor = response["next_cursor"]

print(f"Total pages: {len(all_pages)}")

8. Eight Things to Build with the Notion API

Here are eight real, practical integrations worth building — ranging from simple to advanced:

Build 1
Auto-log Stripe Payments to a Notion Income Tracker

When a Stripe payment_intent.succeeded webhook fires, parse the amount, customer email, and product name, then create a row in your Notion Income database. Pairs perfectly with a Cloudflare Worker that already handles your Stripe webhooks.

Build 2
Sync GitHub Issues to a Notion Project Database

Use GitHub Actions or a GitHub webhook to push new issues into Notion. When an issue is opened, create a Notion page. When it's closed, update the Status. Keeps your Notion project board in sync with your codebase.

Build 3
Create a Page When You Get an Email from a Specific Sender

Use Gmail API + Notion API (or a no-code tool like Zapier or Make.com): when Gmail receives an email with a specific label (like "Client Request"), automatically create a Notion task with the email subject, sender, and a link to the email. Never lose a client request again.

Build 4
Build a Public Website Powered by Notion

Use Notion as a headless CMS: store blog posts in a Notion database with Title, Slug, Body (rich text), Published Date, and Status properties. At build time, query the API to fetch all Published posts and render them as HTML. Many freelancers use this pattern for their portfolio or blog — powered entirely by Notion, no WordPress needed.

Build 5
Auto-Update a Notion Status When a Zapier Action Completes

Use Zapier's "Code by Zapier" step to call the Notion API at the end of a Zap. This means you can chain: form submission → email sent (Zapier) → Notion row updated to "Email Sent". The Notion API becomes a reporting destination for your entire automation stack.

Build 6
Create Daily Journal Entries Automatically at Midnight

Schedule a script (using GitHub Actions cron, a Raspberry Pi cron job, or a free Replit Deployment) to run at midnight each night. It creates a new page in your Notion Daily Journal database with today's date, pre-filled with your reflection prompt template as page content using the blocks API.

Build 7
Sync Google Sheets Data to Notion Weekly

Pull data from a Google Sheet using the Sheets API and push it to Notion via the Notion API. Useful when clients send you data in spreadsheets that you want to track in Notion. Run it weekly via a scheduled Python script.

Build 8
Build a Slack Bot That Creates Notion Tasks

Deploy a Slack app that listens for a /task slash command. When a team member types /task Review proposal for Client X by Friday, the bot parses the text, creates a Notion task with the right properties, and responds with a link to the new task. The Notion API is the write endpoint; Slack's API is the trigger.

9. Notion API Rate Limits and Common Errors

Rate Limits

The Notion API limits you to 3 requests per second per integration. This is plenty for most personal automations but can be hit when syncing large datasets.

How to handle rate limiting in Python:

import time
import requests

def notion_request_with_retry(method, url, headers, json=None, max_retries=3):
    for attempt in range(max_retries):
        if method == "POST":
            response = requests.post(url, headers=headers, json=json)
        elif method == "PATCH":
            response = requests.patch(url, headers=headers, json=json)
        else:
            response = requests.get(url, headers=headers)
        
        if response.status_code == 429:
            wait = 2 ** attempt  # Exponential backoff: 1s, 2s, 4s
            print(f"Rate limited. Waiting {wait}s...")
            time.sleep(wait)
            continue
        
        return response
    
    raise Exception("Max retries exceeded")

Common Errors and What They Mean

Status CodeMeaningFix
400Bad request — invalid JSON bodyCheck your property names and value shapes. Print the response body to see exactly what Notion rejected.
401Unauthorized — bad tokenVerify your Integration Token is correct and complete. Make sure you're not accidentally adding extra spaces.
403Forbidden — integration not sharedGo to the Notion page/database → "..." → "Connections" → add your integration.
404Page not foundCheck the database/page ID. The most common cause: you shared a parent page but not the database, or you're using the wrong ID from the URL.
409Conflict — typically a stale cursorRe-query from the beginning rather than resuming from a cached cursor.
429Rate limitedSlow down requests. Add retry logic with exponential backoff (see above).
500Internal Notion server errorRetry after a few seconds. If persistent, check Notion's status page at status.notion.so.

The Most Common Gotcha: Wrong Property Name Case

Notion property names are case-sensitive and must match exactly. If your property is called "Due Date" and you write "due date" in your API call, you'll get a 400 error. Always copy-paste property names from Notion or from a test query response.

Debugging tip: When your API call fails, first query the database with no filters and print the full JSON response. Look at the shape of one page's properties object — this shows you exactly what property names and value shapes Notion expects.

FAQ: Notion API

What is the Notion API?

The Notion API is an official REST API from Notion that allows developers and automation tools to read from and write to Notion pages and databases programmatically. You can query databases, create pages, update properties, and build full integrations using HTTP requests or the official Notion SDK.

How do I get a Notion API key?

Go to notion.com/my-integrations, click "Create new integration", give it a name and select a workspace, then copy the "Internal Integration Token" — that is your API key. Keep it secret and never expose it in client-side code or public repositories.

Is the Notion API free?

Yes, the Notion API is free for all Notion users including the free plan. There are rate limits (3 requests per second per integration) but no per-request cost. You only pay for Notion itself if you need team or enterprise features beyond the free plan.

Can I use the Notion API to build a website?

Yes. Many developers use Notion as a headless CMS — they store content (blog posts, product info, team profiles) in Notion databases, then query the API at build time or request time to render it on a website. Popular combinations include Notion API + Next.js, Notion API + Astro, and Notion API + Vercel Edge Functions.

What can I automate with the Notion API?

You can automate almost any Notion action: log new Stripe payments to an income tracker, sync GitHub issues to a project database, create daily journal entries, push form submissions into a CRM, update task statuses when external events happen, and build Slack bots that create Notion tasks from slash commands.

How do I query a Notion database with the API?

Use POST /v1/databases/{database_id}/query with your integration token in the Authorization header. The database_id comes from the Notion URL. You can add a "filter" object to filter by property values and a "sorts" array to order results. The response returns a list of page objects with all their properties.
