Notion Linked View: Display Any Database on Any Page with Custom Filters

By FreelanceOS · July 2026 · 7 min read

What Is a Notion Linked View?

A Notion linked view (also called a "linked database") is a powerful display of an existing database that you can embed on any page in your workspace. Think of it as a window into a database — you're not copying the data, you're creating a filtered, custom display of it that lives anywhere you want.

The key advantage: each linked view can have its own independent filters, sorts, and layout choices, completely separate from the source database and from other linked views. You can show the same database 10 different ways on 10 different pages, and all those views pull from the exact same underlying data.

The "one database, many views" pattern: Build one master Tasks database. Create linked views on your Weekly Dashboard (filtered to this week), your Project pages (filtered to each project), your Team page (grouped by assignee), and your Done page (filtered to completed). Same data — four focused windows into it.

Linked View vs. Synced Block: What's the Difference?

FeatureLinked ViewSynced Block
What it connects toA databaseA block of page content
Can be filtered independentlyYesNo — content is identical everywhere
Editing source data updates all instancesYesYes
Layout can vary per instanceYesNo
Best forDashboards, project pages, client viewsTemplates, repeated notices, shared headers

Use linked views for database data you want to display with context-specific filters. Use synced blocks for non-database content that should look identical everywhere — like a recurring meeting agenda, a team notice, or a template header.

How to Create a Linked View in Notion

Step 1 — Open the page where you want the linked view

Navigate to any Notion page — a project page, a client page, your dashboard, or any doc. Click into the page body where you want the database to appear.

Step 2 — Type the linked view command

Type /linked or /link and select "Create linked view of database" from the menu that appears. A search box opens — type the name of the database you want to link.

Step 3 — Select your database

Click the database from the search results. Notion embeds a full view of that database on your page, defaulting to a table layout showing all entries with all default properties visible.

Step 4 — Choose a layout

Click the view name at the top left of the linked database → "+ Add a view" to change to a different layout: table, board, calendar, list, gallery, or timeline. Choose the layout that makes most sense for the context of this page.

Step 5 — Apply filters specific to this page

Click Filter → + Add filter. For example, on a project page, filter to show only entries where Project = "[this project name]." On a "This Week" dashboard, filter to show only entries where Due Date is within the next 7 days. These filters only apply to this linked view — the source database and other linked views are unaffected.

Step 6 — Adjust visible properties and hide clutter

Click Properties to toggle which columns appear in this view. On a project page, you might show Task Name, Assignee, Status, and Due Date — and hide all the other properties that are only relevant in a master overview. Cleaner view = faster scanning = better decisions.

Filtering a Linked View for Maximum Usefulness

Filters are what make linked views genuinely powerful. A linked view without filters is just a duplicate of the source database. A linked view with well-chosen filters turns a general database into a precise, context-specific tool.

Common Filter Patterns

  • Filter by Relation property — if your Tasks database has a "Client" relation property, filter linked views on each client page to show only that client's tasks. This is the most powerful pattern in Notion system design.
  • Filter by date range — show items due this week, this month, or overdue. Date filters are dynamic — they update automatically as time passes.
  • Filter by Status or Select property — show only active items, in-progress projects, or unpaid invoices. Status filters turn a full database into a focused action list.
  • Filter by Assignee / Person property — useful for team workspaces to show only each person's assigned tasks on their profile page.
  • Multi-condition filters — combine filters with AND/OR logic. Example: (Status = "In Progress" AND Due Date = This Week) shows only urgent active tasks. (Client = "Acme" OR Client = "Beta Corp") shows two clients on a shared agency overview.

Filter tip: When building a linked view filtered by a Relation property, the filter is usually "Client → is → [specific client page]." This requires the relation already exists in your database schema. Design your database relations first — linked views become dramatically more useful when your data is well-connected.

Common Mistakes with Linked Views

  • Forgetting to apply filters — an unfiltered linked view shows everything in the source database, which is almost never what you want. Always set at least one filter after creating a linked view.
  • Creating too many separate databases instead of linked views — a common beginner mistake is creating a new Tasks database for every project. Build one Tasks database with a Project relation property and use linked views to see each project's tasks. One database = one source of truth.
  • Confusing layout changes with data changes — changing a linked view from table to gallery layout doesn't change the data or affect other views. It only changes how this specific linked view displays. Data changes (editing property values, adding entries) do affect the source database.
  • Not hiding irrelevant properties — in the linked view settings, hide properties that aren't useful in this specific context. A task list on a project page doesn't need to show the Client property — the client is implied by the page context.

Frequently Asked Questions

What is a Notion linked view?
A Notion linked view (also called a linked database) is a display of an existing database embedded on any page in your workspace. It shows data from the source database but can have its own independent filters, sorts, and layout — giving you a custom view of the same data in any context. Changes to data made through a linked view update the source database immediately.
How do I create a linked database view in Notion?
Type /linked on any Notion page and select "Create linked view of database." Search for and select the database you want to display. The full database appears. Then apply filters, choose a layout (table, board, gallery, etc.), and toggle visible properties to customize the view for that specific page's context.
What is the difference between a Notion linked view and a synced block?
A linked view connects to a database and lets you apply filters to display a subset of its entries in any layout. A synced block copies non-database content (text, images, embeds) and keeps it visually identical across multiple pages. Use linked views for database data with context-specific filters; use synced blocks for repeating content like meeting templates or team notices.
Can I filter a Notion linked view differently than the source database?
Yes — this is the core benefit. Each linked view has completely independent filters, sorts, and layout settings. The source database can show all 200 tasks unfiltered, while a linked view on your weekly dashboard shows only tasks due this week, and a linked view on a client page shows only that client's tasks. All three pull from the same database; none affects the others' filters.
Does editing through a linked view change the original database?
Yes — editing property values, adding entries, or deleting entries through a linked view modifies the source database and those changes appear everywhere the database or its linked views are displayed. Changing the layout, filters, or visible properties of a linked view only affects that specific view instance — it does not modify the source database structure or other linked views.
