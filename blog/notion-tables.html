What Is a Notion Database? (vs. a Simple Table)

Notion has two table-like things and it is important to know the difference.

A simple table (/table in Notion) is a basic visual grid — rows and columns, similar to a Google Sheets table you embed in a document. It has no structured data types, no views, and no individual page per row. Think of it as a formatting tool, not a data system.

A Notion database (/database in Notion) is a structured collection of pages, where each row is actually a full Notion page with its own content. Databases have:

  • Typed properties — date pickers, checkboxes, select menus, people fields, formulas, and 14 more types
  • Multiple views — see the same data as a table, board, calendar, gallery, list, or timeline
  • Filtering and sorting — per view, saved automatically
  • Relations and rollups — link databases to each other and calculate across them
  • Templates — pre-fill new rows with standard content
  • Automation — trigger actions when properties change

Rule of thumb: Use a simple table for quick reference data (a list of links, a comparison of two options). Use a database for anything you want to track, filter, sort, relate to other data, or look at in more than one way.

Everything in your Notion workspace — your task list, your client tracker, your content calendar, your reading list, your CRM — should be a database, not a simple table. Databases scale with you. Simple tables don't.

Creating Your First Database

There are three ways to create a Notion database, each suited to a different situation.

Option 1

Full-page database

Type /database in any page and select "Database - Full page." This creates a standalone database page in your sidebar. Best for large systems (a CRM, a projects tracker) that deserve their own page.

Option 2

Inline database (embedded in a page)

Type /database and select "Database - Inline" — or type /table to jump straight to an inline table view. The database lives inside the current page. Best for dashboards where you want multiple databases visible on one page.

Option 3

Linked database (view of an existing database)

Type /linked or use "Create linked database" to embed a view of an existing database anywhere in your workspace. The data is shared — the same database, shown in a different context. This is how dashboards in Notion work: one master database, multiple linked views on your home page filtered by date, status, or project.

Adding properties to your database

After creating the database, click the + button at the right edge of the column headers (in Table view) to add a new property. Choose a property type from the list, name it, and configure its options. Every database starts with a Title property that cannot be removed — that is the name of each page. Add as many additional properties as you need.

Database Properties Explained: All 18 Types

Notion offers 18 property types. Here is every one of them with the best use case for each.

Text
Basic Input

Free-form text. Use for short notes, descriptions, subtitles, or URL fields you want as plain text.

Number
Basic Input

Integers or decimals. Use for revenue, hours, price, quantity, word count. Supports formatting (currency, percent, etc.).

Select
Single Choice

One option from a pre-defined list. Use for Status, Priority, Category, Stage. Each option gets its own color.

Multi-select
Multiple Choices

Multiple options from a list. Use for Tags, Skills, Content Types, or anything that can have more than one value.

Status
Workflow

Like Select but pre-set to a Not Started / In Progress / Done model. Integrates with Notion's status tracking features.

Date
Time

Date and optional time. Enable "End date" for date ranges (great for Timeline and Calendar views). Use for deadlines, publish dates, start dates.

Person
Team

Assigns workspace members to a row. Use for Assignee, Owner, Reviewer. Multiple people can be added.

Files & Media
Attachments

Upload files or images directly to a database property. Use for storing attachments, screenshots, or documents per row.

Checkbox
Boolean

True or false. Use for Done, Approved, Invoiced, Published — any binary yes/no field. Filterable and sortable.

URL
Link

Stores a clickable link. Use for website URLs, Loom recordings, LinkedIn profiles, source links in a research database.

Email
Contact

Stores an email address as a clickable mailto: link. Use in CRM or contact databases.

Phone
Contact

Stores a phone number. Use in CRM or contact databases.

Formula
Calculated

Calculates a value from other properties using Notion's formula syntax. Use for profit (revenue minus cost), days until deadline, conditional labels, or full name from first + last name.

Relation
Linked Database

Links rows in this database to rows in another database. Use to connect Projects to Tasks, Clients to Projects, or Blog Posts to Authors. The foundation of Notion's relational database system.

Rollup
Calculated

Requires a Relation property. Calculates an aggregate across linked items: count, sum, average, earliest/latest date. Use to count tasks per project, total revenue per client, or average rating across linked items.

Created time
Automatic

Auto-populated with the date and time the row was created. Cannot be edited. Use for sorting by recency in a notes or ideas database.

Created by
Automatic

Auto-populated with the workspace member who created the row. Use in team databases to track who added each entry.

Last edited time
Automatic

Auto-populated with the last time the row or its page content was modified. Use to sort a notes database by "most recently updated."

The 6 Database Views (When to Use Each)

Every Notion database can show the same data in up to six different formats. Add as many views as you need — each view is a separate tab at the top of the database, with its own filters, sorts, and visible properties.

📋

Table View

A spreadsheet-style grid. The default view and the best one for data entry, bulk editing, and seeing many properties at once. Use as your primary editing view. Best for: CRMs, task trackers, any database where you regularly add or edit rows.

📌

Board View (Kanban)

Cards organized in columns by a Select property (typically Status). Drag cards between columns to update the property. Best for: task management, content pipeline, sales CRM stages, editorial workflows.

📅

Calendar View

Items shown on a monthly calendar grid by a date property. Best for: content publishing schedules, deadlines tracker, event planning, editorial calendar at a glance.

🖼️

Gallery View

Items shown as image cards in a grid. Great when items have cover images or when visual scanning matters. Best for: design portfolio, product catalog, template library, inspiration board.

📝

List View

A simple stacked list with minimal properties shown. The cleanest and fastest view to scroll. Best for: reading lists, reference notes, quick lookup databases, mobile-friendly views.

📊

Timeline View

Gantt-style bars on a time axis. Items display as duration bars based on start and end date. Best for: project roadmaps, content calendars, sprint planning, event timelines. See our complete Notion Timeline guide for a full setup walkthrough.

Filtering and Sorting Your Notion Database

Filters

Click Filter at the top of any database view and click "+ Add filter." Choose a property, select a condition, and enter a value. Common filter conditions:

  • Status is "In Progress" — shows only active tasks
  • Assignee is Me — shows only your items
  • Due Date is before Today — shows overdue items
  • Tags contains "SEO" — shows tagged content
  • Checkbox is unchecked — shows incomplete items

Add multiple filters and join them with AND (item must match all) or OR (item must match any). Filter groups let you combine AND and OR logic. Filters are saved per view and do not affect other views of the same database.

Sorting

Click Sort at the top of the database and choose a property to sort by, then ascending or descending. Add multiple sorts — first sort by Priority (High first), then by Due Date (earliest first) for a triage-ready task list. Sorts stack: the second sort applies within groups created by the first sort.

Pro tip: Use filtered views as your daily workspace. Create a "My Tasks" view (filter: Assignee = Me AND Status != Done) and a "Due This Week" view (filter: Due Date is this week). These become your daily entry points instead of seeing the full unfiltered database every time you open it.

Linked Databases and Relation Properties

Relation properties are one of Notion's most powerful and most underused features. They let you connect two databases so rows in one database reference rows in another.

Classic example: A Projects database and a Tasks database. Add a Relation property to Tasks that links to Projects. Now every task can be assigned to a project. On the Project page, you can see all related tasks. This is a one-to-many relationship — one project, many tasks.

How to set up a Relation

  1. In your database, click "+ Add a property."
  2. Choose "Relation" as the property type.
  3. Select the database you want to link to.
  4. Choose whether to show the relation on both databases (recommended — adds a back-relation property to the linked database).
  5. Click "Create relation."

Now when you open any row, you can click the Relation property field and search for items in the linked database to connect them.

Common Relation setups for freelancers

  • Projects → Tasks: See all tasks for a project on the project page
  • Clients → Projects: See all projects per client
  • Blog Posts → Authors: Attribute each post to a writer
  • Resources → Projects: Link reference material to the project it supports
  • Invoices → Clients: Track invoicing history per client

Rollup Properties

A Rollup property calculates an aggregate value from the items linked via a Relation property. Think of it as a SUM or COUNT that reaches across databases.

Example: Your Projects database has a Relation to Tasks. Add a Rollup property to Projects that: points to the Tasks relation, pulls the "Status" property from each task, and calculates "Count all." Now each project row shows the total number of linked tasks. Change the calculation to "Count values where Status = Done" and you get a completion count.

Common Rollup calculations

  • Count all — total linked items (tasks per project, invoices per client)
  • Sum — total of a number property (total invoice value per client)
  • Average — average of a number property (average task hours per project)
  • Earliest date / Latest date — first or last date among linked items (first task start, last task due)
  • Percent checked — if the property is a checkbox, percentage of linked items that are checked (project completion %)
  • Show unique values — display all unique values from a text or select property in linked items (all tags used across linked posts)

Database Templates

Database templates pre-fill new rows with content, properties, and even sub-pages so you don't have to start from scratch every time. They are one of the biggest time-savers in Notion for repetitive workflows.

To create a template: click the dropdown arrow next to the blue "New" button in your database, then click "+ New template." Edit the template page exactly as you want new rows to look — set default property values, add a standard content structure with headers and checklists, and even add linked database views inside the template.

Template ideas for common databases

  • Project database template: Pre-fill Status = "Not Started", add a Project Brief section, a Goals section, a Tasks linked view, and a Meetings section.
  • Blog post template: Pre-fill Status = "Draft", add H2 outline sections for intro, body sections, CTA; add an SEO properties section for target keyword, meta description, and word count.
  • Weekly review template: Pre-fill the current week's date, add standard review sections (What did I complete? What's next? What's blocking me? Weekly highlight).
  • Client onboarding template: Pre-fill Status = "Onboarding", add a welcome email draft, a project kickoff checklist, and an intake form link.
  • Meeting notes template: Add sections for Attendees, Agenda, Action Items, and Decisions Made — pre-linked to the Project relation.

Frequently Asked Questions

What is the difference between a Notion database and a simple table?

A simple Notion table is a basic visual grid with no structured data — like a spreadsheet embedded in a doc. A Notion database is a structured collection of pages with typed properties (date, select, relation, formula, etc.), multiple views, filtering, sorting, and the ability to link to other databases. Every database row is also a full Notion page you can write in.

How do I create a database in Notion?

Type /database in any Notion page and choose "Database - Full page" or "Database - Inline." You can also type /table to create an inline table-view database. Add properties with the + button in the column header, choose property types, and start adding rows — each row is a full Notion page.

What are Notion database views?

Database views let you display the same data in different formats. The six views are: Table (grid), Board (kanban), Calendar (monthly), Gallery (image cards), List (simple list), and Timeline (Gantt bars). You can have unlimited views per database, each with its own filters and sorts.

What is a Notion Relation property?

A Relation property links one database to another. For example, a Projects database can relate to a Tasks database so you can see which tasks belong to which project. When you add a Relation, you can also add a Rollup property to calculate aggregates (count, sum, average) across the linked items.

How do I filter a Notion database?

Click the "Filter" button at the top of any database view. Choose a property, select a condition (equals, contains, is not empty, etc.), and enter a value. You can add multiple filters joined by AND or OR. Filters are saved per view, so different views can have different filters on the same underlying data.

What is a Notion formula property?

A Formula property calculates a value using data from other properties in the same row using Notion's formula syntax (similar to Excel). Common uses: calculate profit (Revenue minus Cost), format a full name from first and last name fields, compute days until a deadline, or create a conditional status label based on other fields.
