Notion Formulas: The Complete Guide (With Real-World Examples)

Master Notion formulas — syntax, 15 practical examples covering dates, progress bars, invoice totals, time tracking, and conditional logic. No coding experience required.

June 2026  ·  15 min read
Home › Blog › Notion Formulas Guide

Notion formulas are the feature that separates a basic Notion setup from a genuinely powerful one. Without formulas, your databases show you what you entered. With formulas, your databases calculate, infer, and surface things you didn't have to enter manually — deadline countdowns, invoice totals, completion percentages, overdue flags, revenue-per-hour rates, and more.

The good news: Notion formulas are far simpler than they look. You don't need to know programming. If you've ever written an Excel formula, Notion formulas will feel familiar. If you haven't, the 15 examples in this guide give you everything you need to copy, adapt, and build your own.

What Are Notion Formulas?

A Notion formula is a calculation you write inside a Formula property in any Notion database. When you add a Formula property, it shows one calculated value per row — computed automatically from other properties in that row. Change a property that a formula references, and the formula result updates instantly.

Formulas can:

  • Perform arithmetic: add, subtract, multiply, divide number properties
  • Calculate dates: days between dates, days until a deadline, week numbers
  • Work with text: join strings, extract substrings, change case
  • Apply logic: return different values based on conditions (if/else)
  • Reference rollup results: use aggregated data from related databases
  • Create visual indicators: emoji progress bars, colored status badges via text

The key constraint: formulas can only reference properties in the same database row. To reference data from another database, you need a Relation + Rollup combination to pull that data into the current row first, then reference it in a formula.

Notion Formula Syntax Basics

Every Notion formula is an expression that returns a single value. The three things you need to understand:

1. Referencing Properties

Use prop("Property Name") to reference another property in the same row. The property name must exactly match (case-sensitive, spaces included). Example: prop("Due Date") returns the value of the Due Date property.

2. Functions

Functions take inputs and return outputs. Notion has 70+ built-in functions. The most important ones:

if(condition, true, false)
Returns the first value if condition is true, second if false. The most used function.
now()
Returns the current date and time. Recalculates on page load.
dateBetween(a, b, unit)
Returns the difference between two dates in "days", "months", "years", "hours", etc.
dateAdd(date, n, unit)
Adds n units to a date. dateAdd(now(), 7, "days") = one week from today.
round(n) / floor(n) / ceil(n)
Round a number to nearest integer, round down, or round up.
empty(value)
Returns true if the value is empty/null. Use to guard against empty property references.
and(a, b) / or(a, b)
Combine multiple conditions. and() requires both true; or() requires either true.
concat(a, b, ...)
Joins text strings together. concat("Hello", " ", "World") = "Hello World".

3. Operators

Math: + - * /   Comparison: == != > < >= <=   Logic: and or not

Formula Editor Tip

Notion's formula editor has auto-complete. As you type prop(" it will suggest your property names. As you type a function name it will suggest the correct function. Use this to avoid typos in property names — the number one cause of formula errors.

15 Practical Notion Formulas (Copy & Use)

1. Days Until Due Date
Dates
dateBetween(prop("Due Date"), now(), "days")
Returns the number of days between today and the Due Date. Positive = days remaining. Negative = overdue. Use in project trackers, client databases, invoice trackers.
2. Overdue Flag with Days Remaining
Dates + Conditional
if(empty(prop("Due Date")), "No date", if(dateBetween(prop("Due Date"), now(), "days") < 0, "⚠️ " + format(abs(dateBetween(prop("Due Date"), now(), "days"))) + " days overdue", format(dateBetween(prop("Due Date"), now(), "days")) + " days left"))
Returns "⚠️ 3 days overdue" for past-due items, "5 days left" for upcoming ones, and "No date" for undated items. Guards against empty Due Date to prevent errors.
3. Completion Percentage
Progress
if(prop("Total Tasks") > 0, round(prop("Done Tasks") / prop("Total Tasks") * 100), 0)
Calculates percent complete from Done Tasks and Total Tasks rollup properties. Requires two Rollup properties: one counting completed tasks, one counting all tasks for that project via a Relation.
4. Visual Progress Bar
Visual
slice("██████████", 0, floor(prop("Progress") / 10)) + slice("░░░░░░░░░░", 0, 10 - floor(prop("Progress") / 10)) + " " + format(prop("Progress")) + "%"
Creates a 10-block emoji progress bar. Requires a Number property called "Progress" (0–100). Output looks like: ████░░░░░░ 40%. Use in project databases, habit trackers, or goal trackers.
5. Invoice Total with Tax
Finance
prop("Rate") * prop("Hours") * (1 + prop("Tax Rate") / 100)
Calculates invoice total from Rate (Number), Hours (Number), and Tax Rate (Number, e.g. 20 for 20%). Requires Rate, Hours, and Tax Rate Number properties. Add formatting by wrapping in format() if you want a $ prefix.
6. Revenue Per Hour
Finance
if(prop("Hours Logged") > 0, round(prop("Project Revenue") / prop("Hours Logged")), 0)
Calculates your effective hourly rate for a project. Useful for identifying which project types are most profitable. Requires Project Revenue (Number) and Hours Logged (Number) properties.
7. Days Since Last Contact (CRM)
CRM / Dates
if(empty(prop("Last Contact")), "Never contacted", if(dateBetween(now(), prop("Last Contact"), "days") > 30, "🔴 " + format(dateBetween(now(), prop("Last Contact"), "days")) + " days", if(dateBetween(now(), prop("Last Contact"), "days") > 14, "🟡 " + format(dateBetween(now(), prop("Last Contact"), "days")) + " days", "🟢 " + format(dateBetween(now(), prop("Last Contact"), "days")) + " days")))
CRM health indicator. Returns green (recent contact), yellow (14–30 days), or red (30+ days) with day count. Flags relationships that need attention before they go cold.
8. Automatic Priority Score
Prioritization
if(prop("Priority") == "P1", 100, if(prop("Priority") == "P2", 50, 10)) + if(dateBetween(prop("Due Date"), now(), "days") < 3, 50, if(dateBetween(prop("Due Date"), now(), "days") < 7, 25, 0))
Calculates a numeric priority score combining Priority level and deadline urgency. P1 with a 2-day deadline scores 150; P3 with no near deadline scores 10. Sort by this formula to auto-prioritize your task list.
9. Time Tracking: Duration in Hours
Time Tracking
dateBetween(prop("End Time"), prop("Start Time"), "minutes") / 60
Calculates session duration in hours from Start Time and End Time (both Date properties with time enabled). Round with round(... * 100) / 100 for 2 decimal places.
10. Week Number of Year
Dates
ceil(dateBetween(prop("Date"), dateSubtract(prop("Date"), date(formatDate(prop("Date"), "D")) - 1, "days"), "days") / 7)
Returns the ISO week number (1–52) for any date property. Useful for grouping entries by week in content calendars, time trackers, and weekly planning databases.
11. Conditional Status Badge
Visual + Conditional
if(prop("Status") == "Complete", "✅ Complete", if(prop("Status") == "In Progress", "🔄 In Progress", if(prop("Status") == "Blocked", "🚫 Blocked", "📋 Not Started")))
Adds emoji badges to status values for visual clarity in gallery and board views. Customize the emoji and text to match your Status property options exactly.
12. Late Payment Flag (Invoices)
Finance + Dates
if(and(prop("Payment Status") != "Paid", dateBetween(now(), prop("Due Date"), "days") > 0), "🔴 LATE — " + format(dateBetween(now(), prop("Due Date"), "days")) + " days", if(prop("Payment Status") == "Paid", "✅ Paid", "⏳ Pending"))
Invoice status tracker that automatically flags overdue unpaid invoices. Requires Payment Status (Select) and Due Date (Date) properties. Shows "LATE — 5 days" for overdue, "Paid" for settled, "Pending" for upcoming invoices.
13. Content Word Count Estimate
Content / Calculation
if(prop("Word Count") < 500, "Short (<500w)", if(prop("Word Count") < 1500, "Medium (500–1500w)", if(prop("Word Count") < 3000, "Long (1500–3000w)", "Deep Dive (3000w+)")))
Categorizes content by length for content calendars. Requires a Word Count (Number) property. Shows a human-readable label instead of a raw number for quick scanning.
14. Reading Time Estimate
Content
format(ceil(prop("Word Count") / 200)) + " min read"
Calculates reading time at 200 words per minute (average adult reading speed). Returns "7 min read" format. Use in content databases, book trackers, and article research databases.
15. Habit Streak Counter
Habit Tracking
if(prop("Completed Today") == true, "🔥 " + format(prop("Current Streak") + 1), if(dateBetween(now(), prop("Last Completed"), "days") > 1, "💔 Streak lost — " + format(prop("Best Streak")), "🎯 Keep going!"))
Habit tracker feedback formula. Requires Completed Today (Checkbox), Current Streak (Number rollup or manual), Last Completed (Date), and Best Streak (Number) properties. Shows motivational feedback based on current state.

Common Notion Formula Errors and Fixes

Error: Property not found / "red underline on prop()"
The property name in your formula doesn't exactly match the property name in the database. Formulas are case-sensitive and space-sensitive.
Fix: Check the exact property name. "due date" ≠ "Due Date". Copy-paste from the property editor to be safe.
Error: Type mismatch — can't use math on Select property
You're trying to do arithmetic on a Select or Text property. Select properties return text, not numbers.
Fix: Use a Number property for any value you want to calculate with. Convert the Select to a Number property, or use if() to map Select values to numbers.
Error: Formula returns blank for some rows
Referenced properties are empty for those rows. A formula referencing an empty Date property returns null, which propagates through calculations.
Fix: Wrap property references in empty() guards: if(empty(prop("Due Date")), "No date set", dateBetween(...))
Error: if() branches return mismatched types
Both branches of an if() must return the same type (both text, or both number, or both date). Returning a number in the true branch and text in the false branch causes an error.
Fix: Use format() to convert numbers to text, or ensure both branches return the same type. if(x > 5, "High", "Low") works. if(x > 5, 10, "Low") does not.
Error: dateBetween returns wrong sign
dateBetween(a, b, unit) returns a - b. If you want days until the due date (positive when future), the order matters: dateBetween(future, now()) gives positive. dateBetween(now(), future) gives negative.
Fix: dateBetween(prop("Due Date"), now(), "days") gives positive for future dates, negative for past dates. This is usually what you want for countdowns.

Advanced Formula Tips

Nest if() for multi-branch logic: Notion's if() is binary (true/false), but you can nest multiple if() calls to create multi-branch logic: if(a, result1, if(b, result2, if(c, result3, default))).

Combine formulas with rollups: Rollup properties aggregate data from related databases. You can reference rollup results in formulas — for example, calculate profitability as a formula using a rollup that sums all invoices related to a client.

Use format() to convert numbers to text: Many formulas need to combine a number with text ("5 days left"). You must convert the number first: format(5) + " days left" works. 5 + " days left" throws a type error.

Use abs() for absolute values: When calculating overdue days, dateBetween returns a negative number for past dates. Wrap in abs() to get the positive value: abs(dateBetween(prop("Due Date"), now(), "days")).

Frequently Asked Questions

What are Notion formulas?
Notion formulas are calculations you can write inside a Formula property in any Notion database. They can reference other properties using prop('Property Name'), perform math operations, manipulate dates and text, and return conditional outputs. Formulas run automatically whenever the referenced properties change.
How do I write a Notion formula?
Add a Formula property to any Notion database by clicking the + button in the database header and selecting Formula. Click the formula property to open the formula editor. Type your formula — reference other properties with prop('Property Name'), use functions like if(), dateBetween(), now(), and operators like +, -, *, /. Notion auto-suggests property names and function names as you type.
How do I calculate days until a due date in Notion?
Use the dateBetween() function: dateBetween(prop('Due Date'), now(), 'days'). This returns the number of days between today and the Due Date. A positive number means days remaining; negative means overdue. Wrap it in an if() to add an overdue flag.
Can Notion formulas reference other properties?
Yes. Use prop('Property Name') to reference any other property in the same database row. The property name must exactly match (including capitalization and spaces). You can reference Text, Number, Select, Date, Checkbox, and Rollup properties in formulas.
What is the Notion if() formula?
The if() formula returns different values based on a condition: if(condition, value_if_true, value_if_false). Example: if(prop('Status') == 'Complete', '✅ Done', '⏳ In Progress'). Combine multiple conditions with and() and or().
Why is my Notion formula showing an error?
The most common errors: (1) Property name typo — prop() is case-sensitive. (2) Wrong property type — you can't do math on a Select property. (3) Null/empty values — wrap in empty() guards. (4) Mismatched output types — both branches of if() must return the same type.
Can I create progress bars with Notion formulas?
Yes. Use slice() to build an emoji progress bar: slice('██████████', 0, floor(prop('Progress') / 10)) + slice('░░░░░░░░░░', 0, 10 - floor(prop('Progress') / 10)). Set your Progress property to a Number between 0 and 100.
What is the difference between Notion formulas and rollups?
Notion formulas calculate values from properties within the same database row. Rollups aggregate data across a Relation from a different database — like summing all invoice amounts related to a client. Formulas are for row-level calculations; rollups are for cross-database aggregation. You can use a rollup result inside a formula.
Are Notion formulas case sensitive?
Property names in prop() are case-sensitive. prop('Status') is different from prop('status'). Function names like if(), dateBetween(), and now() are not case-sensitive. String comparisons (== 'Complete') are case-sensitive — use lower() to normalize case when comparing text values.
