1. What Are Notion Formulas?

A Notion formula is a special property type that calculates a value based on other properties in the same database record. Instead of typing a value manually, you write an expression and Notion computes the result dynamically whenever the underlying data changes.

Think of it like a spreadsheet formula, but inside a Notion database. You reference properties using prop("Property Name"), apply functions like dateBetween() or if(), and the formula property displays the calculated output automatically.

Formulas are powerful for:

  • Deadline countdowns — how many days until this task is due?
  • Financial calculations — profit margins, hourly rates, budget remaining
  • Status automation — automatically flag overdue items with emoji or text
  • Progress tracking — percentage complete, days in pipeline stage
  • String formatting — generating invoice numbers, display labels

Key limit: Notion formulas can only reference properties in the same database record. To pull data from another database, you need a Relation + Rollup first, then reference that rollup in your formula. This three-step chain is covered in the FAQ below.

2. Notion Formula Syntax Basics

Before the examples, it helps to understand the building blocks. Every Notion formula is built from property references, functions, operators, and conditionals.

Property References

To use the value of any property in the same record, wrap the property name in quotes inside prop():

prop("Due Date") prop("Invoice Amount") prop("Status")

Property names are case-sensitive and must match exactly, including spaces and capitalization.

Data Types

Notion formulas are typed. Each value has a type: Number, Text (string), Boolean (true/false), or Date. Functions expect specific types, and mixing types causes errors. Use conversion functions when needed:

  • toText(value) — converts Number or Boolean to Text
  • toNumber(value) — converts Text to Number (if possible)
  • formatDate(date, "format") — converts Date to Text string

Operators

  • Arithmetic: +, -, *, /, % (modulo), ^ (exponent)
  • Comparison: ==, !=, >, <, >=, <=
  • Logic: && (AND), || (OR), ! (NOT)
  • String concatenation: + when used with text values

The if() Conditional

The if() function is your most-used formula tool:

if(condition, valueIfTrue, valueIfFalse)

You can nest if() statements for multiple conditions. Read on for examples showing how to do this cleanly without losing track of parentheses.

Key Date Functions

  • now() — current date and time
  • date(year, month, day) — create a specific date
  • dateBetween(date1, date2, "unit") — difference between dates in "days", "hours", "months", or "years"
  • dateAdd(date, number, "unit") — add time to a date
  • dateSubtract(date, number, "unit") — subtract time from a date
  • formatDate(date, "format") — format a date as text using Moment.js format strings

Tip: Notion's formula editor has autocomplete. Start typing a function name and press Tab to complete it. It also shows expected argument types inline — use this while learning to avoid type errors.

3. 5 Beginner Notion Formulas

These formulas are safe for anyone to try on day one. They use simple arithmetic, basic date functions, and single-level conditionals with no tricky nesting.

Formula 1: Days Until Due

dateBetween(prop("Due Date"), now(), "days")

Shows how many days remain until the due date. Returns a negative number if overdue. Great for task databases and project trackers. Requires a Date property named "Due Date".

Formula 2: Days Since Created

dateBetween(now(), prop("Created"), "days")

Calculates the age of a record in days. Useful for tracking how long a lead has been in your CRM, how old a support ticket is, or how long a task has been open. "Created" should be a Created Time property.

Formula 3: Reading Time Estimate

round(prop("Word Count") / 200) + " min read"

Calculates estimated reading time at 200 words per minute. Requires a Number property named "Word Count". The round() function removes decimals and the string addition appends the unit label.

Formula 4: Overdue Flag

if(dateBetween(prop("Due Date"), now(), "days") < 0, "OVERDUE", "On Track")

Returns a warning text if the due date has passed, a positive indicator if not. Add emoji to taste. Works best when filtered in a view to show only OVERDUE items so you can triage them quickly.

Formula 5: Task Priority Score

if(prop("Priority") == "High", 3, if(prop("Priority") == "Medium", 2, 1))

Converts a text Priority select (High/Medium/Low) into a numeric score (3/2/1). Use this score to sort your database numerically by priority — more reliable than sorting text values alphabetically.

Quick win: Add a Days Until Due formula to your task database right now and sort by it ascending. You will immediately see your most urgent work at the top without manually re-ordering anything.

4. 8 Intermediate Notion Formulas

These formulas involve nested conditionals, multi-property calculations, and more practical business applications. They assume you are comfortable with the basics above.

Formula 6: Days Until Renewal

dateBetween(prop("Renewal Date"), now(), "days")

For subscription and contract trackers, this shows how many days until each subscription renews. Combine with a conditional to flag anything renewing in the next 14 days. Great for a tools and subscription audit database.

Formula 7: Revenue Per Hour (Effective Rate)

if(prop("Hours Logged") == 0, 0, round(prop("Invoice Amount") / prop("Hours Logged")))

Calculates your effective hourly rate for any project. This is the formula freelancers actually need — it reveals which clients are actually eating your time relative to what they pay. The null guard prevents division by zero on new records.

Formula 8: Budget Remaining

prop("Budget") - prop("Spent")

Shows remaining budget for a project. Simple subtraction, but pair it with a conditional for color-coded warnings. Add a status label version: if(prop("Budget") - prop("Spent") is less than 0, "Over Budget", "On Budget").

Formula 9: Status Emoji Badge

if(prop("Status") == "Done", "Done", if(prop("Status") == "In Progress", "Active", if(prop("Status") == "Blocked", "Blocked", "Not Started")))

Maps a Status select property to a text badge. You can add emoji to any of the output strings. Useful for gallery views and embedded Notion pages where you want quick visual scanning without looking at select colors.

Formula 10: Age Calculation

dateBetween(now(), prop("Birthdate"), "years")

Calculates a person's current age in years. Perfect for team directories, client contact databases, or any record with a birthdate. The order matters: now() first, then the birthdate, gives a positive number.

Formula 11: Invoice Number Generator

"INV-" + toText(prop("Invoice #"))

Formats a numeric invoice counter into a proper invoice number string. The prefix is a text literal, toText() converts the number for concatenation. Result: INV-1, INV-2, etc. Change the prefix to match your format (e.g., "2026-" for year-prefixed numbering).

Formula 12: Weighted Priority Score

(if(prop("Impact") == "High", 3, if(prop("Impact") == "Medium", 2, 1))) * (if(prop("Effort") == "Low", 3, if(prop("Effort") == "Medium", 2, 1)))

A classic prioritization matrix. High Impact multiplied by Low Effort = 9 (do first). Low Impact multiplied by High Effort = 1 (do last or skip). Sort your task list by this score descending for an instant priority stack without manual re-ordering.

Formula 13: Days in Current Pipeline Stage

dateBetween(now(), prop("Stage Started"), "days")

Tracks pipeline velocity — how long a deal or project has been in its current stage. Requires a "Stage Started" Date property that you update when you move a record to a new stage. Use in sales CRMs and project pipelines to spot stalled work.

Power move: Use Formula 13 with a conditional to flag stale deals automatically. Anything that has been in the same stage for more than 14 days gets flagged. Filter your CRM view to show only stale deals every Monday morning for your weekly review.

5. 8 Advanced Notion Formulas (Plus 4 Bonus)

These formulas require multiple nested conditions, multi-step calculations, or creative use of Notion's function library. They are battle-tested in real freelance and business workflows.

Formula 14: Freelance Day Rate

round(prop("Monthly Revenue") / 22)

Calculates your effective day rate from monthly revenue. 22 is the standard working days per month (260 days per year divided by 12 months). Use this in a revenue dashboard to track whether your day rate is growing month over month.

Formula 15: Percentage to Goal

if(prop("Goal") == 0, "No goal set", round(prop("Current") / prop("Goal") * 100) + "%")

Progress tracker showing how far you are toward a numeric goal as a percentage string. Works for revenue goals, word count targets, lead counts, or any numeric target. The null guard handles records with no goal set.

Formula 16: Client Tier Classification

if(prop("Annual Revenue") > 10000, "Platinum", if(prop("Annual Revenue") > 5000, "Gold", if(prop("Annual Revenue") > 1000, "Silver", "Bronze")))

Automatically classifies clients by lifetime or annual revenue into four tiers. This powers a filtered CRM view showing only your Platinum clients for quarterly check-ins. Combine with a Rollup of invoices to calculate Annual Revenue automatically.

Formula 17: Next Quarter Date

formatDate(dateAdd(now(), 3, "months"), "MMM D, YYYY")

Returns a formatted text date exactly 3 months from today. Use in quarterly planning databases to auto-calculate review dates, contract renewal check-ins, or goal reset milestones. The formatDate wrapper makes it readable rather than a raw timestamp.

Formula 18: Urgency Score (Deadline + Priority Combined)

(if(dateBetween(prop("Due Date"), now(), "days") < 3, 10, if(dateBetween(prop("Due Date"), now(), "days") < 7, 5, 0))) + (if(prop("Priority") == "High", 5, if(prop("Priority") == "Medium", 2, 0)))

Combines deadline urgency with priority level into a single score. Due in fewer than 3 days adds 10 points. Due in fewer than 7 days adds 5 points. High priority adds 5, Medium adds 2. Sort descending by this score to get your actual daily work order.

Formula 19: Time Tracking Hours with Decimal

round(dateBetween(prop("End Time"), prop("Start Time"), "minutes") / 60 * 10) / 10

Calculates hours worked from Start Time and End Time date properties with one decimal place of precision (e.g., 2.5 hours). Set both as Date properties with time enabled. Use in a time log database to track billable hours automatically.

Formula 20: ISO Week Number

toNumber(formatDate(prop("Date"), "w"))

Returns the ISO week number (1 through 53) for any date. Use in weekly review databases to auto-group entries by week number. The "w" format string returns the week number as text; toNumber() converts it for numeric sorting.

Formula 21: Monthly Contract Value

if(prop("Contract Duration Months") == 0, 0, round(prop("Total Contract Value") / prop("Contract Duration Months")))

For retainer and contract trackers, this breaks a lump sum into monthly value. Useful for understanding your recurring revenue pipeline and cash flow forecasting. The null guard prevents division by zero on contracts with no duration set.

4 Bonus Formulas

Bonus 1: Completion Rate

if(prop("Total Tasks") == 0, "No tasks", round(prop("Completed Tasks") / prop("Total Tasks") * 100) + "% complete")

Shows project completion as a percentage string. Requires Rollup properties counting completed vs. total related task records. The null guard handles projects with no tasks assigned yet. Great for a project portfolio view showing completion at a glance.

Bonus 2: Net 30 Payment Due Date

formatDate(dateAdd(prop("Invoice Date"), 30, "days"), "MMM D, YYYY")

Auto-calculates the Net 30 payment due date from invoice date. Change 30 to 15 or 60 for different payment terms. Combine with the overdue formula to flag late invoices automatically in your invoice tracker.

Bonus 3: Meeting Duration in Minutes

dateBetween(prop("End Time"), prop("Start Time"), "minutes") + " min"

Shows meeting duration as a labeled text string. In a meeting log database, use this to track where your calendar time is going per client or per project. Combine with a Rollup to see total meeting minutes per week.

Bonus 4: Profit Margin Percentage

if(prop("Revenue") == 0, "No revenue", round((prop("Revenue") - prop("Expenses")) / prop("Revenue") * 100) + "%")

Calculates gross profit margin. Essential for any business tracking revenue vs. expenses per project or per month. A healthy freelance project margin is typically 70 to 85 percent, meaning you keep most of your revenue as profit after direct project costs.

7. Common Notion Formula Errors and How to Fix Them

Notion's formula editor is helpful but its error messages can be cryptic. Here are the most common issues and their exact fixes.

Type Mismatch Error

Symptom: "Expected number, got text" or a similar type error on save.

Cause: You are passing a text value to a function that expects a number, or concatenating a number with a string without conversion.

Fix: Use toNumber() or toText() to convert. For concatenation, always convert non-text values: "Invoice #" + toText(prop("Number")).

Empty or Null Property Errors

Symptom: Formula returns empty or errors on some records but works fine on others.

Cause: The formula tries to calculate with a property that is empty on some records.

Fix: Wrap with an empty check before performing the calculation: if(empty(prop("Due Date")), "No date set", dateBetween(prop("Due Date"), now(), "days")).

Division by Zero

Symptom: Formula shows an error or infinity symbol on certain records.

Cause: The denominator property is 0 or empty on those records.

Fix: Always guard division operations: if(prop("Hours") == 0, 0, prop("Revenue") / prop("Hours")).

String Comparison Not Triggering

Symptom: if(prop("Status") == "done") never triggers even when the Status property clearly says "Done".

Cause: Case mismatch. Notion formulas are case-sensitive. "Done" does not equal "done".

Fix: Normalize case before comparing: if(lower(prop("Status")) == "done", ...). This approach always works regardless of how users enter the value.

dateBetween Returns Negative When You Expected Positive

Symptom: dateBetween() returns a negative number when both dates exist and you expected a positive result.

Cause: Argument order. dateBetween(A, B, "days") returns A minus B. If A is earlier than B, the result is negative.

Fix: Reverse the argument order. If you want days since a past date, put now() as the first argument. If you want days until a future date, put the future date first. Wrap with abs() if you only need the magnitude and direction does not matter.

Watch out: Notion's formula editor validates syntax on save but does not always catch runtime errors like division by zero or null reference. Always test your formulas on records with empty properties before rolling them out to a full database.

Frequently Asked Questions About Notion Formulas

Can Notion formulas reference other pages?
No. Notion formulas can only reference properties within the same database record. To pull data from other databases, you need a three-step chain: create a Relation property linking the two databases, create a Rollup that pulls in specific values from the related database, then reference that rollup in your formula using prop("Rollup Name").
Can you use formulas in filters?
Yes. Formula properties appear as filter options in database views. You can filter by whether a formula output equals a specific value, is empty, contains certain text, or meets a numeric condition. This is how you create views like "Overdue tasks only" or "Clients with fewer than 30 days to renewal" — create the formula first, then filter on it.
How do I reference a rollup in a formula?
Use prop("Rollup Property Name") exactly like any other property reference. The critical thing is to ensure the rollup's aggregation type matches what your formula expects. If your formula needs a number, configure the rollup to output Sum, Average, Count, or another numeric aggregation. If the rollup outputs text (like "Show original"), formulas expecting numbers will error.
What is the difference between a formula and a rollup?
A rollup aggregates data from a related database — it sums, counts, averages, or shows values across related records from another database. A formula performs calculations on properties within the same record only. They are complementary: use a rollup to pull aggregate data from related records, then use a formula to transform or combine that rolled-up value with other properties in the same record.
Can formulas pull data from other databases?
Not directly — formulas are scoped to a single record. To access data from another database, you need the three-step chain: Relation property linking the databases, a Rollup on that relation to pull in specific values, and then a Formula referencing the rollup result. It is more steps than most people expect at first, but it unlocks very powerful cross-database calculations once you understand the pattern.
Are Notion formulas case-sensitive?
Yes — string comparisons in Notion formulas are case-sensitive by default. "Done" does not equal "done" or "DONE". The safest approach is to always normalize case before comparing: use lower(prop("Status")) == "done" rather than prop("Status") == "Done". This way your formula works correctly regardless of how team members capitalize status values.
