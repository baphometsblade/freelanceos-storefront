Notion Table View: The Complete Guide to Notion's Database Table Layout

By FreelanceOS · July 2026 · 8 min read

What Is Notion Table View?

Notion table view is the default database layout in Notion — a spreadsheet-style grid where each row is a database entry (a Notion page) and each column is a property (a field). When you create a new database in Notion, table view is what you see first.

But table view in Notion is more than a spreadsheet. Each row is a full Notion page — open any row and you can add text, images, embedded files, subpages, comments, and any other Notion block. The table is the structured overview; the page is the full document underneath. This combination is what makes Notion databases more powerful than traditional spreadsheets for knowledge management.

Table view vs. spreadsheet: A spreadsheet cell holds one value. A Notion table row holds one value per column AND a full rich-text document when opened. A spreadsheet is optimized for calculation. A Notion table is optimized for information management — tracking, organizing, and documenting structured work.

How to Create a Notion Table Database

Method 1 — Inline Table (Embedded in a Page)

Type /table on any Notion page and select "Table" from the menu. A small table with one default column (Name) appears inline in your page. This creates a database that lives inside your current page.

Method 2 — Full-Page Database

Click the + button in the left sidebar to create a new page, then select "Table" from the options. This creates a standalone database page — better for databases you'll access frequently and link to from multiple pages.

Method 3 — Switch an Existing View to Table

On any existing Notion database showing a different layout (board, gallery, etc.), click the view name at top left → + Add a view → select Table. A table view is added alongside the other views, all showing the same underlying data.

Filtering, Sorting, and Grouping in Table View

Filtering

Click Filter in the top bar → + Add filter. Choose a property, then set the condition: is, is not, contains, is empty, is before (for dates), etc. Multiple filters can be stacked with AND (all conditions must match) or OR (any condition matches) logic.

Filters only apply to the current view — they do not change the underlying database or affect other views. Use filters to create purpose-specific views: "Active Projects" (Status = In Progress), "This Week's Deadlines" (Due Date = this week), "Unpaid Invoices" (Payment Status = Unpaid).

Sorting

Click Sort → + Add sort. Choose a property and direction (ascending or descending). Multiple sorts can be stacked — Notion applies them in order. Common sorts: alphabetical by Name, chronological by Date, by Status, or by Priority number.

Grouping

Click the ... icon next to the view name → Group → choose a property. Rows are organized into collapsible sections by their value in the chosen property. Useful for table views of large databases: group a 200-row task database by Project to see tasks organized by project, group a 50-row client database by Status to see Active vs. Past vs. Prospective clients in distinct sections.

Important: Group is different from board view. Grouping adds expandable sections to a table layout. Board view automatically groups by a Status or Select property in a horizontal card-column layout. Both show grouped data, but table view + grouping is better for data-dense databases; board view is better for workflow visualization.

Managing Table Columns: Hide, Reorder, Resize

  • Hide properties: Click the Properties button (top bar) → toggle any property off. Hidden properties still exist in the database — they're just not visible in this view. Other views or linked views can still show them. Use hiding aggressively to keep your table focused.
  • Reorder columns: Drag any column header left or right to reorder. The Title property is always first — you can't move it, but you can resize it.
  • Resize columns: Hover over the right edge of any column header until the resize cursor appears, then drag. Wider columns for text-heavy properties, narrower for checkboxes and numbers.
  • Wrap text: Click the … icon next to a view name → Turn on/off text wrapping. With wrapping off, long text is truncated at the cell boundary (cleaner). With wrapping on, tall rows show more text (better for notes).
  • Calculate column totals: Click the bottom of any Number property column to add a calculation — Count, Sum, Average, Min, Max. Useful for tracking total invoice amounts or average project values in a financial database.

Table View vs. Other Notion Database Views

Notion databases can display the same data in six different layouts. Use the right one for the task:

  • Table — best for structured data browsing, bulk editing, and seeing many properties across many entries at once. The spreadsheet of Notion.
  • Board (Kanban) — best for workflow tracking. Groups entries by status columns. Visual, drag-to-update. Best for project tasks and pipelines.
  • Gallery — best for visual content browsing. Shows cover images prominently. Best for portfolios, product catalogs, inspiration boards.
  • Calendar — best for date-based planning. Shows entries on a monthly or weekly calendar by their date property. Best for content scheduling.
  • List — most minimal layout. Title only, with properties visible on hover. Best for simple to-do lists or reference databases where you just need the name.
  • Timeline (Gantt) — best for project planning with date ranges. Shows entries as horizontal bars across a time axis. Best for project schedules with multiple phases.

Frequently Asked Questions

What is Notion table view?
Notion table view is the default database layout in Notion — a spreadsheet-style grid where each row is a database entry (a full Notion page) and each column is a property (a structured field). Unlike a traditional spreadsheet, each row can hold rich content when opened (text, images, files, checklists), and you can switch the same database to board, gallery, calendar, list, or timeline view without changing the underlying data.
How is Notion table view different from a spreadsheet?
Notion table view looks like a spreadsheet but works like a database. Each row is a full Notion page — you can add paragraphs, images, files, and subpages when you open it. Property types are more structured than spreadsheet cells (Select, Multi-select, Date, Relation, Rollup, Formula). You can switch to six different layouts showing the same data. The main limitation: Notion formulas are less powerful than Excel or Sheets for complex calculations; for financial modeling or heavy math, a spreadsheet is better.
How do I add a column to a Notion table?
Click the + icon at the right end of your column headers. A menu appears listing all property types — Text, Number, Select, Multi-select, Date, Person, Files, Checkbox, URL, Email, Phone, Formula, Relation, and Rollup. Click the type you want, name the column, and it's added to every row in the database.
Can I filter a Notion table view?
Yes. Click Filter in the top bar of your table → + Add filter. Choose a property, set the condition, and enter the value. The table updates immediately to show only matching rows. Filters apply only to the current view — the underlying database is unchanged. You can stack multiple filters using AND (all conditions match) or OR (any condition matches) logic for complex filtering needs.
How do I group rows in a Notion table?
Click the ... icon next to your view name at the top left, then select "Group." Choose a Select, Multi-select, or Person property to group by. Rows organize into collapsible sections by their value in that property. Grouping transforms a flat table into a sectioned view without changing any data or creating separate databases.
