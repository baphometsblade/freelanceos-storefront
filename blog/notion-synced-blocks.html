Notion Synced Blocks: How to Sync Content Across Multiple Notion Pages

Updated July 2026 · By FreelanceOS Team
Home › Blog › Notion Synced Blocks

You have been there: you copy your team norms to five different pages. Three weeks later, the norms change, and you remember — with a sinking feeling — that you now need to track down and update all five copies. Notion Synced Blocks solve this problem entirely.

A synced block is a piece of content that lives in multiple places simultaneously. Edit it on any page, and every other instance updates the moment you stop typing. This guide covers everything: how to create synced blocks, how to copy and link them, how to edit and remove them, and the five best real-world use cases.

What Are Notion Synced Blocks?

A synced block is a block of content — text, lists, images, embeds, or any combination — that appears on multiple Notion pages at once. Under the hood, there is one version of the block and multiple references to it. When you change the content anywhere, every reference immediately shows the update.

Think of it as a live component, not a static copy. This is fundamentally different from duplicating content, where each copy is independent and diverges the moment either is changed.

Why this matters for freelancers and creators:

  • Global navigation — a sidebar or header that appears on every project page, always current
  • Recurring side panels — "Current focus" or "This week's goals" synced to all dashboards
  • Announcement banners — team-wide announcements that update everywhere from one edit
  • Team norms or policies — write them once, embed them in every relevant page, update once when they change
  • Standard footnotes — disclaimer text or legal language that needs to be identical across all pages
  • Shared templates — a reusable block structure that all team members reference
Key Concept

Synced blocks appear with a red outline in edit mode to remind you that changes will propagate everywhere. This is a visual signal — not a warning, just a helpful indicator that you are working with synced content.

How to Create a Synced Block

Step 1
Open the slash command menu

On any Notion page, type /synced block and press Enter (or click "Create a synced block" from the dropdown). A red-outlined block container appears on your page.

Step 2
Add content inside the synced block

Click inside the red-outlined area and add any content you want: paragraphs, bullet lists, images, callout blocks, even other block types. Everything you add inside the boundary is part of the synced block.

Step 3
Place it on other pages using "Copy and Sync"

Hover over the synced block and click the three-dot menu that appears, then select "Copy and sync." Navigate to any other page and paste (Ctrl+V / Cmd+V). The pasted block is now a live copy, fully synced with the original.

Pro Tip

You can paste synced blocks into subpages, linked databases, or any page in your workspace. There is no limit to how many pages can contain the same synced block.

How to Use "Copy and Sync"

The "Copy and Sync" option is the core workflow for spreading a synced block across your workspace. Here is what happens step by step:

  1. Hover over the synced block until the block controls appear
  2. Click the three-dot menu (⋯)
  3. Select Copy and sync from the dropdown
  4. Navigate to the destination page
  5. Paste with Ctrl+V (Windows) or Cmd+V (Mac)

The newly pasted block is immediately live. Both the original page and the destination page now show the same content, and edits on either page propagate to the other instantly.

You can repeat this as many times as needed. Paste the same synced block across 20 pages, and a single edit updates all 20.

Editing Synced Blocks

Editing a synced block is exactly like editing any other Notion block. Click inside the content, make your changes, and Notion saves automatically. The change appears on every other page containing that synced block within seconds.

Important behaviors to know:

  • Edit from any instance — there is no designated "source" page. You can edit the synced block from any page it lives on, and the change propagates everywhere.
  • Last save wins — if two people happen to edit the same synced block at nearly the same moment, Notion resolves the conflict by keeping the most recent save.
  • Block types inside are fully editable — you can change text, add or remove bullet points, add images, swap callout icons. The sync applies to everything inside the block boundary.
  • Cannot move a synced block into a database property — synced blocks live on pages, not inside property fields.
Editing Tip

The red outline around a synced block is only visible to editors in edit mode. Viewers in read-only mode see the content without any visual indicator that it is synced.

Removing a Page's Instance Without Breaking Others

You may want to remove a synced block from one page without affecting all the other pages that contain it. Here is how:

  1. Hover over the synced block on the page where you want to remove it
  2. Click the three-dot menu
  3. Select Unsync

What "Unsync" does: it disconnects that page's instance from the sync. The content stays on that page as a regular, independent block — but it is no longer connected to the original. Changes to the synced original and changes to the unsynced copy are now independent.

What "Unsync" does not do: it does not delete the content, and it does not break the sync relationship between all the other pages. Every other page that contains the synced block continues to update normally.

To delete a synced block instance from a page entirely (not just unsync it), hover and click the delete option, or press Backspace with the block selected. This only removes it from that specific page.

5 Best Use Cases for Notion Synced Blocks

1. Team Handbook Header/Footer

Put your company values, key contacts, or "how to use this wiki" instructions in a synced block that appears at the top of every handbook page. Update the header once; all 40 pages reflect it immediately.

2. Recruiting / Open Roles Banner

Add a "We're hiring for [Role]" banner to your team pages and project wikis. When the role is filled or a new one opens, update one synced block and every page updates at once.

3. Shared Navigation Block

Build a sidebar-style navigation block with links to your key pages. Embed it on every project page and team dashboard. Add a new link once; it appears on all pages instantly.

4. Legal Disclaimers or Standard Terms

If your templates require identical disclaimer language, put it in a synced block. When your terms change, you update once and every document is current — no manual find-and-replace hunt.

5. Weekly Team Updates Block

A "This week's priorities" block synced to every team member's personal dashboard and the project overview. One update from the team lead; every dashboard shows it.

Limitations of Notion Synced Blocks

What Synced Blocks Cannot Do

  • Cannot span workspaces — synced blocks only work within a single Notion workspace
  • Cannot be embedded in database properties — they live on pages, not in text, URL, or other property fields
  • Cannot sync across pages in different-owner workspaces — guest-shared pages have limited sync behavior
  • May affect page performance — very large synced blocks with embedded databases can slow page load on pages with many such blocks
  • Not available as a database template block — you cannot use a synced block as a database row template that propagates globally
  • No cross-workspace sharing — synced blocks do not appear in Notion's public web publishing features in a synced way

Frequently Asked Questions

What is a Notion synced block?
A Notion synced block is a block of content that appears on multiple pages simultaneously. When you edit the block on any page, the change is instantly reflected on every other page that contains the synced copy. It solves the problem of copy-pasting content that then goes stale across your workspace.
How do I create a synced block in Notion?
Type /synced block on any Notion page and press Enter. A red-outlined block container appears. Add your content inside it. To place the same synced block on another page, hover over the original block, click the three-dot menu, and select "Copy and sync." Then paste it on the target page with Ctrl+V or Cmd+V.
Can I edit a synced block from any page?
Yes. You can edit a synced block from any page that contains it, and the change appears everywhere instantly. There is no "original" in terms of editing priority — the most recent save wins if edits happen nearly simultaneously on different pages.
What happens if I unsync a synced block?
Unsyncing removes the sync connection for that particular page's instance only. The content stays on the page as a regular block, but changes to other synced instances no longer affect it. The original synced block and all other copies remain fully synced with each other — unsyncing is local to the one page you unsync it from.
Can synced blocks work across different Notion workspaces?
No. Synced blocks only work within the same Notion workspace. You cannot sync content across different workspaces or share a synced block with a guest who is in a separate workspace.
