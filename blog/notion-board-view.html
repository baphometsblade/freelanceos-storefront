Notion Board View: The Complete Guide to Kanban Boards in Notion

Create kanban boards in Notion, customize columns and cards, filter by assignee, and use board view for project management, content pipelines, and client CRM.

July 2026  ·  10 min read
Home › Blog › Notion Board View Guide 2026

Notion's Board View is one of its most visually powerful features — a kanban-style layout that turns any database into a drag-and-drop workflow board. Whether you're managing a content pipeline, tracking client projects, running a CRM, or planning a product launch, the board view makes the status of every item instantly visible.

This guide covers how to create a board view from scratch, customize it for your specific use case, and use filters and grouping to build a board that actually reflects how you work.

What Is Notion Board View?

Notion Board View is a visual layout for any Notion database that groups records into columns based on a Select or Status property. Each option in that property becomes a column — and each database record (a project, task, article, client, etc.) appears as a card within the matching column.

The core mechanics:

  • Columns = property options — If your Status property has options like "Backlog," "In Progress," "Review," and "Done," those become your four kanban columns.
  • Cards = database records — Each record in your database appears as a card in the column matching its current status.
  • Drag to update — Drag a card from one column to another and the Status property updates automatically. No clicking into the record to change it.
  • Multiple views, same data — The board view is just one view of your database. Your table view, calendar view, and board view all show the same records — just presented differently.

How to Create a Board View in Notion

From an Existing Database

If you already have a database in Notion, adding a board view takes seconds:

  1. Open the page containing your database.
  2. Click the + Add a view button at the top left of the database (it may appear as "Add view" or a "+" icon depending on your layout).
  3. Select Board from the list of view types.
  4. Name the view (e.g., "Kanban Board," "Project Pipeline," "Content Calendar").
  5. Notion automatically groups by your Status property, or the first Select property it finds. To change this: click Group in the view toolbar, then select the property you want to group by.
  6. Your board is live — existing records appear in columns based on their current property value.

Creating a New Database as a Board View

  1. On any Notion page, type /board and select "Board" from the block options.
  2. Notion creates a new empty board-view database with a default Status property and sample columns.
  3. Rename the columns by clicking the column header and editing the property options.
  4. Add properties to your database (due date, assignee, priority, etc.) by clicking the + button in a card or by going to the database properties.
  5. Add cards by clicking the + New button at the bottom of any column, or the + button that appears when you hover over a column.
Tip
Use a Status property (not a plain Select) as your grouping property when possible. The Status property type has built-in color coding (gray for "Not started," blue for "In progress," green for "Done") and integrates with Notion's task-completion features. Plain Select works too, but Status gives you visual polish without extra setup.

Customizing Board Columns

The columns in your Notion board view are determined by the options in the grouping property. Here's how to manage them:

Adding, Renaming, and Reordering Columns

  • Add a column: Click + Add a group at the far right of the board to add a new option to your Select/Status property.
  • Rename a column: Click the three-dot menu on the column header and select "Rename." This renames the property option.
  • Reorder columns: Drag the column header left or right to reorder. The column order in the board view reflects the option order in the Select/Status property — change either one to reorder.
  • Change column color: Click the three-dot menu on the column header and select a color. This changes the property option color.
  • Hide a column: Click the three-dot menu on the column header and select "Hide." The column becomes invisible in this view but the property option still exists. Cards assigned to that status are hidden from the board (not deleted).

Hiding Empty Columns

By default, Notion shows all property options as columns even if they have no cards. To hide empty columns: click the Group button in the view toolbar and toggle on "Hide empty groups". The board now only shows columns that contain at least one card.

Card Properties on the Board

Each card on the board can display a selection of the database's properties beneath the card title. Showing the right properties — and only those properties — is key to a readable board. Too many properties per card creates visual noise; too few leaves you clicking into every card for basic information.

Recommended Properties to Show on Cards

  • Always Due date — the most critical piece of information for any workflow board. Color-codes red when past due.
  • Team use Assignee — shows a profile photo or avatar so you can see at a glance who owns each card.
  • Optional Priority — a Select property (High / Medium / Low) with color options. Useful for prioritized backlogs.
  • Optional Client / Project — a Relation property linking the card to a client or parent project. Shows which client or project context a task belongs to.
  • Avoid Long text / Notes — multi-line text fields look bad on cards and take up too much space. Keep notes in the card's full-page view, not the board card.

How to Control Which Properties Show on Cards

  1. In the board view, click Properties in the view toolbar (or the three-dot menu → Properties).
  2. Toggle properties on or off to control what appears on each card in this view.
  3. Properties are view-specific — showing a property in the board view doesn't affect what appears in the table view.

Board View vs. Table View vs. List View

Notion lets you view the same database in multiple ways. Here's when to use each:

View Type Best For Strengths
Board View Tracking workflow stages, seeing what's in progress Visual status overview, drag to update, at-a-glance pipeline health
Table View Viewing all properties, bulk editing, data entry Most information dense, easy to filter/sort, spreadsheet-like
List View Simple to-do lists, reading long titles Clean, minimal, shows full titles without truncation
Calendar View Deadline-driven planning, content scheduling Visual timeline, see all due dates in a monthly/weekly grid
Gallery View Image-heavy databases, visual portfolios Shows cover images prominently, good for design assets or product catalogs

The most effective Notion setups use the board view as the primary working view — the place you check daily — and keep the table view available for bulk edits, filtering across all records, or exporting. Both views draw from the same database, so changes in either view update everywhere instantly.

Filtering and Grouping in Board View

Filters in Notion board views are powerful but underused. The most impactful filter combinations:

Filter by Assignee (Your Personal View)

In a shared team board, create a filtered board view showing only cards assigned to you: click Filter → Add filter → Assignee → is → [Your name]. Save this as a named view like "My Tasks." Now you have a personal board showing only your work, while the unfiltered board shows the full team pipeline.

Filter by Due Date (What's Due This Week)

Add a filter: Due Date → is within → The next 7 days. This creates a "Due This Week" board view that focuses you on what actually needs to move. No more scrolling past backlog items to find urgent work.

Group by a Different Property

You can change the grouping property to get a completely different perspective on the same data:

  • Group by Assignee — see each team member's workload as a column
  • Group by Priority — see High / Medium / Low priority work in separate columns
  • Group by Client — see all work for each client in its own column (great for freelancers managing multiple active clients)

Frequently Asked Questions

What is Notion Board View?
Notion Board View is a kanban-style view of a Notion database that organizes records into columns based on a Select or Status property. Each option in the property becomes a column, and each database record appears as a card within the appropriate column. You can drag cards between columns to update their status, and cards show customizable properties like due dates, assignees, and tags.
How do I create a board view in Notion?
To create a board view in Notion: open any existing database or create a new one, click the + Add a view button at the top of the database, select Board from the view type options. Notion will automatically group by the Status or first Select property it finds. You can change the grouping property in the view options. The board view is created instantly with your existing records distributed into columns based on their current status values.
Can I have multiple board views in Notion?
Yes. A single Notion database can have unlimited views, including multiple board views with different grouping properties, filters, and visible properties. For example, you could have a board view grouped by Status for your main project pipeline and a separate board view grouped by Assignee to see each team member's workload. All views draw from the same underlying data.
What's the difference between Notion Board View and Table View?
Notion Board View (kanban) organizes records into visual columns based on a status or category property — best for tracking workflow stages, seeing what's in progress, and moving items through a pipeline. Table View shows all records in a spreadsheet-style grid — best for viewing all properties at once, sorting, bulk editing, and data-heavy workflows. Both views display the same underlying database data; the choice depends on whether you need workflow visibility (board) or data density (table).
How do I hide empty columns in Notion Board View?
To hide empty columns in Notion Board View: click the Group button in the view toolbar and toggle on "Hide empty groups." This hides any column that has no cards. You can also hide individual columns by clicking the three-dot menu on a column header and selecting Hide. Hidden columns don't delete the property option — they just make it invisible in this view.
