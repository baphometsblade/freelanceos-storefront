1. What Is a Notion Database?

A Notion database is one of the most powerful features Notion offers — and one of the most misunderstood. At its core, a database is a structured collection of pages that all share the same set of properties. Think of it like a spreadsheet where every row is also a full document you can open and write inside.

This is what sets Notion databases apart from traditional tools like Excel or Google Sheets. In a spreadsheet, a row is just a row — a flat set of cells. In Notion, that same row (called an entry or item) is a rich page where you can write meeting notes, embed images, track tasks, attach files, and more.

Compared to Airtable, which is the closest competitor, Notion databases offer the advantage of living inside your broader workspace alongside your documents, wikis, and personal notes. Airtable is more powerful for complex relational data and external integrations, but Notion wins for individuals and small teams who want everything in one place.

When should you use a Notion database?

Use a Notion database any time you have a collection of items that share common attributes. Classic examples include:

  • A client tracker (each client is a page with properties like status, contract value, and deadline)
  • A content calendar (each post is a page with publish date, platform, and draft status)
  • A project manager (each project has owner, priority, start date, and linked tasks)
  • A reading list (each book has genre, author, rating, and notes)
  • An invoice tracker (each invoice has client, amount, due date, and paid status)

If you're listing more than five or six items that all look the same, it's time to build a database.

2. The 6 Notion Database Views Explained

One of Notion's most underused superpowers is that the same database can be displayed in six completely different ways — without changing the underlying data. Each view is just a lens. Here's what each one is best for:

ViewDescriptionBest For
TableClassic spreadsheet grid with rows and columnsReviewing all properties at once, data entry, sorting large datasets
BoardKanban-style columns based on a Select propertyProject management, tracking status stages (To Do → In Progress → Done)
GalleryVisual card grid, can show cover imagesVisual content (design assets, portfolio pieces, product catalogs)
CalendarMonth/week grid based on a Date propertyContent calendars, scheduling, event planning
TimelineGantt-style horizontal bar chart with start and end datesProject planning, showing overlapping tasks, milestone tracking

To add a new view to an existing database, click the "+ Add a view" button at the top of any database. You can create as many views as you need — some users have 5 or 6 views on the same database, each filtered and sorted differently for different use cases.

Pro Tip: Create a "Master" Table view where all properties are visible, then build filtered views for specific workflows. For example, a "My Active Projects" Board view filtered to show only items assigned to you with status "In Progress."

3. Creating Your First Notion Database

Building your first Notion database takes less than 10 minutes. Here's the step-by-step process:

1

Open a new Notion page

Click the "+" button in your left sidebar to create a new page. Give it a name like "My First Database." This page will house your database.

2

Type /database and choose your view

On the blank page, type /database to bring up the command menu. Select "Table" for your first database — it's the easiest view to understand and navigate.

3

Name your columns (properties)

Notion creates a default "Name" column. Click the "+" button to add more columns. For a simple task database, add: Status (Select), Due Date (Date), Priority (Select), and Notes (Text).

4

Add your first few rows

Click an empty row or press the "New" button to add your first entry. Fill in each property. Click the row to open the full page and add richer content inside it.

5

Add a second view

Click "+ Add a view" at the top. Choose "Board" and group by Status. Now you have a Kanban board of the same data — without creating anything new.

6

Save and start using it

Your database auto-saves. Bookmark it in your sidebar by dragging it to a prominent spot. Start adding real data and you'll quickly see how the views help different workflows.

Quick win: After creating your database, immediately add a few test entries with dummy data. Seeing your properties in action — especially sorting and filtering — makes the learning curve disappear fast.

5. Filters and Sorts: Show Only What Matters

Filters and sorts transform a database from a messy pile of data into a focused, actionable view. The key insight: filters and sorts are per-view. You can have one view showing all items sorted by date, and another showing only overdue items sorted by priority.

Adding Filters

Click the Filter button at the top of any database view. You can filter by any property — for example:

  • Status is "In Progress" — show only active items
  • Due date is before Today — show overdue tasks
  • Assigned to is Me — show only your tasks
  • Priority is "High" AND Status is not "Done" — show urgent open items

You can stack multiple filters using AND/OR logic. Click "Add filter" to add more conditions. Use OR groups when you want entries that match any one of several criteria.

Adding Sorts

Click the Sort button to add a sort rule. You can sort by any property, ascending or descending. You can stack sorts too — for example, sort by Priority (high to low) first, then by Due Date (oldest first) as a tiebreaker.

Important: When you manually drag and reorder items in a database, Notion sets the sort to "Manual." This overrides any sort rules you've added. If your sort stops working, check whether manual ordering is active.

6. Linked Databases: Connect Everything Together

A linked database is a view of an existing database that you embed somewhere else in your workspace. The data is the same — you're just looking at it through a new lens placed in a different location.

For example, your master Projects database might live in a "Projects Hub" page. But you also want to see relevant projects from your client's dedicated page. Rather than duplicating data, you create a linked database view on the client page, filtered to show only their projects.

To create a linked database, type /linked view of database in any page. Select which database to link, then apply filters and sorts specific to that context.

7. Database Relations and Rollups: The Power Feature Beginners Ignore

Relations and Rollups are the features that turn Notion from a glorified note-taking app into a genuine relational database. Most beginners skip them entirely — and their databases suffer for it.

How Relations Work

A Relation property links entries in one database to entries in another. For example:

  • In your Projects database, add a Relation to your Clients database
  • Now each project can be linked to the client it belongs to
  • On the client's page, you automatically see all their related projects

To add a relation: click "+" to add a property → choose "Relation" → select the database you want to link to → decide whether to show the relation on both databases (recommended).

How Rollups Work

A Rollup property aggregates data from a related database. Once you have a relation set up, you can add a rollup to pull statistics across the link:

  • Count: How many projects does this client have?
  • Sum: What's the total invoice value for all their projects?
  • Earliest date / Latest date: When is their next deadline?
  • Percent checked: What percentage of their tasks are complete?

To add a rollup: add a Relation first → add a new property → choose "Rollup" → select the relation, the property to pull from, and the calculation type.

Real-world example: Link your Tasks database to your Projects database. Add a rollup on Projects to count checked tasks and total tasks. Now each project automatically shows its completion percentage — no manual updating needed.

8. Notion Formulas: 10 Practical Formulas to Get Started

Notion formulas use a syntax similar to spreadsheet formulas. They're written in the Formula property and can reference other properties in the same database using prop("Property Name"). Here are 10 formulas you can copy and use immediately:

1. Days Until Deadline

dateBetween(prop("Due Date"), now(), "days")

Shows the number of days remaining until the due date. Returns a negative number if overdue.

2. Is Overdue?

and(not(prop("Done")), dateBetween(prop("Due Date"), now(), "days") < 0)

Returns true if the item is not done AND the due date has passed.

3. Priority Score (for sorting)

if(prop("Priority") == "High", 3, if(prop("Priority") == "Medium", 2, 1))

Converts a Select property into a number so you can sort numerically by priority.

4. Days Since Created

dateBetween(now(), prop("Created"), "days")

Shows how old an entry is in days.

5. Total with Tax

prop("Amount") * 1.2

Adds 20% tax to an invoice amount. Change 1.2 to your local tax rate.

6. Status Emoji

if(prop("Status") == "Done", "✅", if(prop("Status") == "In Progress", "🔄", "⏳"))

Adds a visual emoji to your status for quick scanning in gallery or list views.

7. Word Count Estimate

length(prop("Content")) / 5

Rough word count estimate from a text property (assumes average 5 characters per word).

8. Deadline This Week

and(dateBetween(prop("Due Date"), now(), "days") >= 0, dateBetween(prop("Due Date"), now(), "days") <= 7)

Returns true if the deadline falls within the next 7 days.

9. Full Name

prop("First Name") + " " + prop("Last Name")

Concatenates two text properties into a full name.

10. Completion Percentage Display

format(round(prop("Tasks Done") / prop("Total Tasks") * 100)) + "%"

Converts a numeric ratio to a readable percentage string.

9. Gallery Databases: Make Your Data Visual

The Gallery view transforms your database entries into a grid of visual cards. Each card can display an image (from a page cover, file property, or URL), making it ideal for visual content like design assets, products, client logos, or portfolio pieces.

Setting up Gallery view

  • Click "+ Add a view" → choose Gallery
  • Click the three-dot menu → "Properties" to choose which properties appear on each card
  • To show images, go to "Layout" settings and change "Card preview" to "Page cover" or "Files & media"
  • Set card size to "Small," "Medium," or "Large" depending on image importance

Best use cases for Gallery databases

  • Portfolio tracker: Each project has a cover image, client name, and completion date
  • Product catalog: Images of each product with price and status
  • Design asset library: Screenshots of UI components or brand assets
  • Team directory: Headshots of team members with role and contact info

Let's build a real, useful database from scratch: a freelance client tracker. This is one of the most valuable databases any freelancer can have.

Add these properties

Create views

Add a Board view grouped by Status. Add a Table view sorted by Start Date descending. Add a Calendar view based on End Date to see upcoming deadlines.

4

Add a formula for days remaining

Add a Formula property: dateBetween(prop("End Date"), now(), "days") — this shows how many days until the contract ends.

5

Add a filter to the board view

On the Board view, filter to show only items where Status is not "Complete." This keeps your active clients front and center.

6

12. Common Mistakes That Break Your Notion Database

After helping thousands of freelancers set up Notion, here are the mistakes we see most often:

Mistake 1: Using text properties for everything

Text is flexible but unsortable, unfiltered, and hard to analyze. Replace free-text status fields with Select properties, and free-text numbers with Number properties.

Mistake 2: Creating too many properties upfront

Start with 5–7 essential properties. You can always add more later. An overcrowded database with 20 properties becomes overwhelming to maintain and visually cluttered.

Mistake 3: Not using views

Keeping one unfiltered table view for all use cases means you're always sorting and searching through noise. Create dedicated views for each workflow — one for active items, one for this week's deadlines, one for the kanban board.

Mistake 4: Duplicating data instead of linking databases

If you have a client's name in both your Projects and Invoices databases, that's a sign you should be using Relations instead of copy-pasting. Duplication creates inconsistency and maintenance headaches.

Mistake 5: Ignoring the page inside each entry

Click any database row and you'll find a full blank page. Most beginners never use it. This is where the real power is — add meeting notes, deliverable checklists, email history, and project briefs directly inside each entry.

Avoid this: Deleting a database property deletes all the data stored in that property across every entry — permanently. Always archive or rename instead of deleting when you're unsure.

Frequently Asked Questions

What is a Notion database?
A Notion database is a structured collection of pages that all share the same set of properties. Unlike a spreadsheet row, each database entry is a full page you can open and write inside — combining the structure of a spreadsheet with the richness of a document.
What are the six Notion database views?
Notion offers six views: Table (spreadsheet grid), Board (Kanban columns), Gallery (visual card grid), Calendar (date-based month/week layout), Timeline (Gantt-style bar chart), and List (compact vertical list). All six views show the same data — they're just different lenses on the same database.
Can I link two Notion databases together?
Yes. Notion's Relation property lets you link entries in one database to entries in another. Once linked, the Rollup property lets you aggregate data across the connection — for example, counting all tasks linked to a project, or summing all invoice amounts for a client.
What is the difference between a Notion database and a spreadsheet?
The key difference is that each row in a Notion database is a full page where you can write documents, embed media, and build sub-tasks. Spreadsheets are flat grids of cells. Notion also supports multiple simultaneous views, rich property types like relations and rollups, and per-view filters and sorts.
Are Notion formulas difficult to learn?
Basic Notion formulas are approachable if you've used Excel or Google Sheets. They use similar syntax with functions like if(), dateAdd(), and prop(). Most common formulas — like calculating days until a deadline or flagging overdue items — can be copied from templates and adapted without deep formula expertise.
Is Notion free for database use?
Yes. Notion's free plan supports unlimited databases, all six views, and most property types including relations and formulas. Paid plans add advanced permissions, unlimited team blocks, and higher API rate limits — but for individual freelancers, the free plan covers all core database functionality.
