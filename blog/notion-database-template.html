If you have ever tried to run a freelance business, manage a content pipeline, or stay on top of a complex project using simple lists and sticky notes, you already know the problem: information lives in six different places, nothing connects, and finding what you need takes longer than actually doing the work. Notion databases solve exactly this problem — and a good Notion database template gets you past the blank-canvas paralysis that stops most people from building systems in the first place.

A Notion database template gives you a pre-built system with the right properties, views, and filters already configured — so you can start capturing information in minutes instead of spending hours figuring out the optimal setup. Whether you need a CRM to track clients, a content calendar to manage your publishing schedule, an expense log to monitor spending, or a task tracker to stay on top of deliverables, there is a database structure that fits your workflow immediately.

In this guide, you will learn what makes Notion databases uniquely powerful compared to plain tables and spreadsheets, how to use the six core property types and five view modes, and get detailed setup instructions for 15 of the most useful database templates you can build or import today. We also cover advanced features — relations between databases, rollup calculations, and formula fields — that transform a simple table into a fully interconnected workspace where information flows between systems automatically.

1. What Is a Notion Database?

Before diving into templates, it is worth understanding what separates a Notion database from a regular table or spreadsheet. On the surface they look similar — rows and columns — but the underlying capabilities are radically different, and understanding the distinction will change how you think about organizing your work.

A plain Notion table (inserted with /simple table) is a basic, static grid. You can type text into cells and adjust column widths, but that is about it. There is no filtering, no sorting, no multiple views, and no ability to link to other pages or databases. It is the equivalent of a basic HTML table: useful for presenting simple data, not designed for managing it dynamically over time.

A Notion database, by contrast, treats every row as a full page. Click the open icon on any row and you get a complete Notion page with a rich text body — you can write notes, embed files, add comments, link to other pages, and include any block type Notion supports. The column headers in a database are not just labels: they are typed properties (text, number, date, select, checkbox, relation, and more) that enable filtering, sorting, aggregation via rollups, and computation via formulas.

What Databases Enable That Tables Do Not

  • Multiple views on the same data — Switch between Table, Board, Calendar, Gallery, and List views without duplicating any information
  • Filtering and sorting — Show only tasks with Status = "In Progress" or sort invoices by Due Date descending, with filters that persist per view
  • Relations between databases — Link a Task to a Project and a Project to a Client, creating a navigable web of information across your entire workspace
  • Rollup properties — Pull aggregate values from related databases (sum all invoice amounts per client, count open tasks per project, find the earliest deadline across related entries)
  • Formulas — Compute values from other properties (days until deadline, progress percentage, conditional status labels, total cost calculations)
  • Row-level pages — Each row has its own rich text body, so a task entry can hold a full brief, or a client entry can hold a running meeting log
  • Database row templates — Create row templates with pre-filled properties and body content so new entries start in a consistent state every time
  • Saved views with different configurations — Each view can have its own filters, sorts, and visible properties, letting you have a data-entry view, a review view, and a board view all coexisting on the same database

This is why a Notion database template is fundamentally different from a spreadsheet template. You are not just getting a formatted table — you are getting a data architecture with built-in behaviors, navigation, and the ability to scale into a multi-database system.

Full page vs. inline: When you create a database, decide whether you want it full page (the database takes over the whole Notion page — best for primary working databases like your Task Tracker or CRM) or inline (embedded within a larger page alongside other content — best for supplementary reference data). You can convert between them, but starting with the right format saves restructuring later. Most high-use databases benefit from full-page mode.

2. The 6 Notion Property Types

Notion offers over 20 property types, but the vast majority of practical database templates are built on just six core types. Master these and you can design virtually any system you need — from a simple reading list to a fully interconnected project management suite.

1. Text

The most basic property type — a free-form text field that holds any string of characters. Use it for vendor names, descriptions, URLs, short notes, and any information that does not need structure or type enforcement. Text properties appear as a single line in table view but can hold much more content when a row is opened as a page. Common uses: Contact email (when you want plain text, not a clickable mailto), Company name, Short description, LinkedIn URL, Invoice number, Notes.

2. Number

Stores numeric data with optional formatting — plain number, dollar currency (with locale-appropriate symbol), percent, or other units. Numbers are required for Rollup aggregations: you cannot sum text fields, so any value you want to total across related entries must be stored as a Number property. They also enable formula-based calculations like progress bars, budget remaining, and per-unit costs. Common uses: Invoice amount, Hourly rate, Estimated hours, Word count, Priority score, Rating out of 10, Budget, Servings, Prep time in minutes.

3. Select and Multi-select

Select lets you choose exactly one option from a defined list; each option gets a color label. Multi-select lets you pick multiple options from the same list — displayed as stacked color chips. Both types are ideal for status tracking, categorization, and any attribute where the values are finite and predictable. Select is particularly important because Board view groups rows by a Select property — each option becomes a column in the kanban board. Common uses: Status (To Do / In Progress / Done), Priority (High / Medium / Low), Category (Marketing / Finance / Admin), Platform (Twitter / LinkedIn / Instagram), Content Type, Genre, Cuisine.

4. Date

Stores a date (and optionally a start-and-end date range or a specific time). Date properties unlock Calendar view — any database with at least one Date property can switch to Calendar mode and display entries on a monthly or weekly calendar. You can also attach reminders to Date properties and reference them in formula properties for deadline calculations. Common uses: Due date, Publish date, Application date, Last contact date, Start date, End date, Date finished, Apply-by date, Next meeting.

5. Checkbox

A simple boolean toggle — checked (true) or unchecked (false). Checkboxes are ideal for task completion, habit logging, binary flags, and filtering. In formula properties, you can reference checkboxes to conditionally compute values: if the "Paid" checkbox is true, the formula returns "Closed"; otherwise it checks whether the due date has passed and returns "Overdue" or "Open." Rollup properties can count checked vs. unchecked rows across related entries. Common uses: Completed, Paid, Tax deductible, Reimbursable, Applied, Priority flag, Followed up, Published.

6. Relation

The most powerful property type in Notion's toolkit. A Relation creates a persistent, two-way link between rows in two different databases (or rows within the same database). Once a Relation is established, each row displays clickable links to its related rows — click to navigate directly to the related page. Relations also enable Rollup properties, which aggregate data from the related database automatically. Common uses: Task → Project, Project → Client, Invoice → Client, Article → Content Series, Goal → Milestone, Expense → Project, Book Note → Reading List entry.

Other property types worth knowing: File and Media (attach receipts, cover images, PDFs directly to a row), URL (clickable links that open in a new tab — use this instead of Text for links you want to click), Email (mailto-linked addresses — one click to open your mail client), Phone (click-to-call on mobile), Formula (computed values from other properties), Rollup (aggregated values from a related database), Created time / Last edited time (auto-populated timestamps — great for audit trails and activity logs), Person (assigns workspace members to rows, enabling filtering by assigned user).

3. The 5 Notion Database Views

Every Notion database can display its data through multiple views simultaneously. Adding a view does not copy or duplicate the data — it is simply a different way of rendering the same rows and properties. You can have a Table view for bulk data entry, a Board view for workflow management, and a Calendar view for scheduling, all pointing at the same underlying database and updating in sync.

Table View — The Spreadsheet

The default view when you create a database. Shows all rows as a grid with properties as column headers. Best for data entry, bulk editing, sorting, and getting a comprehensive overview of all entries simultaneously. This is the view you use when you want to see everything at once and work with raw data. Each column can be shown or hidden independently per view — your Table view can display only the four properties you use most often, while your Gallery view still renders all properties on each card. Use the column footer "Calculate" feature to see Count, Sum, Average, or other aggregates directly in the table without building a rollup.

Board View — The Kanban

Groups rows into vertical columns based on a Select property. The quintessential use case is Status (To Do / In Progress / In Review / Done), where each card moves from column to column as work progresses. Drag cards horizontally to update the Select property — no opening the row required. Board view is the best way to visualize workflow stages and identify bottlenecks: a column with 20 cards and no movement signals a problem you can spot in three seconds. Excellent for task management, sales pipelines, content production workflows, and any process with distinct sequential stages.

Gallery View — The Visual Cards

Displays rows as a grid of visual cards, optionally showing a cover image, page cover, or File and Media attachment as the card's thumbnail. Best for content libraries, recipe books, inspiration boards, reading lists, mood boards, and any database where a visual element adds meaningful context or makes scanning faster. To make Gallery view compelling, add a File and Media property and upload an image to each row, or set a page cover inside the row — Notion uses it as the card thumbnail automatically. Configure which properties appear on each card for a clean, information-rich grid.

Calendar View — The Scheduler

Maps rows onto a monthly (or weekly) calendar based on whichever Date property you choose as the calendar source. Essential for content calendars, editorial planning, deadline tracking, project timelines, and scheduling anything time-sensitive. Cards on the calendar are draggable — move a row to a different date and the Date property updates automatically without opening the row. Filter the Calendar view to show only specific statuses (e.g., only entries where Status = Scheduled) to eliminate noise and keep the calendar focused on committed commitments. Use a date range property to show entries that span multiple days.

List View — The Minimal Mode

The simplest view Notion offers. Displays rows as a clean, compact list with only the title property visible by default, plus any properties you choose to show inline. No visual noise, no column grid — just entries and their most critical metadata, tightly packed. Best for reading lists, log entries, quick-reference checklists, and any database where information density matters more than visual richness. List view loads exceptionally fast and renders cleanly on mobile — for databases you access frequently on your phone, List is often the most practical primary view.

View Best For Key Configuration
Table Data entry, full overview, bulk edits, sorting Show/hide columns per view; use Calculate footer for aggregates
Board Workflow stages, pipeline management, status tracking Group by any Select property; drag cards to update status
Gallery Visual libraries, recipe books, inspiration, content archives Cover image source (File property or page cover)

4. Task Tracker Database

The Task Tracker is the foundational Notion database template — the one most people build first, and the one that immediately demonstrates why Notion outperforms a conventional to-do app. Unlike a simple checklist, a Notion task tracker treats each task as a rich document: the row is the task, and the page body holds the full brief, research, sub-tasks, and context. Nothing lives in a separate doc, email thread, or sticky note.

Core Properties to Set Up

  • Name (Title) — The task description, written as an outcome ("Draft homepage copy," not "Homepage")
  • Status (Select) — To Do / In Progress / Blocked / In Review / Done — color-coded green for Done, red for Blocked
  • Priority (Select) — High / Medium / Low — color-coded red, yellow, gray for fast visual scanning
  • Due Date (Date) — With a Notion reminder set 24 hours before for high-priority items
  • Project (Relation) — Links each task to a row in your Project Hub database
  • Estimated Hours (Number) — For capacity planning; roll these up in the Project Hub to see total hours per project
  • Completed (Checkbox) — Toggle when truly done; use this in filter views rather than Status = Done for faster toggling
  • Context (Text) — One line of brief context that appears in table view without opening the row

Setup Instructions

Create the database with the properties above, then build four views: (1) a Table view for data entry with all properties visible, (2) a Board view grouped by Status for workflow management, (3) a Calendar view by Due Date for deadline visibility, and (4) a filtered Table view showing only rows where Completed = unchecked and Due Date is within the next 7 days — save this as "This Week" and set it as your default view so you open to your immediate priorities every time.

Inside each task row, write the deliverable brief in the page body. Use Notion to-do checkbox blocks for sub-tasks, paste reference links, and describe the specific output expected. This transforms the task tracker from a list of labels into a genuine project management tool where every entry is actionable without opening another application or asking a colleague for context. Set up a row template with pre-filled headings ("Goal," "Deliverable," "Blockers," "References") so every new task starts structured.

Saved view strategy: Create at least three named views on your Task Tracker — "All Tasks" (unfiltered Table), "This Week" (filtered by upcoming Due Date and Completed unchecked), and "By Project" (grouped by the Project Relation). You will switch between these constantly. The five seconds to click a named view versus manually adjusting filters every session adds up to a meaningful workflow improvement over time.

5. CRM and Client Database

A Notion CRM template might be the highest-leverage database a freelancer can build. It centralizes everything you know about every client — contact details, project history, communication logs, invoicing status — in one place that is actually connected to your other databases. This replaces the scattered combination of email threads, spreadsheet contacts, and mental notes that most freelancers depend on.

Core Properties to Set Up

  • Client Name (Title) — Company or individual name; make this the company for B2B clients
  • Contact Person (Text) — Primary point of contact name at the company
  • Email (Email) — Click-to-email directly from the table view row without opening the entry
  • Phone (Phone) — Click-to-call on mobile; useful for immediate access during calls
  • Status (Select) — Lead / Active / On Hold / Past Client / Lost — your entire client lifecycle in one property
  • Last Contact (Date) — When you last communicated; powers "due for follow-up" filtering
  • Industry (Select) — Sector the client operates in; useful for segmenting your pipeline
  • Website (URL) — Their site; opens in one click for quick research before calls
  • Source (Select) — Where the lead came from: Referral / Inbound / Cold Outreach / Social / Event
  • Projects (Relation) — Links to your Project Hub database; navigate to all their projects in one click
  • Invoices (Relation) — Links to your Invoice Tracker; see all billing associated with this client
  • Total Revenue (Rollup) — Sums all invoice amounts from the Invoices Relation; shows lifetime client value automatically

Setup Instructions

Build the database with all properties above, then create three saved views: (1) a Table view with all properties visible for data management, (2) a Board view grouped by Status to visualize your lead pipeline and client lifecycle stages at a glance, and (3) a List view with Email and Last Contact showing inline for a fast reference view during outreach sessions. Inside each client row, maintain a running log of meeting notes in the page body — use dated headings (e.g., "2025-07-03 — Kick-off call") to create a timestamped history of every interaction.

The relational power of this setup becomes clear once you add the Projects and Invoices Relations. Navigate from a client to all their active and past projects in one click, and see their total lifetime spend via the Rollup without any manual summation. When a client calls, open their record and you have their contact, status, project history, invoice totals, and full communication log in a single page. This turns a contacts list into a real business CRM without any third-party tool or monthly subscription.

Follow-up filter: Create a saved view called "Follow Up Needed" with a compound filter: Status = Active AND Last Contact is before 14 days ago. Every Monday morning, open this view. Any client whose relationship has gone quiet surfaces immediately. Proactive outreach from this list prevents client churn and often generates new project conversations before you have to hunt for work.

6. Content Calendar Database

A content calendar database gives you a bird's-eye view of your entire publishing schedule and a production management system for each piece — from idea to published. If you are creating content across multiple platforms (blog, social, YouTube, newsletter), a Notion content calendar is far more practical than a spreadsheet because each row can hold the full draft, outline, research notes, and image assets alongside the scheduling metadata — all in one place.

Core Properties to Set Up

  • Title (Title) — The content piece title or a working title during drafting
  • Platform (Multi-select) — Blog / Twitter / LinkedIn / YouTube / Newsletter / Instagram — use Multi-select to flag cross-posted content
  • Status (Select) — Idea / Outlining / Drafting / Editing / Scheduled / Published
  • Publish Date (Date) — Target publication date; this drives the Calendar view
  • Word Count (Number) — Track actual word count on completion; compare against targets
  • SEO Keyword (Text) — Primary target keyword for blog and YouTube content
  • Content Type (Select) — Article / Video / Thread / Newsletter / Carousel / Podcast Episode
  • CTA (Text) — The call-to-action in this piece; keeps messaging consistent
  • Performance (Number) — Post-publish metric: views, clicks, or engagement score; update after 30 days
  • Series (Relation) — Optional link to a Content Series database if you run recurring topic clusters

Setup Instructions

Add all properties above, then set up a Calendar view by Publish Date as your primary working view — this is the view you should open by default when you sit down to manage content. Filter it to show Status = Scheduled or Published only, so the calendar shows your committed pipeline without the noise of ideas and drafts. Add a Board view grouped by Status for production workflow management — cards move from Idea through the production pipeline as work progresses. Add a filtered Table view by Platform = Blog for your blog-specific editorial pipeline.

Inside each content row, write the full draft in the page body. Use Notion headings for article structure, paste in research notes and sources, embed images, and add a Notion to-do block for the pre-publish checklist (proofread, SEO check, featured image added, social snippets written). This makes the content calendar the single source of truth for your entire content operation — no hopping between Google Docs for drafts, Trello for workflow, and a separate Google Calendar for scheduling. Everything is in one database.

Monthly batch workflow: At the start of each month, spend 30 minutes creating all planned content entries with working titles and target Publish Dates, set to Status = Idea. Open Board view — your Idea column shows everything that needs to be written, ordered by publish date. Work through the pipeline left to right. The Calendar view validates whether your publishing volume is realistic before you commit to it.

7. Expense Log and Finance Database

Tracking business expenses in Notion is cleaner than a spreadsheet once you configure the right properties and views. A Notion expense tracker captures every business cost with enough detail for bookkeeping and tax preparation, without requiring separate accounting software for day-to-day tracking. The File and Media property makes it uniquely valuable: you can attach the actual receipt to the expense entry, eliminating the shoebox problem entirely.

Core Properties to Set Up

  • Description (Title) — What the expense was for, written plainly: "Adobe CC subscription — July 2025"
  • Amount (Number, currency format) — Dollar amount of the expense; use Notion's currency number format
  • Category (Select) — Software / Hardware / Marketing / Professional Development / Travel / Office / Contractor / Other
  • Date (Date) — When the expense occurred, not when you logged it
  • Vendor (Text) — Who you paid: Adobe, AWS, Upwork, Delta, etc.
  • Receipt (File and Media) — Attach the receipt image or PDF immediately when logging
  • Payment Method (Select) — Business Credit Card / Business Debit / PayPal / Bank Transfer / Personal (to be reimbursed)
  • Tax Deductible (Checkbox) — Flag deductible business expenses; filter this at tax time
  • Project (Relation) — Link to client project if the expense is billable or project-specific
  • Reimbursable (Checkbox) — Flag if you paid personally and need to bill the client back

Setup Instructions

Create the database with all properties above. Set up a Table view as the primary data-entry interface with Amount, Category, Date, and Vendor visible without scrolling. Enable the "Calculate" footer on the Amount column and select "Sum" — this shows your total spend for whatever filter is currently applied, giving you instant totals without a separate rollup. Add a Board view grouped by Category to see spending distribution at a glance. Create quarterly filtered views (Date is within Q1, Q2, Q3, Q4) and a Tax Deductible view filtered to Checkbox = checked for year-end reporting.

The most important habit: log expenses and attach receipts the moment they occur. The 60 seconds it takes to photograph and attach a receipt when the expense is fresh is infinitely easier than reconstructing months of spending from bank statements and email confirmations in April. On mobile, open the Notion app, create a new row, enter the amount and category, and use your phone camera to capture the receipt — done before you have put your wallet away.

Invoice Tracker (Template 14)

Closely related to the expense log, the Invoice Tracker is essential for freelancers who need to monitor cash flow and catch overdue payments before they become a problem. Core properties: Client (Relation to your CRM), Invoice Number (Text — your sequential numbering), Amount (Number, currency), Invoice Date (Date — when sent), Due Date (Date — typically 30 days out), Status (Select: Draft / Sent / Paid / Overdue / Cancelled), and Notes (Text — payment terms, late fee policy, reference numbers). Set the Board view grouped by Status as your primary view — overdue invoices appear in their own column, impossible to miss. Use a Rollup in your Client CRM to sum total revenue from all linked invoices, giving you lifetime value per client automatically.

The remaining templates cover the full spectrum of tracking, knowledge management, and life organization use cases that make Notion a genuinely comprehensive workspace. Each is self-contained but integrates cleanly with the CRM and Task Tracker via Relations.

The Project Hub is the connective tissue of a freelance Notion workspace — the database everything else links back to. Properties: Project Name (Title), Client (Relation to CRM), Phase (Select: Discovery / Active / In Review / Complete / Archived), Budget (Number, currency), Actual Spend (Rollup from Expenses Relation), Start Date (Date), End Date (Date), Tasks (Relation to Task Tracker), Total Tasks (Rollup: count all linked tasks), Completed Tasks (Rollup: count where Completed = checked). Create a Board view grouped by Phase as your primary project overview — you see your full pipeline at a glance, from Discovery through Active to Complete. Set up a Calendar view spanning Start Date to End Date to visualize project timelines and spot scheduling conflicts before they happen.

5. Reading List

A deceptively simple database that becomes a powerful knowledge asset over time. Properties: Title (Title), Author (Text), Genre (Select: Business / Productivity / Fiction / Design / Technology / Biography), Status (Select: To Read / Reading / Done), Rating (Number, 1 to 5), Date Finished (Date), Source (Select: Bought / Library / Gift / Borrowed), Key Takeaway (Text — one sentence compressed for table view). 8. Recipe Book

A personal recipe database that works equally well for food bloggers and home cooks who want to stop losing recipes across bookmarks, screenshots, and handwritten cards. 9. Job Search Tracker

For freelancers prospecting new clients through outbound approaches, or anyone in an active job search. Properties: Company (Title), Role (Text), Application Date (Date), Status (Select: To Apply / Applied / Phone Screen / Interview / Take-Home / Final Round / Offer / Rejected / Accepted / Withdrawn), Contact Name (Text), Contact Email (Email), Job URL (URL), Notes (Text — specific details from each stage), Follow-Up Date (Date — when to check in). 10. Travel Planner

Plan and document trips with a database that covers logistics, budget, and itinerary in one place. Properties: Destination (Title), Trip Dates (Date with start and end range), Budget (Number, currency), Actual Spend (Number — update from expense log after the trip), Accommodation (Text), Flight Info (Text — flight numbers and confirmation codes), Status (Select: Planning / Booked / In Progress / Complete), Tags (Multi-select: Business / Personal / International / Domestic / Solo / Group). Inside each trip row, write a day-by-day itinerary in the page body using Notion table blocks for accommodation details, maps links for each location, and to-do checkbox blocks for packing lists. Relation to your Expense Log lets you link all trip costs to the right travel entry for post-trip budget review.

11. Book Notes

A knowledge repository for book summaries and synthesis — distinct from a Reading List, which just tracks what you have read. The Book Notes database is where the intellectual work gets recorded and made retrievable. Properties: Book Title (Title), Author (Text), Date Completed (Date), Key Ideas (Text — top 3 concepts compressed to one line for table view), Rating (Number, 1 to 5), Action Committed (Checkbox — whether you have committed to implementing something), Source Book (Relation to Reading List). Inside each entry, write a full structured summary: core concepts with examples, memorable quotes with page numbers, specific actions you commit to taking, and connections to other books or ideas in your system. This database compounds in value over years — search it before starting any new project or making a significant decision.

12. Meeting Notes

Structured meeting records that make follow-up automatic and create a searchable history of every client interaction. Properties: Meeting Title (Title), Date (Date), Attendees (Multi-select of common contacts), Meeting Type (Select: Client Discovery / Project Review / Team Sync / 1:1 / Sales Call / Workshop), Client (Relation to CRM — links meeting history to the right client record), Status (Select: Upcoming / Complete), Action Items Summary (Text — brief for table view). Inside each meeting entry, write a structured body with headers for Agenda, Key Discussion Points, Decisions Made, Action Items (with assigned owner and due date as Notion to-do items), and Next Meeting date. Link the meeting row to a Task for each action item using the Task Relation — action items immediately enter your task management system.

13. Goal Tracker

Annual and quarterly goals with milestone tracking and progress visibility. Properties: Goal (Title), Category (Select: Revenue / Learning / Health / Creative / Relationships / Business), Deadline (Date), Target Value (Number — the measurable outcome you are aiming for), Current Value (Number — updated as you make progress), Progress Percent (Formula: round(prop("Current Value") / prop("Target Value") * 100)), Status (Select: Not Started / In Progress / On Track / Off Track / Complete / Abandoned). Create a Gallery view to display goals with their progress percentages as subtitle text — or keep a Table view sorted by Deadline ascending so the most urgent goals appear at the top. Review every Sunday: update Current Values, assess which goals are on or off track, and plan the week's work accordingly.

15. Learning Log

A capture system for lessons learned from books, courses, podcasts, events, and experience — the database that stops valuable insights from evaporating. Properties: Topic (Title), Source (Text — the specific book, course, or event), Source Type (Select: Book / Course / Article / Podcast / Video / Event / Experience), Date Learned (Date), Key Takeaways (Text — 2 to 3 bullet points compressed for table view), Apply By Date (Date — a self-imposed deadline for putting this insight into practice), Applied (Checkbox — toggle when you have actually acted on it). Filter Applied = unchecked to see your entire library of learned-but-not-yet-applied knowledge. This is often the most valuable view in the database — a curated list of things you already know you should be doing differently. The Notion formulas guide covers how to build a formula that flags overdue Apply By dates automatically.

9. Relations, Rollups, and Formulas Explained

The three advanced features that separate a basic Notion database from a fully interconnected workspace are Relations, Rollups, and Formulas. Once you understand how they work together, you can build systems that compute and surface information automatically — no manual tallying, no copy-pasting data between places, no stale numbers because someone forgot to update a spreadsheet cell.

Relations: Linking Two Databases

A Relation property creates a persistent two-way link between rows in different databases. To add one: open a database, click the + icon to add a new property, choose the "Relation" type, then select the target database from the list of all databases in your workspace. You immediately get a property that behaves like a searchable multi-select — click it to search for and select rows from the related database by name.

When you create a Relation in Database A pointing to Database B, Notion automatically creates a corresponding back-relation in Database B — so both databases can navigate to the other's rows without you building anything in the second database. You can name both sides of the Relation independently to make navigation intuitive.

The most valuable Relations to build in a freelance workspace:

  • Tasks to Projects — Each task belongs to one project; navigate from task to project context in one click
  • Projects to Clients — Each project belongs to one client; creates a three-level navigation chain from task to project to client
  • Invoices to Clients — Each invoice is tied to a client; enables revenue rollup per client in the CRM
  • Expenses to Projects — Link billable expenses to the right project for reimbursement tracking and profitability analysis
  • Book Notes to Reading List — Each set of notes corresponds to one reading list entry; navigate between quick reference and deep notes
  • Meeting Notes to Clients — Every client call has a linked meeting record; see the full communication history from the client page

Rollups: Aggregating Related Data

Once you have a Relation, you can add a Rollup property that aggregates values from the related database into the parent row. Click + to add a property, choose "Rollup," select which Relation to use, choose which property from the related database to aggregate, and pick a function from the list.

Available Rollup functions include: Count all, Count values (non-empty), Count unique values, Count checked (for checkboxes), Count unchecked, Sum, Average, Median, Min, Max, Range, Early date, Latest date, Date range, Show original, and Show unique values. This covers virtually any aggregation you would want to compute.

Practical Rollup examples to build immediately:

Project Hub: Total Hours

Rollup on Tasks Relation → Estimated Hours property → Sum function. Shows total estimated hours across all tasks in a project. Compare against your budgeted hours to see margin at a glance.

Project Hub: Task Progress

Rollup on Tasks Relation → Completed checkbox → Count checked. Add a second Rollup with Count all. Use in a Formula: checked divided by total gives completion percentage.

CRM: Lifetime Revenue

Rollup on Invoices Relation → Amount property → Sum function. Shows total revenue from every invoice linked to this client. Updates automatically as you add new invoices.

CRM: Open Invoices

Rollup on Invoices Relation → Status property → Count unique values, or filter the rollup to show only rows where Status is not Paid. Instantly see how many open invoices exist per client.

Content Series: Article Count

Rollup on Articles Relation → Name property → Count all. Shows how many articles belong to each content series, useful for managing editorial depth across topic clusters.

Goals: Milestone Progress

Rollup on Milestones Relation → Completed checkbox → Count checked. Divide by Count all in a Formula to get percent of milestones achieved. Shows goal progress without manual calculation.

Formulas: Computing Values from Properties

Formula properties let you write expressions that compute a new value based on other properties in the same database row. The formula editor supports conditional logic, date math, string manipulation, and arithmetic. If you know spreadsheet formulas, the concepts translate directly — the syntax is different but the logic is the same.

Common formula patterns for freelance and productivity databases:

  • Days until deadline: dateBetween(prop("Due Date"), now(), "days") — returns a negative number when the task is overdue (days since the deadline passed), positive when time remains. Use this to sort by urgency independent of the due date property.
  • Progress percentage: round(prop("Completed Tasks") / prop("Total Tasks") * 100) — requires two Rollup properties (completed count and total count) and produces a clean integer progress percentage.
  • Overdue invoice flag: if(prop("Paid"), "Closed", if(dateBetween(prop("Due Date"), now(), "days") < 0, "⚠ OVERDUE", "Open")) — checks the Paid checkbox first, then whether the due date has passed, and returns a human-readable status string.
  • Reading time estimate: round(prop("Word Count") / 200) + " min read" — divides word count by average reading speed (200 wpm), rounds to the nearest integer, and appends the unit. Useful in a Content Calendar.
  • Total recipe time: prop("Prep Time") + prop("Cook Time") — adds two Number properties. Assign a unit of "minutes" to the Formula output for clean display.
  • Days since last contact: dateBetween(now(), prop("Last Contact"), "days") — shows how many days have passed since the last client touchpoint. Use as a sort key in your CRM Follow-Up view.
Formula debugging: If a formula returns an error, the two most common causes are: (1) a property name is spelled incorrectly inside prop("Property Name") — names are case-sensitive and must match exactly including spaces, and (2) a type mismatch — for example, trying to do arithmetic on a Text property instead of a Number. Check both by hovering over the error indicator in the formula editor for Notion's hint about what went wrong.

Filtering and Sorting Tips

Getting the most from any database requires saving the right filters and sorts as named views. Every time you manually adjust filters to find what you need, you are doing work that should be automated by a saved view. Here are the essential filter patterns to build and save:

  • "This Week" filter: Due Date is within the next 7 days AND Completed = unchecked — your immediate priority list, always current
  • "High Priority" filter: Priority = High AND Status is not Done — everything urgent that is still open
  • "Active Clients" filter: Status = Active — your current working client list in the CRM
  • "Overdue Invoices" filter: Status = Sent AND Due Date is before today — invoices that need follow-up immediately
  • "This Month Content" filter: Publish Date is within the current month AND Status is not Published — your remaining publishing commitments for the month
  • "Unpracticed Learning" filter: Applied = unchecked in Learning Log — every insight that is waiting to be implemented

For sorting, the most powerful pattern is multi-level sorting: primary sort by Priority descending (High first), secondary sort by Due Date ascending (earliest deadline first within each priority group). This ordering surfaces the work that matters most and is most urgent simultaneously — no decision required about what to tackle next.

Gallery Covers: Making Visual Databases

Gallery view is significantly more useful when each card has a meaningful thumbnail. There are three ways to add cover images to gallery cards, each with different trade-offs.

Page cover: Open a database row as a full page and click "Add cover" at the top. Choose an Unsplash photo, upload an image, or pick from your Files. The page cover appears as the gallery card thumbnail automatically. Best for databases where you want a curated visual: recipes, book notes, content pieces.

File and Media property: Add a File and Media property to the database and upload an image directly in the property field without opening the full page. In Gallery view settings, set the card preview to show this property. Faster to populate than page covers and visible in table view as well.

External URL: Paste an image URL into a URL property, then use a formula to construct a styled output — or simply reference the URL in the page body as an embedded image block. Less clean than the above two options but works when you are linking to external hosted images like book covers via API.

For a recipe database, photograph each finished dish and upload to the File property. For a reading list, search for the book cover on Google Images or use Open Library's cover API. For a content library, screenshot the published piece and attach it to the database row after publishing. The few minutes spent adding covers to a Gallery database make the difference between a tool you use and a tool that feels like work to open.

10. Frequently Asked Questions

What is the best Notion database template?

The best Notion database template depends on your primary use case, but for freelancers the highest-leverage combination is a Task Tracker, CRM, and Invoice Tracker — all three linked via Relations. The Task Tracker keeps daily work organized, the CRM keeps client relationships warm, and the Invoice Tracker keeps cash flow visible. If you want all three pre-built and cross-linked without spending hours on setup, FreelanceOS includes 15+ databases already wired together so your tasks roll up into projects, your projects connect to clients, and your invoices show lifetime revenue per client automatically. For content creators, a Content Calendar linked to a Reading List and Learning Log is the equivalent foundational combination.

How do I create a Notion database from scratch?

Open any Notion page and type /table or /database in the body to open the block menu. Select "Table — Full page" for a database that occupies the whole page, or "Table — Inline" for one embedded within a larger page. Start with the default Name (Title) column, then click the + button at the top right of any column to add new properties — choose types like Select, Date, Number, Checkbox, or Relation for each attribute you want to track. Add rows with the + New button at the bottom, fill in data by clicking cells, and open a row as a full page by clicking the open icon on the left side of any row. Add views (Board, Calendar, Gallery, List) by clicking "+ Add a view" at the top of the database. The whole process for a basic setup takes under 10 minutes.

Can I share a Notion database?

Yes, and Notion gives you fine-grained control over sharing. Click Share in the top-right corner of any database page. You can invite specific people by email and assign them permissions of "Can edit," "Can comment," or "Can view." You can also generate a public share link that anyone with the link can access — useful for sharing a reading list or content calendar with a client or collaborator without requiring them to create a Notion account. Before enabling public sharing, review exactly what data is in the database, since all rows and properties become visible. For client-facing databases with sensitive financial data, prefer email-based invitations with limited permissions over public links.

How do Notion relations work?

A Relation property creates a persistent two-way link between rows in two databases. To set one up: add a new property to a database, choose "Relation" as the type, and select the target database. Each row can then link to one or more rows in the related database — click the Relation property cell to search for and select related entries. Notion automatically creates a back-relation in the target database so both sides can navigate to each other. Once a Relation is established, you can add Rollup properties to aggregate data from the linked rows — for example, summing invoice amounts per client, counting tasks per project, or finding the earliest deadline across all related entries. Relations are the foundation of a connected Notion workspace.

What's the difference between a database and a table in Notion?

A Notion database is fundamentally more powerful than a plain table, despite looking similar on the surface. A plain table (inserted with /simple table) is a static visual grid — you can type text into cells, but there is no filtering, sorting, multiple views, or ability to open rows as pages. A Notion database, by contrast, treats every row as a full Notion page with a rich text body; supports five different view types (Table, Board, Calendar, Gallery, List); enables Relations that link to other databases; supports Rollup properties that aggregate related data automatically; and allows Formula properties that compute values from other fields. The database also supports templates within itself, reminders, and Notion AI integration on any row. If you need to manage and act on data over time, always use a database — not a simple table.

Can I use Notion databases offline?

Notion has limited offline capability. The desktop and mobile apps cache recently viewed pages locally, so you can typically read data from pages you have opened recently even without an internet connection. However, changes made offline are not guaranteed to sync reliably once you reconnect — Notion does not officially support offline editing as a feature, particularly for databases with active collaboration. For reliable offline access to your most important data, export your database as CSV periodically: open the database, click the three-dot menu, select Export, and choose CSV. This gives you a static snapshot you can open in any spreadsheet application. For notes you know you will need offline, consider writing them in a regular Notion page (not a database) which tends to cache more reliably.

How do I duplicate a Notion database template?

To duplicate a database within your own Notion workspace, open the database page, click the three-dot menu (···) in the top-right corner, and select "Duplicate." Notion copies the entire database including all properties, views, filters, and sorts — and optionally all existing rows if you select that option in the duplicate dialog. To import a database template from another person's shared workspace, open their Notion share link in a browser, and click the "Duplicate" button that appears in the top-right corner of the page (visible to all visitors of a shared Notion page). This copies the database structure and any sample data into your workspace. Once duplicated, delete the sample rows, rename properties to fit your workflow, and start adding real data. Duplicating takes about 30 seconds and preserves all the setup work the template creator did.

What are Notion rollup properties?

Rollup properties aggregate data from a related database and display the result directly in the parent database row — automatically, in real time. To add a Rollup, first ensure you have a Relation property set up, then add a new property, choose "Rollup" as the type, select the Relation to aggregate from, choose which property in the related database to compute, and select an aggregation function: Sum (for totaling numbers), Count (for counting rows), Average, Min, Max, Count checked (for checkboxes), Earliest/Latest date, or Show unique values (to display a list of related entries). The most common use cases for freelancers are summing invoice amounts per client, counting total and completed tasks per project, and summing estimated hours across all tasks in a project. Rollup values update automatically whenever related rows change — you never have to manually recalculate totals.

The most important principle for Notion database design: Build one database at a time, prove the habit, then add connections. The biggest Notion mistake is architecting a complex 10-database system before proving you will actually use any of it. Start with a Task Tracker. Use it daily for two weeks. When you genuinely feel the friction of not knowing which client a task belongs to — that is the moment to add the CRM and link them. Build the system to match your actual workflow, not the idealized version you imagine having on day one. Complexity that you grow into is sustainable; complexity you impose upfront gets abandoned.
