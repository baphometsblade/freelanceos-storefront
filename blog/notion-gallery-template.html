Notion Gallery View & Templates: Create Beautiful Visual Databases

How to set up Notion gallery view, the best use cases, customization tips, and 5 gallery template ideas for freelancers — with step-by-step setup instructions.

June 2026  ·  10 min read
Home › Blog › Notion Gallery Template

Most Notion databases default to a spreadsheet-style table view. Table view is efficient for data entry and editing, but it's the worst format for browsing visual collections. If you're managing a portfolio of client work, a catalog of products, a reading list with book covers, or a content library with thumbnail images — gallery view is what you actually want.

Notion's gallery view turns any database into a visual card grid where images take center stage. Each entry becomes a card with a thumbnail, cover image, or color gradient — making it easy to browse collections visually the way you'd scroll through a Pinterest board or a product catalog. It's one of Notion's most aesthetically compelling features, and one of its most underused.

This guide covers exactly how to set up gallery view, the five best use cases for freelancers, customization tips that make the difference between a gallery that looks good and one that looks professional, and five complete gallery template ideas you can build or borrow.

What Is Notion Gallery View?

Gallery view is one of Notion's six database view types (alongside Table, Board, Calendar, Timeline, and List). In gallery view, each database entry is displayed as a card with an optional visual element at the top — either the entry's page cover image, an image from a File & Media property, or a preview of the page content.

How to Set Up Notion Gallery View (Step by Step)

1
Step 1: Create or Open a Notion Database
Gallery view works on any existing Notion database. Open your database or create a new one with /database inline. Add the properties you need: Name (Title), Category (Select or Multi-select), Status (Select), and any other fields relevant to your use case. Gallery view doesn't require any special properties — it works with whatever your database already has. The difference comes from the page covers you add in the next steps.
2
Step 2: Add Gallery View
Click "Add a view" at the top left of your database (the button with a + icon). In the view type menu, select "Gallery." The database immediately switches to card layout — each entry becomes a card, initially without images (just the entry name on a colored card background). You can have multiple views on the same database — keep your Table view for data entry and use Gallery view for visual browsing. Switch between views using the tabs at the top of the database.
3
Step 3: Set the Card Cover Image Source
Click the "..." (three dots) at the top right of the database, then click "Properties." Find the "Card Preview" section at the top. You have three options: None (no thumbnail — just the card title and properties), Page Cover (uses the cover image set at the top of each entry's page), or File & Media property (uses images uploaded to a File & Media property in the database). For most visual databases, "Page Cover" is the cleanest option. For product catalogs where you want to upload specific product images, a File & Media property gives you more control.
4
Step 4: Configure Card Size and Properties
In the Properties settings, set Card Size: Small (4–5 columns, compact), Medium (3 columns, balanced), or Large (2 columns, image-dominant). Also toggle which properties appear on each card. Best practice: show Name (always on), one status/category field, and one key piece of information (date, client name, etc.). Showing more than 3 properties creates visual clutter that defeats the purpose of gallery view. Less is more — the image should be the main content.
5
Step 5: Add Covers to Your Database Entries
Open each database entry (click on its name to open the full page). At the top of the page, click "Add Cover." You can upload an image, use a Notion gradient, or search Unsplash directly from Notion. For a cohesive gallery aesthetic, use images with a consistent visual style — same color temperature, similar composition, or consistent framing. Notion also lets you position the cover image (drag up/down in the page editor) so you control which part of the image shows in the card thumbnail. Once covers are added, your gallery transforms from colored cards to a visual showcase.

Customization Tips for a Professional Gallery

Consistent Cover Image Styling

Nothing undermines a gallery faster than inconsistent thumbnails — some landscape, some portrait, some colorful, some plain white. Choose a visual style and stick to it: all flat color backgrounds with centered icons, all screenshot-style product images, or all photography with the same filter. Notion lets you search Unsplash directly for covers, which is useful for finding images with consistent aesthetics.

Using Gradients When You Don't Have Images

Notion's built-in gradient covers (accessible when adding a page cover → pick from Notion's presets) are underrated. A gallery that uses consistent Notion gradients in your brand colors looks deliberately designed. Pick 3–5 gradient options and rotate them by category — purple for design projects, blue for development, green for strategy work.

Grouping Gallery Cards

Gallery view supports grouping just like Board view. Group by Category to create section headers that organize your gallery into visual sections — "Branding Projects," "Web Design," "Content Strategy." Each section becomes its own row of cards. This is more visual hierarchy with zero extra work.

Sharing Tip

When sharing a Notion gallery publicly (for a portfolio or services page), create a dedicated gallery view with filters showing only the entries you want the public to see. Then share that specific page link. Visitors see only the filtered view — not your full database with all its private properties and unpublished entries. This way, your Notion database is your source of truth and your public portfolio simultaneously.

Frequently Asked Questions

What is Notion gallery view?
Notion gallery view is a visual card layout for database entries. Instead of rows in a table, each entry appears as a visual card — optionally with a cover image, thumbnail, or file preview. Gallery view is ideal for visual content like portfolios, product catalogs, reading lists, recipe databases, and any collection where you want to browse visually rather than scan text rows.
How do I add a gallery view in Notion?
Click 'Add a view' at the top of any Notion database and select 'Gallery'. The database immediately switches to card layout. In the view settings, configure the card cover source (Page Cover, a File property, or no cover), card size (Small/Medium/Large), and which properties appear on the card face.
Can I add images to Notion gallery cards?
Yes, in two ways. First: open any database entry and click 'Add Cover' at the top of the page — this adds a full-width header image that appears as the card thumbnail in gallery view. Second: add a File & Media property to your database, upload images directly, and configure the Gallery card preview to show that property.
What is the best use for Notion gallery view?
Notion gallery view works best for: portfolio databases (client work samples with project thumbnails), product catalogs (e-commerce or digital products with cover images), content databases (blog posts, videos, social content with visual previews), reading lists (book covers), and recipe databases (food photography). Any collection where visual browsing beats text scanning benefits from gallery view.
How do I change card size in Notion gallery view?
Click the '...' (three dots) at the top right of the database in gallery view, or click the view name to open view settings. Look for 'Card Size' and toggle between Small, Medium, and Large. Small shows more cards per row (4–5 on desktop); Large shows 2–3 per row with bigger thumbnails.
Can I filter a Notion gallery view?
Yes. All Notion view types including gallery support filters, sorts, and groups. Click Filter at the top of the gallery view to add filter conditions. Click Sort to reorder cards. Click Group to group cards by a property. Filters and sorts are per-view, so you can have multiple gallery views with different filtered perspectives on the same database.
How do I use gallery view in Notion for a portfolio?
Create a Projects database with Name, Client, Category, Status, Tags, and a File & Media property for project images. Add a gallery view, set card preview to the image property or Page Cover, and card size to Medium or Large. Upload a thumbnail for each project. Filter to Status = 'Complete' to show only finished work. Share as a public Notion link for client proposals.
