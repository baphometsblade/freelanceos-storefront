Notion Toggle: Create Collapsible Sections and Nested Content in Notion

Updated July 2026 · By FreelanceOS Team
Home › Blog › Notion Toggle Guide

Long Notion pages can become overwhelming — walls of text that require endless scrolling to navigate. Toggle blocks solve this by letting you collapse sections of content behind a clickable header, revealing detail only when the reader wants it. The result is a page that is clean and scannable at a glance, with all the detail still available on demand.

This guide covers how to create toggles, use the fastest shortcut, nest any content type inside them, use toggle headings, and the best use cases for toggle blocks in real Notion workspaces.

What Is a Notion Toggle?

A Notion toggle block is a collapsible container. It has a visible header line (the toggle label) with a small triangle arrow (►) to its left. Clicking the arrow expands the toggle to reveal any content nested inside. Clicking it again collapses the content.

The toggle header is always visible. The content inside is visible only when the toggle is open. This is what makes toggles useful for long pages — readers can scan the headers and open only the sections they need.

Toggles are ideal for:

  • FAQ pages where questions are visible but answers are hidden until clicked
  • SOPs and procedures where step details can be collapsed
  • Long meeting notes where agenda items can be collapsed individually
  • Documentation with sections that only some readers need
  • Client proposals with expandable pricing or scope sections
  • Reference lists where you want a clean outline but full detail available

How to Create a Toggle Block

There are two ways to create a toggle in Notion:

Method 1: Slash Command

Type /toggle on any Notion page and press Enter (or click "Toggle list" from the dropdown). A toggle block appears with an empty header. Type your toggle label, then press Enter to move to a new line. Press Tab to indent the new line inside the toggle — any content you add at that indent level is now inside the toggle.

Method 2: The > Shortcut (Fastest)

At the start of any new line, type > (greater-than sign) followed by a Space. Notion immediately converts it into a toggle block. Type your label, press Enter, and Tab to add content inside.

> Space

Instant Toggle Shortcut

Type > then Space at the start of any line. Fastest way to create a toggle — no slash command menu required. Works anywhere on a Notion page.

Adding Content Inside a Toggle

After creating a toggle and typing the label, press Enter then Tab to indent. Everything you type at that indentation level is inside the toggle. Press Shift+Tab or Backspace on an empty line to return to the normal page level outside the toggle.

What Can You Nest Inside a Toggle?

Any Notion block type can be placed inside a toggle:

  • Text paragraphs — the most common use: question as toggle label, detailed answer inside
  • Bullet and numbered lists — step-by-step instructions inside a collapsed section
  • Code blocks — code snippets hidden until needed
  • Images and files — visuals that do not need to be visible by default
  • Callout blocks — formatted callouts with icons inside a toggle
  • Tables — data tables collapsed until relevant
  • Embedded databases — full Notion databases nested inside a toggle (useful but can affect page performance)
  • Other toggles — toggles can be nested inside toggles for unlimited hierarchy depth
  • Dividers, column blocks, and more — any Notion block works
Nested Toggles

You can put a toggle inside a toggle inside a toggle — there is no depth limit. This is useful for deeply hierarchical content like multi-level SOPs or reference libraries where you want 3-4 levels of expandable detail.

Toggle Headings: Collapsible Section Headers

Notion has a special block type called a Toggle Heading that combines a heading and a toggle into one block. The heading text is visible on the page, formatted at H1, H2, or H3 size, and clicking the arrow collapses or expands the entire section of content beneath it.

How to create a Toggle Heading:

  • Type /toggle heading 1 for a large section header toggle
  • Type /toggle heading 2 for a medium section header toggle
  • Type /toggle heading 3 for a smaller subsection header toggle

Toggle headings are the best choice for long documentation pages, wikis, SOPs, and knowledge bases where you want the page structure to be scannable without scrolling through pages of content. The headings provide the outline; readers expand only what they need.

Example uses for Toggle Headings:

  • A company handbook with major sections (Onboarding, Benefits, Policies) as Toggle H1 blocks, subsections as Toggle H2 blocks
  • A client-facing SOP document where each phase of a project is a Toggle H2 that hides the detailed steps
  • A resource library where each category is a Toggle H3 with links inside

Open and Close All Toggles at Once

When you have a page with many toggles, you can open or close all of them simultaneously with a keyboard shortcut:

Alt + Click

Open / Close All Toggles

Hold Alt (Windows/Linux) or Option (Mac) while clicking any toggle arrow. This toggles ALL toggle blocks on the page open or closed at once. Click while everything is closed to open all; click while everything is open to close all.

This shortcut is particularly useful when you are sharing a page with collaborators and want to expand the full content at once, or when you have a reference page you want to see as a clean collapsed outline.

6 Best Use Cases for Notion Toggles

1. FAQ Page

Questions as toggle labels, answers inside. The classic use case. Readers scan the question list and open only what they need. No scrolling past irrelevant answers.

2. SOP with Collapsible Steps

Each major step as a toggle heading, detailed sub-steps inside. Experienced team members can skip familiar steps; new members can expand them. The same document works for both.

3. Long Meeting Notes

Each agenda item as a toggle. The page shows only the agenda overview by default. Click to expand individual agenda items for notes, decisions, and action items.

4. Client Proposal

Project overview visible, pricing and detailed scope inside toggles. Clients can read the summary and drill into specifics at their own pace. Professional and clean.

5. Code Reference Guide

Function names or concepts as toggle labels, code blocks and examples inside. Developers scan the function list and expand only the snippet they need without scrolling past code they know.

6. Daily Notes with Collapsed Logs

Each day's log entry in a toggle within a weekly or monthly page. View the week as a clean list of dates; expand any day to see the full notes for that day.

Limitations of Notion Toggles

What Notion Toggles Cannot Do

  • Performance on very large pages — pages with many deeply nested toggles containing databases can become slow to load. If you notice lag, consider splitting the content across multiple pages linked together.
  • Toggle content hidden in linked database previews — if you link to a page with toggles from a database, the database card preview shows the page title, not the toggle content. Toggle content is only visible when the page is open.
  • No default open/closed state per toggle — toggles always start closed when a page is loaded fresh. You cannot save a "default open" state for specific toggles on a shared page.
  • Notion search works inside toggles — the Notion search function and Ctrl+F page search both find content inside closed toggles, so content is not hidden from search even when collapsed.

Frequently Asked Questions

How do I create a toggle in Notion?
Type /toggle on any Notion page and press Enter, or use the keyboard shortcut: type > (greater-than sign) followed by a Space at the start of a new line. Both methods create an instant collapsible toggle block. Type the toggle label, then press Enter and Tab to add content inside the toggle.
What can I put inside a Notion toggle?
You can put any block type inside a Notion toggle: text paragraphs, bullet lists, numbered lists, code blocks, images, callout blocks, tables, dividers, even embedded databases. You can also nest toggles inside toggles for unlimited depth. The toggle collapses and hides all nested content when closed.
How do I create a toggle heading in Notion?
Type /toggle heading 1, /toggle heading 2, or /toggle heading 3 in Notion to create a heading that also functions as a collapsible toggle. The entire section under the toggle heading collapses when you click it, making it ideal for long wiki pages, documentation, and SOPs where you want to hide detail under section headers.
How do I open or close all toggles on a Notion page at once?
Hold Alt (Windows) or Option (Mac) while clicking any toggle arrow to open or close all toggles on the page simultaneously. This is particularly useful on pages with many toggle sections — use it to expand everything for a full overview, or collapse everything to get a clean outline view of the page.
