Before You Build — Plan on Paper First

The most common Notion setup mistake is opening the app and starting to build before you know what you are building. You end up with ten pages of random notes, three disconnected task lists, and two databases that were almost the same thing. Sound familiar?

Before touching Notion, answer these five questions on paper or in a simple text doc:

  1. Who do I work with? (How many active clients, what information do I need per client?)
  2. What do I deliver? (What are my project types? What are the stages of each project?)
  3. How do I track my work? (Tasks by project? Daily to-do lists? Sprint-based?)
  4. What content do I create? (Blog posts? Social media? Newsletters? Client deliverables?)
  5. How do I manage my money? (Invoices? Income by client? Expenses? Tax tracking?)

Your answers map directly to the five core databases you will build. Do not add complexity before you need it — start with this stack and expand later.

Notion Core Concepts — 5-Minute Primer

If you are new to Notion, here are the six concepts you need to understand before building:

  • Pages: Everything in Notion is a page. Notes, databases, dashboards — all pages. Pages can contain other pages, creating infinite nesting.
  • Databases: Special pages that contain structured entries. Each entry in a database is itself a page with properties (date, status, tags, numbers). Think: a spreadsheet where every row opens into a full document.
  • Views: Different ways to display the same database. One database can be a Table, a Board (Kanban), a Calendar, a Gallery, a List, or a Timeline — all showing the same data, just formatted differently.
  • Relations: Links between databases. A Task can be related to a Project, which is related to a Client. Relations create a web of connected information — the key to a powerful workspace.
  • Rollups: Aggregate data from related database entries. A Client's total contract value can be a rollup that sums all related Project values. A Project's total hours can roll up from related Tasks.
  • Templates: Saved page structures within a database. When you create a new project, a template auto-populates the right sections — brief, deliverables, timeline — so every project starts consistently.

The 5 Essential Freelance Databases

Here is the complete stack — every database you need, the properties to add, and the views to create:

1. Client Database

Your single source of truth for every client relationship — past, present, and prospective.

Name (Title) Status (Select: Active / Lead / Inactive / Past) Industry (Select) Contact Email (Email) Website (URL) Contract Value (Number) Start Date (Date) Related Projects (Relation → Project DB) Total Revenue (Rollup from Finance DB)
Best views: Table (main view with all properties) · Gallery with client logos · Board by Status to see pipeline stages

2. Project Database

Every engagement you take on — from a single-deliverable gig to a six-month retainer — lives here.

Project Name (Title) Client (Relation → Client DB) Status (Select: Proposal / Active / Review / Complete / Archived) Deadline (Date) Project Value (Number) Deliverables (Multi-select) Priority (Select: High / Medium / Low) Related Tasks (Relation → Task DB) Task Count (Rollup from Task DB)
Best views: Board by Status (Kanban) · Timeline view for deadline management · Table with client column · Filtered: Active only

3. Task Database

The daily work layer. Every task tied to a project, with priority, due date, and estimated time.

Task Name (Title) Project (Relation → Project DB) Client (Rollup from Project → Client name) Status (Select: Backlog / Today / In Progress / Done) Priority (Select: 🔴 High / 🟡 Med / 🟢 Low) Due Date (Date) Estimated Hours (Number)
Best views: Board by Status (daily Kanban) · Filtered: Due Today · Filtered: This Week · Table sorted by Priority

4. Content / Ideas Database

Blog posts, social content, newsletters, videos, and content ideas — all in one place with a publication pipeline.

Title (Title) Type (Select: Blog / Email / Social / Script / Video / Newsletter) Platform (Select: Website / LinkedIn / X / YouTube / Email) Status (Select: Idea / Drafting / Review / Scheduled / Published) Publish Date (Date) Target Keyword (Text)
Best views: Calendar view for content schedule · Board by Status for pipeline · Table filtered by Platform

5. Finance Database

Every income and expense transaction tracked, categorized, and ready for tax time.

Description (Title) Type (Select: Income / Expense / Mixed Use) Category (Select: mapped to Schedule C categories) Amount (Number) Date (Date) Client (Relation → Client DB) Tax Deductible (Checkbox) Invoice # (Text) Paid (Checkbox)
Best views: Table (main) · Filtered: Current month · Board by Type (Income vs Expense) · Grouped by Category

Setting Up Relations and Rollups — The Power Move

Relations are what separate a collection of spreadsheets from an actual business operating system. Here is how to connect your five databases into a unified network:

The Relation Chain

Tasks → Projects → Clients. This three-level hierarchy means:

  • When you open a Client page, you see all their Projects (via the Relation) and their total revenue (via Rollup)
  • When you open a Project page, you see all related Tasks (via Relation) and their count / completion status (via Rollup)
  • When you open a Task, you see which Project and Client it belongs to (via Rollup from Project's Client Relation)

Creating Your First Relation (Step by Step)

  1. Open your Project database in table view
  2. Click the + icon at the far right of the property headers to add a new property
  3. Name it "Client" and choose type: Relation
  4. Select your Client database as the target
  5. Enable "Show on Client" to create the reverse relation automatically
  6. Now in your Client database, a Projects property appears showing all linked projects
  7. Add a Rollup property to Client called "Total Project Value" — rollup field: Project Value, calculate: Sum

Useful Rollup Examples

  • On Client database: Roll up "Contract Value" from Projects → Sum = total revenue per client
  • On Project database: Roll up "Status" from Tasks → Count all = total tasks in project
  • On Project database: Roll up "Estimated Hours" from Tasks → Sum = total estimated hours
  • On Finance database: (formula) multiply Amount by Tax Deductible (1/0) to calculate total deductible expenses

Building Your Dashboard Homepage

The dashboard is the most important page in your workspace. It is the page you open every morning, and it should show you everything relevant to today without requiring you to navigate anywhere else.

Creating Your Dashboard

  1. Create a new page: title "🏠 Dashboard" with a rocket or house emoji icon
  2. Add it to Favorites (right-click → Add to Favorites) so it appears at the top of your sidebar
  3. Apply a dark cover image or Notion gradient that sets the visual tone
  4. Add a greeting heading with today's date using the @today mention

Dashboard Sections to Include

Critical Rule: When adding database views to your dashboard, use "Create linked view of database" — NOT a new database. Linked views display data from your existing databases without duplicating anything. Every entry you create in a linked view appears in the original database automatically.
  • Today's Tasks — Linked view of Task DB, filtered to Status = Today or Due Date = Today, sorted by Priority
  • Active Projects — Linked view of Project DB, filtered to Status = Active, gallery or board view
  • This Week's Schedule — Linked calendar view of Task DB filtered to current week
  • Money Snapshot — Two small linked views: income this month (Finance DB filtered by Type=Income and current month) and expenses this month
  • Quick Links Section — Text block with internal @mentions linking to all five databases and important sub-pages

Notion AI — What It Can Actually Do for Freelancers

Notion AI (included with the AI add-on, $10/month, or bundled with Plus in some plans) adds genuine utility for freelancers rather than just being a gimmick. Here is what actually saves time:

  • Meeting note summaries: Paste raw meeting notes → AI generates a structured summary with decisions made and action items. Saves 10-15 minutes per client call.
  • Proposal drafting: Provide a project brief → AI drafts a full proposal outline that you edit. First draft in 90 seconds rather than 30 minutes.
  • Database autofill: AI can fill database properties based on page content — auto-categorize expenses by reading the description, or tag content type based on the body text.
  • Q&A over your pages: Ask "What is the current status of the Johnson project?" and Notion AI searches your workspace to answer.
  • Writing assistance: Improve, shorten, translate, or change tone of any block — useful for polishing client-facing deliverables.

Common Setup Mistakes — And How to Fix Them

Mistake 1: Building disconnected databases

You have a task list, a client list, and a project list — but they are not related to each other. Tasks do not link to projects, projects do not link to clients.

Fix: Add Relations first, then populate data. A workspace without relations is just separate spreadsheets.

Mistake 2: Building the system before testing with real work

Spending days perfecting an empty workspace before using it means you build for imagined needs, not actual ones. Inevitably you rebuild everything after a week of real use.

Fix: Build the minimum viable database, use it for two weeks with real data, then refine based on what is actually missing.

Mistake 3: No templates = inconsistent pages

Every project page looks different. Some have client notes, some have deliverable checklists, some have nothing. You spend time recreating structure every time.

Fix: Create one template per database immediately. Templates take 20 minutes to build and save hours over the life of your workspace.

Mistake 4: Duplicate databases instead of linked views

You create a new "Today's Tasks" database on your dashboard instead of using a linked view of your main Task database. Now you have two separate task lists that are never in sync.

Fix: Always use "Create linked view of database" on your dashboard. One database, multiple views. Never duplicate.

Mistake 5: Never using views

Using only the default Table view for every database. Missing the calendar view for content scheduling, the board view for project status, and the gallery view for visual project browsing.

Fix: Create at least two views per database immediately: a default table and one context-specific view (board for tasks, calendar for content, timeline for projects).

What is the best Notion structure for freelancers?
The five-database stack — Clients, Projects, Tasks, Content/Ideas, and Finance — all linked together with Relations, with a central dashboard homepage showing linked views of each. This structure mirrors how a freelance business actually works: clients have projects, projects have tasks, work generates income. When all five are connected, you get real-time business intelligence from a single workspace: which clients are most profitable, which projects are overdue, which tasks are blocking revenue.
Is Notion free or do I need a paid plan?
Notion's free plan is generous for solo freelancers: unlimited pages and blocks, up to 10 guests, 7-day page history. The Plus plan ($10/month or $8/month billed annually) adds unlimited guests, 30-day version history, synced databases across multiple workspaces, and access to Notion Forms. Most solo freelancers can start on the free plan and upgrade only if they need to share extensively with clients or want longer version history. The Notion AI add-on ($10/month) is separate and optional, but genuinely useful for the summarization and autofill features.
Notion vs Obsidian vs Roam — which should I use?
Each tool wins in different contexts. Notion excels at structured databases, client management, project tracking, and team collaboration — making it the clear choice for freelancers running a business. Obsidian excels at pure note-taking with local file storage (your data never leaves your computer), bidirectional links, and advanced graph visualization of your knowledge — better for researchers and writers who prioritize note-linking over structure. Roam Research excels at daily note-taking and atomic thinking with its outliner format. For the combination of business operations and knowledge management that most freelancers need, Notion wins handily.
How do I migrate from Trello to Notion?
Notion has a built-in Trello import tool. Go to Settings & Members → Import → Trello. You will need to authorize your Trello account, then select which boards to import. Notion converts each Trello board into a Notion database with Board view, cards become database pages, lists become board columns (Status select property), and labels become tags (Multi-select property). Attachments and descriptions import as well. After import, add a Table view to the database and set up any additional properties — due dates, relations to your Client database — that Trello did not track.
Can I use Notion for invoicing?
Partially. Your Notion Finance database can track invoice numbers, amounts, due dates, client relations, and payment status — giving you a complete invoice ledger. But Notion cannot generate professional PDF invoices or send them via email. For that layer, pair Notion with a dedicated invoicing tool: Wave (free), Invoice Ninja (free), FreshBooks ($17/month), or QuickBooks Self-Employed. Your Notion Finance database becomes the master record; the invoicing tool handles the actual document generation and payment collection.
Is Notion good for teams or just solo freelancers?
Both — Notion scales well from solo to team use. Solo freelancers use it as a personal business operating system with no collaboration required. Small agency teams (2–10 people) use it for shared wikis, project management, and client collaboration (you can invite clients as guests to specific pages). Larger teams use Notion's Business plan ($15/user/month) which adds advanced permissions, audit logs, and SAML SSO. The free and Plus plans support guest collaboration, so even solo freelancers can share specific project pages with clients without paying for their access.
