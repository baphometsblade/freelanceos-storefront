Notion Table of Contents: 3 Ways to Create a TOC in Notion (Step-by-Step)

Add a table of contents to any Notion page — the native TOC block, manual page links, and a sticky sidebar setup for long documentation pages.

July 2026  ·  9 min read
Home › Blog › Notion Table of Contents Guide

Long Notion pages can become a nightmare to navigate. You scroll past dozens of sections trying to find that one subsection, or share a page with a client who has no idea how to orient themselves. A table of contents solves this — and Notion has three distinct ways to create one, each suited to different situations.

This guide covers all three methods, how to structure your headings for the cleanest TOC output, what the limitations are, and when to use each approach. Let's start with the quickest option: the native block.

What Is the Notion Table of Contents Block?

Notion introduced the Table of Contents block as a native block type — a first-class citizen in the editor, not a hack or workaround. It works by scanning all heading blocks on the same page (H1, H2, and H3) and generating a clickable list of links that jump to each heading when clicked.

Key characteristics of the native TOC block:

  • Auto-populated: It reads your headings automatically — you don't add items manually.
  • Live-updating: As you add, rename, or delete headings, the TOC updates in real time.
  • Clickable navigation: Each entry is a jump link that scrolls to the corresponding heading.
  • Same-page only: It only reads headings on the page where the TOC block lives — not linked pages or databases embedded on the page.
  • Hierarchical display: H1 headings appear at the top level, H2 indented one level, H3 indented two levels.

The TOC block is most useful at the top of long pages — SOPs, meeting notes, wikis, project briefs — where readers need to jump to a specific section without scrolling through everything.

Method 1 — The Native /Table of Contents Block

The fastest way to add a TOC to any Notion page. Here is the exact process:

  1. Open the Notion page where you want the table of contents.
  2. Click on a blank line at the top of your page content (where you want the TOC to appear).
  3. Type /table of contents and press Enter (or click the Table of Contents option from the command palette).
  4. The TOC block appears immediately and auto-populates with all H1, H2, and H3 headings currently on the page.
  5. If you add new headings to the page, the TOC updates instantly — no refresh needed.
  6. Click any entry in the TOC to jump to that heading on the page.
Pro Tip
Place the TOC block inside a Callout block (type /callout) for visual separation from the rest of your content. Use an icon like 📋 or 🗂️ to make it easy to spot. This is a common pattern in Notion wikis and SOPs.

Where to Place the TOC Block

The most common placements are:

  • Top of the page — below the page title and any intro paragraph. Readers see it immediately and can jump to any section.
  • Left column in a two-column layout — see Method 3 for the sticky sidebar setup.
  • Inside a toggle block — useful if the TOC is long and you want it collapsible. Click to expand when needed, collapsed by default.

Method 2 — Manual TOC with Page Links and Text Anchors

The native TOC block only works within a single page. If you need navigation that links to different pages — such as a wiki homepage linking to section pages — you need a manual TOC using page links.

When to use
Manual TOC: Best for multi-page navigation and custom layouts
Use this approach when you want to link to other pages, when you need more visual control than the native block provides, or when you want a pinned table of contents at the top of a hub page that links out to sub-pages.

How to create a manual TOC in Notion:

  1. At the top of your hub page or section, add a header like "Contents" or "In This Guide."
  2. Create a bulleted or numbered list. Each item will be a link.
  3. To link to another page: type @ and start typing the page name. Select it from the dropdown. This creates a page mention that links to that page.
  4. To link to a specific heading on the current page: Notion does not support anchor links to headings natively via the editor (unlike the TOC block). Use the TOC block for same-page navigation and manual links for cross-page navigation.
  5. Add descriptive text after each link to explain what's in that section.

The trade-off with manual TOCs: they don't auto-update. If you rename a page or restructure your wiki, you need to update the TOC manually. For large wikis, this maintenance overhead adds up. The native TOC block eliminates this for same-page navigation.

Tip
For a Notion wiki homepage, combine both approaches: use page links (manual) to navigate between major sections, and place a native TOC block at the top of each section page to navigate within it. Best of both worlds.

Method 3 — Sticky Sidebar TOC Using Column Layout

This is the most powerful setup for long reference pages and documentation. The idea is to put the TOC in the left column of a two-column layout, with your main content in the right column. As the reader scrolls through the content, the TOC stays visible on the left — functionally similar to the fixed sidebars on documentation sites.

Note: Notion's column layout is not truly CSS-sticky from a technical standpoint — both columns scroll together. But on a wide enough screen with a short TOC, the TOC remains in view much longer than if it were at the top of the page, making it feel semi-sticky and dramatically improving navigation on long pages.

How to Set Up the Two-Column Sticky TOC

  1. On your Notion page, type /2 columns (or drag a block to the right side of another block to create a column layout).
  2. In the left column, type /table of contents and press Enter to add the TOC block.
  3. In the right column, add all your page content — headings, paragraphs, images, toggles, etc.
  4. Make the left column narrower: hover over the divider between columns and drag it to give about 25-30% to the TOC column and 70-75% to the content column.
  5. The TOC in the left column automatically reads all headings in the right column (as long as they're on the same page).
Best use case
Ideal for SOPs, wiki pages, client guides, and project documentation
Any page that is read more than written — where someone needs to find a specific section quickly — benefits from the two-column TOC layout. It's especially effective when sharing pages with clients who are not familiar with the page structure.

How to Structure Headings for a Good TOC

The quality of your table of contents depends entirely on the quality of your heading structure. The TOC block reads H1, H2, and H3 — and displays them in a hierarchy. A flat, inconsistent heading structure produces a flat, confusing TOC. Here's the structure that produces clean, navigable output:

H1 — Major page sections (2-4 per long page)
H2 — Sub-sections within each H1
H3 — Details or steps within each H2
H2 — Another sub-section
H3 — Detail point
H3 — Another detail
H1 — Next major section

Heading Best Practices

  • Use only one H1 as the page title if you want a clear top-level anchor — or use H2s as your primary navigation points on most pages.
  • Keep heading text descriptive — "Setting Up Your Client Portal" is better than "Setup" for TOC readability.
  • Don't skip heading levels — jumping from H1 to H3 without an H2 creates visual gaps in the TOC hierarchy.
  • Avoid overusing H1 — if every section is H1, the hierarchy collapses and the TOC looks flat.
  • Don't use bold text as fake headings — only actual H1/H2/H3 blocks appear in the TOC. Bold text is invisible to the TOC block.

TOC Limitations in Notion

The native TOC block is powerful but has several constraints worth knowing before you rely on it:

Same-page headings only
The TOC block reads headings on the current page only. If you've embedded a linked database or a synced block from another page, headings within those blocks won't appear in the TOC.
No anchor links to headings via @mentions
You can't create a clickable link to a specific heading from another page using Notion's @ link system. The TOC block is the only native way to jump to in-page headings. Third-party tools like Notion Enhancer add anchor links but aren't official.
Toggle headings may not appear
Headings inside toggle blocks are sometimes excluded from the TOC depending on their nesting. For maximum TOC coverage, use standard heading blocks rather than toggle headings.
No custom numbering
Notion's TOC block doesn't add section numbers (1.0, 1.1, 1.2) natively. If you want numbered sections in the TOC, you need to include the numbers in the heading text itself — e.g., "1. Introduction" — which means maintaining numbers manually.
Not truly sticky on mobile
The two-column sticky sidebar approach collapses to a single column on mobile, so the TOC appears above the content rather than beside it. For mobile-heavy audiences, place the TOC at the top of the page rather than in a sidebar column.

Use Cases for a Notion Table of Contents

The TOC block is most valuable on high-information pages where readers need to navigate rather than read linearly. The strongest use cases:

📚
Wiki and Documentation Pages
Team wikis, internal documentation, and knowledge bases benefit most from TOC navigation — readers come looking for specific answers, not to read from top to bottom.
📋
Meeting Notes
Long meetings with multiple agenda items benefit from a TOC at the top — attendees can jump directly to the section with their action items.
⚙️
SOPs and Process Docs
Standard operating procedures often have 10-20 numbered sections. A TOC lets team members jump to the step they need without scrolling through the entire document.
📁
Project Briefs and Proposals
Client-facing project briefs shared as public Notion pages look more professional with a TOC — clients can navigate to pricing, timeline, or deliverables sections directly.

Frequently Asked Questions

Does Notion have a built-in table of contents?
Yes. Notion has a native Table of Contents block you can add to any page by typing /table of contents. It automatically reads all H1, H2, and H3 headings on that page and generates clickable jump links. The TOC updates live as you add or rename headings.
How do I add a table of contents in Notion?
Click anywhere on a Notion page, type /table of contents, and press Enter. The TOC block appears and immediately populates with all headings on the page. Each entry is a clickable link that jumps to that heading. You can move the block to the top of the page or into a sidebar column using a two-column layout.
Does the Notion TOC update automatically?
Yes. The native Table of Contents block in Notion is live — it reflects your current headings in real time. When you add a new heading, rename one, or delete one, the TOC updates instantly without any manual refresh or action on your part.
Can I have a sticky table of contents in Notion?
You can simulate a sticky TOC using Notion's column layout. Create a two-column block, place the TOC block in the left column, and put your page content in the right column. As you scroll through the content, the TOC remains visible on the left. This is the closest Notion gets to a true sticky sidebar TOC, and it works especially well for long wiki pages and SOPs viewed on desktop.
Why is my Notion table of contents empty?
The TOC block only reads headings added using Notion's heading blocks (H1, H2, H3). If your page uses bold text or large text instead of proper heading blocks, the TOC will be empty. Also, toggle headings may not appear in the TOC by default. Convert your text to proper heading blocks using the block type selector (the six-dot handle on the left of each block) to fix an empty TOC.
