Notion Gallery View: The Complete Guide to Gallery Databases in Notion

How to create, configure, and use Notion gallery view for portfolios, catalogs, reading lists, and more — with pro tips for beautiful card layouts.

Notion gallery view turns any database into a visual card grid — the kind of layout you see in Pinterest boards, product catalogs, and portfolio sites. If you have been storing information in Notion tables but find the rows-and-columns format uninspiring or hard to browse visually, gallery view is the answer.

This guide covers everything about Notion gallery view: what it is, how to set it up from scratch, all the configuration options, the six best use cases, and pro tips for making your gallery look polished and professional. What Is Notion Gallery View?

Notion gallery view is one of the six view types available for any Notion database. Instead of displaying database entries as spreadsheet rows (table view) or sticky notes on a board (board view), gallery view renders each entry as a visual card arranged in a responsive grid.

Each card can display a cover image — pulled from a page cover photo, the contents of a Files & Media property, or even a preview of the page content itself. Below the image, cards show whichever database properties you choose to make visible. You can configure card size, image display style, and visible properties independently per gallery view.

Key distinction: Gallery view is a view of a database — not a separate database type. The same data that appears as cards in gallery view can also be seen as spreadsheet rows in table view, columns in board view, or dates on a calendar view. You can add gallery as a second (or third) view to any existing database at any time without changing a single property.

Gallery view is part of Notion's free plan and is available in all workspace tiers. There are no limits on the number of gallery views you can create.

Gallery View vs. Board View: What is the Difference?

Both gallery and board views display cards, but they serve different purposes. Board view (kanban) organizes cards into vertical columns grouped by a select or status property — it is built for workflow tracking, where you drag cards through stages. Gallery view is a flat grid without column grouping (unless you add a group filter) — it is built for browsing and showcase, not for tracking progress.

Use board view when you need to move items through stages. Use gallery view when you need to browse, explore, or visually present a collection.

Creating Your First Gallery Database

Setting up a Notion gallery database takes under five minutes. Here is the step-by-step process.

1
Create a new database Type /database on a blank Notion page and select "Database — Inline" or "Database — Full Page". A new table database appears with three default properties: Name (title), Tags (multi-select), and Created (date). This is your starting data structure.
2
Add the properties you need Click the + button at the far right of the table header to add properties. For a portfolio database, add: Category (select), Client (text), Year (number), Status (status), and Cover Image (Files & Media). The property types you add now will appear on your gallery cards.
3
Switch to Gallery view Click "Add a view" (or the view name at the top left of the database). Select "Gallery" from the list. The database immediately redraws as a card grid. Your entries now appear as cards — plain white cards with names, since you have not added cover images yet.
4
Configure the card preview Click the "..." (three dots) at the top right of the database, then select "Layout". Under "Card Preview," choose your image source: Page Cover, Page Content, or a specific Files & Media property from your database. For a portfolio, select your Cover Image property. For a reading list with imported covers, select Page Cover.
5
Add cover images to your entries Open each database entry (click the entry name or the card). If using page cover: click "Add Cover" at the top of the open page and upload or drag in your image. If using a Files & Media property: scroll to the property field and upload your image directly. Close the entry — your gallery card now shows the image.
6
Choose which properties appear on cards Click "..." → "Properties" in the gallery view. Toggle off properties you do not want visible on card faces. For a portfolio gallery, you might show Category and Year but hide Created By and Last Edited. You can show up to 5–6 properties on a card without it feeling cluttered.

Gallery View Settings Deep Dive

Notion gallery view has several configuration options that significantly change how your gallery looks and behaves. Access all of them via the "..." menu at the top right of the database → "Layout".

Card Size

Notion offers three card sizes:

SizeCards per Row (Desktop)Best For
Small4–5 cardsDense collections where you want to see many items at once — book lists, icon libraries, quick references
Medium3–4 cardsMost use cases — balanced between density and image clarity. Recommended default.
Large2–3 cardsVisual showcases where image quality matters most — portfolio pieces, product catalogs, photography

Card Preview Options

The card preview setting controls what appears in the visual area at the top of each card. Options depend on your database properties:

  • Page Cover: Uses the page's cover image (set inside the page with "Add Cover"). This is the most common approach for portfolios and visual collections.
  • Page Content: Shows a preview of the text content inside the page — useful for writing samples, notes, or content libraries where the text itself matters.
  • Files & Media property: If you have a Files & Media property in your database, you can set it as the preview source. This lets you upload images directly in the database table without opening individual pages.
  • None: No image preview — just the card title and properties. Useful when visual representation adds no value (task lists, contact databases).

Fit Image Setting

When you have a card preview image, Notion lets you choose how it fills the card:

  • Cover (Crop): The image fills the entire card preview area and is cropped to fit. Cards have uniform height and a polished, professional look — but parts of tall or wide images may be cut off.
  • Contain (No Crop): The full image is shown within the card without cropping, with empty space (letterboxing) if the aspect ratio doesn't match. Use this when you need to show the full image, such as product screenshots or logos.
Design Tip

For the most polished gallery, prepare all cover images at the same aspect ratio before uploading — 3:2 (1200×800px) or 16:9 (1280×720px) works best with Notion gallery cards in Cover mode. Inconsistent image sizes create an uneven, amateurish grid even with Notion's cropping.

Showing / Hiding Properties on Cards

In gallery view's "Properties" settings, you control exactly which properties appear as text on each card below the image. Properties are stackable — you can show 2, 4, or 6 fields. The property values appear in small text under the card image. Keep this to 2–3 properties maximum for visual cleanliness; too many properties crowd the card and defeat the purpose of a visual layout.

Filters and Groups in Gallery View

Gallery view supports all standard Notion filter and sort options. Click "Filter" to add conditions (show only Status = Published, for example). Click "Group" to organize cards into labeled rows by a property — grouping by Category creates "Design," "Writing," "Development" row headers with their respective cards beneath.

Customizing Gallery Cards

Cover Images: Page Cover vs. Files Property

You have two methods for adding visual covers to gallery cards. The right choice depends on your workflow.

Page Cover method: Open each database entry → click "Add Cover" → upload or paste an image URL. Pros: clean interface inside the page; supports Unsplash image search built into Notion. Cons: you must open each page to set the cover, which is slow for large databases.

Files & Media property method: Add a Files & Media property to your database. Upload images directly in the database table view without opening entries. Set gallery card preview to this property. Pros: faster data entry for large databases; images are separate from the page content (useful if the page itself has a different header image). Cons: one more property column to manage.

For most personal galleries (under 50 entries), the Page Cover method is simpler and cleaner. For catalogs and databases you update frequently, the Files property method saves significant time.

Setting Cover Images via Image URL

You can set a Notion page cover from any public image URL without uploading a file. Open the page, click "Add Cover," then click "Upload file" dropdown and paste a URL instead. This is useful for pulling in Unsplash images or product images hosted elsewhere. The image renders directly in Notion's gallery card.

# Unsplash URL pattern for free high-quality covers: https://images.unsplash.com/photo-[PHOTO_ID]?w=1200&q=80 # Use Notion's built-in Unsplash picker for consistent sizing: Add Cover → Unsplash → search your term

Grouping Gallery Cards

Gallery view supports grouping by any select, multi-select, status, or checkbox property. Click "Group" at the top of the database, then pick your grouping property. Cards organize into labeled horizontal sections — each section is a collapsible row.

This is particularly useful for: grouping portfolio pieces by service type, grouping a content library by content format, or grouping products by category. Grouping gives you the visual hierarchy of a board view while keeping the aesthetic of gallery layout.

Pro Tips for Beautiful Gallery Databases

  • 1
    Standardize image aspect ratios before uploading. Choose one ratio (3:2 or 16:9) and resize all images before uploading. Mixed portrait and landscape images in a gallery create an uneven grid even with Notion's cropping. Tools like Squoosh, Canva, or Preview (Mac) make batch resizing quick.
  • 2
    Use Unsplash images at 1200×630 for free, consistent covers. Notion's Unsplash picker gives you consistently sized, beautiful images. Searching "gradient purple" or "minimal dark" creates a cohesive visual brand for your gallery without any design work.
  • 3
    Create multiple gallery views with different filters on one database. A single Projects database can have a "Portfolio (Public)" gallery filtered to Status = Featured, a "Full Archive" gallery showing all projects, and an "In Progress" gallery filtered to Active work. One database, three different windows into your data — all in gallery format.
  • 4
    Combine gallery + table views for different team members. Your clients browse your portfolio in gallery view. You manage it in table view for fast data entry. Both views live on the same database. Share the public gallery link with clients and bookmark the table view for your own work — they are looking at the same data through different lenses.
  • 5
    Set a "Featured" checkbox property and filter gallery to it. Add a checkbox property called Featured (or Showcase). Check the boxes for your best 5–8 pieces. Create a gallery view filtered to Featured = Checked. Now you have a curated highlight reel separate from your full archive — switch to the full gallery to add new entries without them appearing publicly until you check the Featured box.
  • 6
    Use the Large card size for public-facing galleries and Small for internal tracking. When sharing a Notion gallery publicly (portfolio, catalog, team directory), Large card size is more impressive and easier to browse on mobile. For internal databases where you update entries frequently, Small or Medium cards let you see more items per screen without scrolling.
Common Mistake

Showing too many properties on gallery cards. Three or more property lines below the image looks cluttered and makes the card feel more like a table row than a visual card. Stick to 1–2 key properties visible on the card face (like Category and Year) and let the full details live inside the card page — opened with one click.

Gallery View for Public Notion Portfolios

Notion's public share feature — enabled via "Share" → "Share to Web" on any page — lets you publish a gallery database as a public URL. For freelancers, this means a no-cost, zero-code portfolio that lives inside your existing Notion workspace.

The workflow: create a Projects database, add a gallery view, upload project covers, filter to your best work, and share the public link. Clients click the link and see a clean, visual portfolio grid. Each card opens to a full case study page with your project write-up, screenshots, and outcomes.

Sharing Tip

When sharing a Notion gallery publicly, share the specific gallery view's page link — not the database parent page. This means visitors land directly in gallery view, not in the default table view. In Notion, click the gallery view tab so it is active, then copy the URL from your browser — that URL loads gallery view by default for anyone who visits it.

The main limitation of Notion as a public portfolio: no custom domain without a third-party tool (Super.so, Potion.so), and limited styling options compared to dedicated portfolio platforms. For freelancers just starting out, a public Notion gallery is a perfectly professional free solution. For established freelancers targeting high-value clients, a custom website with a portfolio section built from a template will outperform on first impression.

Frequently Asked Questions

What is Notion gallery view?
Notion gallery view is a visual card layout for database entries. Instead of rows and columns, each entry appears as a visual card — optionally showing a cover image, page thumbnail, or file preview — with your chosen properties listed underneath. It is ideal for visual collections: portfolios, product catalogs, reading lists, recipe databases, and team directories. Gallery view is available on all Notion plans including the free tier.
How do I add gallery view to a Notion database?
Click "Add a view" at the top of any existing Notion database, then select "Gallery" from the view type list. The database immediately switches to card layout. You can then configure card size (Small, Medium, Large), card preview source (page cover, page content, or a File property), and which properties appear on each card face — all in the "..." → "Layout" and "Properties" settings.
How do I add images to Notion gallery cards?
There are two ways. First, open any database entry and click "Add Cover" at the top of the page — this sets a header image that becomes the gallery card thumbnail when the card preview is set to "Page Cover." Second, add a Files & Media property to your database, upload images directly to each entry in the database, and configure the gallery card preview to display that property. For large databases, the Files property method is faster since you never have to open individual pages to upload images.
What are the best use cases for Notion gallery view?
The six best use cases for Notion gallery view are: portfolio databases with project screenshots, reading lists with book cover images, recipe databases with food photography, product or service catalogs with visual cards, team directories with profile photos, and travel planning boards with destination photos. Any collection where visual browsing beats reading rows of text benefits from gallery view.
How do I change card size in Notion gallery view?
Click the view name or the "..." menu at the top of the gallery database to open view settings. Look for "Card Size" and toggle between Small (4–5 cards per row), Medium (3–4 per row), and Large (2–3 per row with bigger thumbnails). Small is best for dense lists. Large is best for visual showcases where image quality matters more than item density.
Can Notion gallery view be filtered and sorted?
Yes. All Notion database view types — including gallery — support filters, sorts, and groups. Click Filter to add conditions (for example, show only entries where Status equals Published). Click Sort to reorder cards by any property. Click Group to organize cards into labeled sections by a select or status property. All settings are per-view, so different gallery views on the same database can have completely different filters and sorts without affecting each other.
