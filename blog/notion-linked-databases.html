Notion Linked Databases: The Complete Guide to Relations, Rollups, and Linked Views

15 min read • Updated July 2026 • FreelanceOS Team

Relations, rollups, linked views, and synced blocks — the four features that turn Notion from a note-taking tool into a relational database system. Master all of them here.

What Are Notion Linked Databases?

The term "linked databases" in Notion covers two distinct but related features — and it's worth knowing the difference from the start:

  • Relations — a property that connects individual pages in two different databases. A Task can be linked to its Project. A Project can be linked to its Client. This connection lets data flow between databases.
  • Linked Views (Linked Database Views) — a view of an existing database placed on a different Notion page. The data is identical to the source database — you're just showing it in a new context, typically with a filter applied.

Both features are the foundation of sophisticated Notion systems. Without them, Notion is a collection of disconnected lists. With them, Notion becomes a fully relational workspace where everything connects.

Real-world example: A freelancer's Notion system connects three databases: Clients → Projects → Tasks. Each Task links to a Project. Each Project links to a Client. A rollup on Projects counts open tasks. A rollup on Clients sums total project revenue. One workspace, fully connected.

Understanding Database Relations

A Relation property in Notion creates a direct link between pages in two databases. Think of it like a foreign key in a traditional SQL database — except it's visual, clickable, and requires no code.

One-to-Many vs. Many-to-Many

Relation TypeExampleNotion Setting
One-to-manyOne project has many tasks. Each task belongs to one project.Default (no change needed)
Many-to-manyOne task can belong to multiple projects. One project has many tasks.Also default in Notion — just link multiple items
One-to-oneEach invoice belongs to exactly one project.Convention only — Notion doesn't enforce cardinality

Notion creates bidirectional relations by default. When you add a Relation property to Database A pointing to Database B, Notion automatically adds a matching relation property on Database B pointing back to Database A. You can rename or hide either side, but both are always present.

Step-by-Step: Build Your First Notion Relation

Step 1 — Create a Tasks database

In your Notion workspace, create a new page and add a database (full page or inline). Name it Tasks. Add properties: Name (title), Status (Select), Due Date (Date).

Step 2 — Create a Projects database

Create another database on a different page. Name it Projects. Add properties: Project Name (title), Client (text), Status (Select), Start Date (Date).

Step 3 — Add a Relation property to Tasks

In your Tasks database, click + Add a property. Select Relation as the property type. A dropdown appears asking which database to link to — select Projects. Click Add relation.

Step 4 — Add a task and link it to a project

Create a new task row. Click the Project relation field. A search popup appears — type the name of a project and select it. The task is now linked to that project.

Step 5 — See the link on both sides

Open your Projects database. Notice Notion has added a Tasks relation property automatically. Click on a project row — it shows all tasks linked to that project. The connection is bidirectional.

Rollup Properties Explained

A Rollup property reads data from related pages and calculates a value. It's how you answer questions like "how many tasks does this project have?" or "what's the total revenue from this client?" — all automatically, always up to date.

Setting Up a Rollup

  1. In your database, click + Add a property
  2. Choose Rollup as the property type
  3. In the rollup settings, select the Relation property to pull data from (e.g., "Tasks")
  4. Select the property from the related database to roll up (e.g., "Status")
  5. Select the calculation function:

Linked Views vs. Filtered Views

This is one of the most misunderstood distinctions in Notion. Both involve showing a subset of database data — but they work differently.

Filtered Views (in the Source Database)

A filter applied to a view within the original database page. Shows a subset of the same data. Both the filter and the data live in the same place.

Linked Database Views (on a Different Page)

A copy of the database view placed on a different Notion page. The underlying data is the same database — no duplication — but the view appears in a new context with its own filters and layout.

The killer use case: Open a Project page. Add a linked view of your Tasks database. Set a filter: "Project equals this page." Now every Project page automatically shows only its own tasks — no manual sorting, no data entry. This is the backbone of professional Notion project management.

How to Add a Linked Database View

  1. Navigate to the page where you want the linked view (e.g., a specific Project page)
  2. Type /create linked view of database or /linked
  3. Search for and select your Tasks database
  4. The database appears on the page — now add a filter: "Project → is → [this page]"
  5. The view now shows only tasks linked to this specific project

Synced Blocks

Synced blocks are different from linked databases — they're for content, not data. A synced block lets you write a block of content once and have it appear identically on multiple pages. Edit one copy and all copies update automatically.

When to Use Synced Blocks

  • Navigation headers that appear on multiple pages in a Notion section
  • Legal disclaimers or standard terms that need to be identical everywhere
  • Recurring weekly check-in prompts shared across team member pages
  • Product descriptions shared between a sales page and a project page

How to Create a Synced Block

  1. Write any block (or group of blocks) on a Notion page
  2. Click the six-dot handle on the left of the block
  3. Select Turn into synced block
  4. The block turns red — this is the "source" synced block
  5. Copy it (Cmd+C) and paste on another page — it appears with a red border, synced to the original

Frequently Asked Questions

What is a Notion linked database?
A Notion linked database can mean two things: (1) a Linked View — a view of an existing database placed on a different Notion page, showing the same data without duplication; or (2) a Relation — a property that connects two separate databases so items in one can reference items in the other.
How do I create a relation in Notion?
Open a database, click the + button to add a new property, select Relation as the property type, then choose the database you want to link to. Notion creates a bidirectional relation automatically — the linked database gets a matching relation property pointing back to yours. You can then click any relation field to search and select related pages.
What is a rollup in Notion?
A Rollup is a property type that calculates a value based on items in a related database. For example, a Projects database can have a rollup that counts the number of related Tasks, or a Clients database can roll up the sum of all project invoices. Rollups update automatically whenever the related data changes.
Can I link two Notion databases?
Yes — use a Relation property to link two Notion databases. Once linked, items in each database can reference items in the other. You can then add Rollup properties to calculate aggregated values from the related database, like counts, sums, averages, or earliest/latest dates.
What is the difference between a synced block and a linked database?
A synced block duplicates a block's content across multiple pages — edits to one copy automatically update all copies. It's for static content like headers and disclaimers. A linked database view is a filtered view of a database placed on another page — it shows dynamic data that updates as the database changes. They solve completely different problems.
How do I show related items on a Notion page?
Add a linked database view to the page by typing /linked or /create linked view of database. Select your database, then set a filter condition like "Project equals this page." The linked view will then show only items related to that specific project page — the standard pattern for project management in Notion.
