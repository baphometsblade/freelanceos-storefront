Notion Checklist: 4 Ways to Create Checklists and To-Do Lists in Notion

8 min read • Updated July 2026

Home › Blog › Notion Checklist Guide 2026

Notion gives you multiple ways to create checklists and to-do lists — from a simple inline checkbox to a full database-powered task management system. Which method you use depends on what you're trying to track. A quick shopping list is different from a recurring project checklist, which is different from a team-wide task database.

This guide walks through all four methods, when to use each, and how to build a complete weekly task system that uses Notion's checkboxes to their full potential.

The 4 Ways to Create Checklists in Notion

Method 1
The /To-do Block — Inline Checklists

The fastest way to create a checklist in Notion. Type [] (two square brackets, no space) at the start of any line to instantly create a to-do checkbox item. Alternatively, type /to-do in the slash command menu and select "To-do list."

How it works: each to-do block is an independent checkbox item on your page. Check a box and the text is automatically struck through and grayed — a satisfying visual indicator of completion. You can create as many consecutive to-do items as you need, and they behave like a simple bullet list with checkboxes.

Best for: ad-hoc task lists, meeting action items, quick personal checklists, grocery lists, and any one-off list where you don't need filtering or database features.

Limitation: to-do blocks are page-bound. They don't integrate with database views, can't be filtered across pages, and don't connect to dates, assignees, or properties. They're checklists, not task management.

Method 2
Checkbox Property in a Database — The Power Move

The most powerful way to use checkboxes in Notion. Add a "Checkbox" type property to any database — a Tasks database, a Project tracker, a Client Onboarding database — and each record gets a true/false toggle.

How to add it: open any database in Notion → click the + icon in the property header row → select Checkbox → name it "Done," "Complete," "Shipped," or whatever fits your workflow.

The real power: filter your database by Checkbox = Unchecked to see only incomplete tasks. Save this as a view named "Active Tasks." Check a box and the item disappears from your active view automatically. This is living, auto-updating task management — not a static list.

Combine with other filters: Due Date = this week AND Done = unchecked creates your weekly to-do view. Assignee = Me AND Done = unchecked creates your personal task queue from a shared team database.

Best for: recurring task systems, project management, client deliverables, team task tracking, and any work where you need to filter, sort, or query your tasks.

Method 3
Toggle Blocks With Nested To-Dos — Collapsible Checklist Sections

Use a Toggle block as a collapsible section header, then nest /to-do items inside it. This creates a structured checklist with expandable sections — ideal for SOPs, onboarding guides, or any multi-section process documentation.

How to create: type /toggle to insert a Toggle block. Name it your section (e.g., "Pre-Launch Checklist"). Click the triangle to expand it. Inside the expanded toggle, type [] to add to-do items. Each item is indented under the toggle.

Practical example: a "Client Project Kickoff" toggle might contain 12 steps: contract signed, deposit received, brand assets received, Notion workspace created, first call scheduled, etc. Collapse the toggle when the section is complete for a clean overview of your checklist's status.

Best for: SOPs and standard operating procedures, onboarding checklists, multi-step guides, and any checklist that benefits from collapsible sections.

Method 4
Template Buttons — Reusable Checklists on Demand

Template Buttons let you create a pre-built checklist that regenerates fresh on demand with a single click. This is the solution for recurring checklists: weekly reviews, client onboarding sequences, project kickoff checklists, content publishing workflows.

How to create one: on any Notion page, type /template to insert a Template Button. Click "Edit" to configure it. Give the button a name (e.g., "Start Weekly Review"). Inside the template area, add all your /to-do items with every step pre-populated. Close the editor. Now each time you click the button, Notion creates a fresh, fully checked-out version of your checklist below — ready to work through.

Alternative: create a database Template (not a button) with a pre-filled page template that includes a checklist. Every new record added to the database gets the checklist pre-populated. Ideal for client onboarding records, project pages, or weekly sprint entries.

Best for: weekly/monthly review checklists, client onboarding sequences, content publishing workflows, launch checklists, and any process you run on a regular schedule.

Keyboard Shortcuts for Notion Checklists

Speed up your checklist creation with these shortcuts:

  • [] — creates a to-do block instantly (type at the start of a blank line)
  • /todo — slash command to insert a to-do list block
  • Enter — after checking/writing a to-do item, creates the next to-do item automatically
  • Tab — indents a to-do item (creates a sub-item, useful for hierarchical checklists)
  • Shift + Tab — un-indents (moves an item back to the parent level)
  • Cmd/Ctrl + / — opens the block type menu to convert any text block to a to-do block
Pro Tip

If you've already typed a list of tasks without checkboxes, you can convert them all at once: select all the lines, press Cmd/Ctrl + / and select "To-do list." All selected lines become checkbox items instantly.

Formatting and Customizing To-Do Blocks

Notion's to-do blocks are plain by default, but you can add visual hierarchy to make priority clear at a glance:

Color Coding for Priority

Click any to-do item's text → use the block menu (Cmd/Ctrl + /) → select a text color or background color. A common system: red background for urgent tasks, yellow for medium priority, green for nice-to-have. The color persists even after checking the box.

Emoji Priority Labels

Add emoji at the start of to-do item text for instant visual priority scanning:

  • 🔴 — Critical, must complete today
  • 🟡 — Medium priority, complete this week
  • 🟢 — Low priority, when time allows
  • ⏳ — Waiting on someone else before I can proceed
  • 📅 — Has a specific deadline

Bold Text for Task Names, Normal for Context

Use bold for the core task name and regular text for context notes on the same line: Send proposal to Sarah — waiting for Zoom recording to attach. This keeps your checklist scannable without losing important context.

Building a Weekly Task Checklist With Database + Filtered Views

This is the real Notion task management workflow — the one that replaces apps like Todoist or TickTick for most users.

1

Create a Tasks Database

Add a new full-page database. Name it "Tasks" or "My Tasks." Add properties: Task Name (title), Done (Checkbox), Due Date (Date), Priority (Select: High/Medium/Low), Project (Relation to your Projects database — optional).

2

Create a "This Week" Filtered View

Add a new List view. Click Filter → Add Filter → Due Date → "is within" → "the next 7 days." Add another Filter → Done → "is unchecked." This view shows only tasks due this week that aren't complete. As you check tasks, they disappear. New tasks added with this week's due date appear automatically.

3

Create a "Today" View

Duplicate the view. Change the filter to Due Date = "today" AND Done = unchecked. This becomes your daily dashboard. Each morning, open this view and work through the list. At end of day, anything unchecked gets rescheduled to tomorrow.

4

Create a "Completed This Week" Archive View

Add a view filtered by Done = checked AND Due Date = the past 7 days. This is your weekly accomplishment log — motivating to review on Fridays and useful for client status reports and performance reviews.

Checklist vs. Database Tasks: When to Use Which

One of the most common questions for new Notion users is when to use a simple to-do block versus a proper database with a Checkbox property. Here is the decision framework:

Use to-do blocks (Method 1 or 3) when:

  • The list is one-off and won't need to be tracked after completion
  • You don't need to filter, sort, or query the items
  • The list is 5-15 items and doesn't need dates or assignees
  • You're capturing quick action items from a meeting or brainstorm
  • The checklist lives inside a specific document for context

Use a Checkbox property in a database (Method 2) when:

  • Tasks are recurring or ongoing — not one-off
  • You need to filter to see only incomplete items
  • Tasks have due dates, priorities, or assignees
  • You want a "Today" or "This Week" view that auto-updates
  • You're managing tasks across multiple projects or clients
  • You want to see completion trends over time
The Right Mental Model

To-do blocks are for notes and thinking. Database checkboxes are for managing work. If you're writing a page about a project and noting next steps inline, use to-do blocks. If you're managing the actual tasks of a project across time and team members, use a database.

Frequently Asked Questions

How do I create a checklist in Notion?
The fastest method: type [] (two square brackets) at the start of a line to create a checkbox to-do item instantly. You can also type /todo from the slash command menu. For database-powered checklists, add a Checkbox property to any database and filter by "Checkbox is unchecked" to see only incomplete tasks.
What is the keyboard shortcut for a to-do in Notion?
Type [] (two square brackets with no space) at the beginning of a line to instantly create a to-do checkbox. You can also type /todo in the slash command menu. To convert existing text to a to-do block, select it and press Cmd/Ctrl + / to open the block menu and choose "To-do list."
How do I create a reusable checklist in Notion?
Use Notion's Template Button block. Type /template to insert one, name it (e.g., "Start Weekly Review"), then add your checklist items inside the template configuration. Each time you click the button, Notion creates a fresh copy of your full checklist. Perfect for recurring processes like weekly reviews, client onboarding, content publishing, or project kickoffs.
Can I filter a database by checkbox in Notion?
Yes. Add a Checkbox property to any database, then click Filter → Add a filter → select the Checkbox property → choose "Is unchecked" (for incomplete tasks only) or "Is checked" (for completed tasks). Combine with other filters — like Due Date = this week AND Done = unchecked — to build a live weekly task view that auto-updates as you check items off.
What's the difference between a to-do block and a checkbox property in Notion?
A to-do block is a simple inline checklist item on a page — great for ad-hoc lists. It can't be filtered, sorted, or linked to other data. A checkbox property is a property on a database record — it can be filtered, sorted, and combined with dates, assignees, and other properties to build a full task management system. Use to-do blocks for quick notes; use a checkbox database property for managing ongoing work.
