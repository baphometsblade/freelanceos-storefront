Notion Recurring Tasks: The Best Methods to Automate Repeating Tasks in Your Workspace

Notion doesn't have native recurring tasks — but with the right setup, you can automate your repeating task system better than most dedicated apps.

10 min read  ·  July 2026  ·  FreelanceOS Team
Home›Blog›Notion Recurring Tasks

If you've ever searched "how do I make a recurring task in Notion," you've hit the same wall as thousands of other Notion users. Notion doesn't have a native "repeat every X days" feature the way Todoist, TickTick, or Apple Reminders does. You can't just check a box that says "recur weekly." The feature simply doesn't exist — not as a native, automatic function.

But Notion's flexibility means you can build a recurring task system that's more customized and more powerful than any app with a built-in repeat toggle. This guide covers the four best methods, from simple (takes 5 minutes to set up) to powerful (takes 20 minutes, works automatically). By the end, you'll know exactly which method fits your needs.

Quick answer: For simple recurring tasks you'll manually trigger, use a Template Button. For tasks that must recur on a calendar schedule without your involvement, use Zapier or Make. For a middle ground with Notion automations, use the "On Done → Create New" automation. For a calculated next-due date system, use the Formula method.

Why Notion Doesn't Have Native Recurring Tasks

Notion is a document and database tool, not primarily a task manager. Its databases are flexible containers for any type of structured data — but that flexibility means it doesn't make opinionated decisions like "this task should repeat every Tuesday." Apps like Todoist are purpose-built for task management and bake in scheduling features at their core.

Notion has added automations in recent years, which can approximate recurring tasks — but the automation system doesn't include a scheduled trigger (e.g., "fire at 8am every Monday"). It can trigger on property changes and time-based conditions on existing pages, but not on a blank-slate calendar schedule.

The good news: this is a well-understood limitation, and the community has developed solid workarounds for every level of complexity.

Method 1 — The Template Button (Simple, Manual)

This is the easiest method and works well for recurring tasks that follow a consistent pattern. You create a database template pre-filled with the right properties, then press a button to instantly create a new copy of that task. One click, one new task, all properties set.

1

Set Up Your Tasks Database

Create a database with these properties: Task Name (Title), Status (Select: To Do / In Progress / Done), Due Date (Date), Priority (Select), Repeat Frequency (Select: Daily / Weekly / Monthly). This last property is just for your reference — it doesn't auto-trigger anything.

2

Create a Template for Each Recurring Task

Open your database. Click the dropdown arrow next to the blue "New" button. Click "Edit templates." Create a new template for each recurring task (e.g., "Weekly Client Update Email"). Pre-fill all the properties except due date — you'll set that when you create each instance.

3

Use the Template Button Weekly

Each Monday (or whatever your cadence is), open your tasks database and use the template dropdown to create instances of each recurring task for that week. Set the due date. This takes about 3 minutes and gives you a weekly planning ritual that doubles as your "start the week" moment.

Best for: Freelancers with 3–8 weekly recurring tasks who don't mind a 3-minute Monday setup ritual. This is the method most Notion power users actually use — it's fast, reliable, and zero complexity.

Method 2 — Notion Automations (On Completion → New Page)

Notion's built-in automation engine lets you trigger actions when database properties change. You can set up an automation that fires when a task's Status changes to "Done" — and automatically creates a new instance of that task with a due date set X days in the future. This is the closest Notion gets to native recurrence without third-party tools.

1

Open Automations on Your Tasks Database

In your database, click the three-dot menu (or the lightning bolt icon) and select "Automations." Click "New automation."

2

Set the Trigger

Trigger: "When a page property is edited." Property: Status. Condition: Status equals "Done." Add a filter: Repeat Frequency is not empty (so only recurring tasks trigger this, not regular one-off tasks).

3

Set the Action — Add Page to Database

Action: "Add page to [Tasks database]." Set the new page's properties: copy the Task Name from the triggering page, set Status to "To Do," set Due Date to "Now + 7 days" (or 1 day / 30 days for daily/monthly tasks). You can reference the original page's Repeat Frequency to make the interval dynamic.

4

Test the Automation

Mark a recurring task as Done. Within a few seconds, a new instance should appear in your database with Status = To Do and the due date set forward. Check the new page's properties — make sure all fields carried over correctly.

Limitations of This Method

  • The new task is created when you mark the old one Done — if you complete a task late, the new instance's due date is relative to the completion date, not the original schedule
  • Notion automations don't trigger on a calendar schedule (every Monday at 9am) — they only trigger on property changes
  • Complex properties (relations, rollups) may not copy cleanly — test thoroughly before relying on this for critical tasks

Method 3 — Formula-Calculated Next Due Date

This method doesn't create new task pages automatically — instead, it keeps a single "master" entry for each recurring task and uses a formula to calculate when it should next be done based on when you last completed it. It's elegant and minimalist: one row per recurring task, a clean "Next Due" column that updates automatically.

Database Setup

  • Last Completed (Date property) — you update this each time you finish the task
  • Interval Days (Number property) — how many days between repetitions (7 for weekly, 30 for monthly, 1 for daily)
  • Next Due (Formula property) — calculated automatically

The formula for Next Due:

dateAdd(prop("Last Completed"), prop("Interval Days"), "days")

You can also add an "Overdue" formula to flag tasks that are past their due date:

if(now() > prop("Next Due"), "⚠️ Overdue", "✅ On Track")

To use this system: when you complete a recurring task, update the Last Completed date to today. The Next Due formula instantly recalculates. Filter your view to show tasks where Next Due is before or equal to today — these are the tasks that need doing now. This single view replaces a traditional recurring task list.

Best for: Minimalists who want one record per recurring task (no clutter of past instances). Great for maintenance tasks like "back up files," "send monthly invoice," or "review client list" — things with a fixed interval where you want a running log of completion dates.

Method 4 — Zapier or Make (Fully Automatic, Scheduled)

For true recurring tasks that appear in your Notion database on a schedule — regardless of what you do — Zapier or Make is the solution. You set a schedule trigger ("every Monday at 8am") and an action ("create a page in [Tasks database]"). Notion never needs to know about the recurrence — Zapier handles the clock.

Zapier Setup (Quick Version)

  1. Trigger: Zapier "Schedule by Zapier" — set your interval (daily, weekly, monthly, custom cron)
  2. Action: Notion → "Create Database Item" — connect your workspace and select your Tasks database
  3. Map fields: Set Task Name (static text like "Weekly Invoice Check"), Status = To Do, Due Date = Zap trigger time
  4. Turn on the Zap. Your recurring task now appears automatically on schedule, every time, without touching Notion.

Which Method Should You Use?

The answer depends on two questions: how many recurring tasks do you have, and how bad would it be to miss one?

  • 5 or fewer recurring tasks, nothing critical: Template Button. Fastest setup, easiest to maintain, and the weekly creation ritual is actually useful for planning.
  • Tasks where timing is relative to completion (physical maintenance, health habits): Formula Next Due Date. One record, auto-updating, clean.
  • Tasks that must appear on a schedule without your intervention: Zapier or Make. If forgetting to create the task has real consequences (invoice reminder, client check-in, compliance task), automate it fully.
  • Middle ground: Notion Automations + On Done trigger. Works well if you're disciplined about marking tasks Done and don't mind the due date being relative to completion rather than the original schedule.

Frequently Asked Questions

Does Notion have recurring tasks?
Notion does not have a native recurring task feature that automatically creates repeating tasks like Todoist or TickTick. However, you can replicate recurring tasks using Notion's template buttons (to manually clone a task with one click), Notion automations (to create a new page when a condition is met), formula-calculated next-due dates, or third-party integrations like Zapier or Make to create Notion entries on a schedule.
How do I create a repeating task in Notion?
The easiest method is a template button. Create your recurring task as a database entry with all the right properties filled in. Then create a button (using the button block or the database template feature) that duplicates this task with a single click. Each Monday morning, press the button to create that week's recurring tasks. For fully automatic recurring tasks, use Zapier or Make to create a Notion database entry on a schedule.
Can Notion automations create recurring tasks?
Notion automations can help with recurring tasks but have limitations. You can create an automation that triggers when a task is marked "Done" and creates a new page — effectively cloning the task for next time. However, Notion automations don't natively trigger on a calendar schedule (e.g., every Monday). For true scheduled recurrence, Zapier or Make is the most reliable solution.
What is the best way to handle recurring tasks in Notion?
For simple recurring tasks you'll manually trigger, use a template button. For tasks that need a calculated next due date based on completion, use the Last Completed + formula method. For tasks that must recur on a schedule regardless of whether you complete them, use Zapier or Make to create Notion entries on a cron schedule. For complex workflows, combine a template button with Notion automations to auto-assign dates and properties.
Is Notion good for task management?
Notion is excellent for task management when you need deep customization, project linking, and a workspace that integrates your tasks with your documents, wikis, and databases. It is less ideal if you need a fast, simple to-do list with native reminders and recurring tasks out of the box. Apps like Todoist or TickTick are simpler for pure task management. Notion shines when your task system is part of a larger personal OS.
