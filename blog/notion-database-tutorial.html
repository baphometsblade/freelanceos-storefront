Notion Database Tutorial: The Complete Guide for Beginners (2026)

Everything you need to know about Notion databases — views, properties, filters, relations, rollups, and the best setups for real work.

Notion databases are where Notion stops being a note-taking app and starts being a proper operating system for your work. They are the feature that separates Notion power users from people who use it as a fancy document editor. Once you understand how databases work, you can build a CRM, a project tracker, a content calendar, and an invoice system — all in the same workspace, all linked together.

This tutorial covers everything from the basics (what is a Notion database?) to intermediate features (filters, sorts, linked databases) to the most useful database setups for freelancers and knowledge workers. By the end, you will have everything you need to build your first useful database from scratch.

What Is a Notion Database (vs a Regular Page)?

A regular Notion page is like a document — free-form text, images, headers, lists. A Notion database is a structured collection of pages where every page shares the same set of properties (columns). Each row in the database is a full Notion page with its own content, plus shared structured fields.

Think of it like a spreadsheet where every cell in the first column links to a full document. You get the structure of a spreadsheet — sortable, filterable, groupable data — combined with the flexibility of rich text pages inside each row.

The key difference is that a database can be viewed in multiple ways. The same data that looks like a spreadsheet in Table view becomes a visual gallery in Gallery view, a drag-and-drop pipeline in Board (kanban) view, and a monthly planner in Calendar view — all without touching the underlying data.

The 6 Notion Database View Types

Every Notion database supports six view types. You can add as many views as you want to a single database — switching between them is instant and free.

▢
Table View
Best for: data entry, sorting, reviewing all fields at once
The classic spreadsheet view. Every row is a database entry and every column is a property. Best for entering data quickly and scanning across all fields. Add, hide, resize, and reorder columns with a click. This is the default view when you create a new database.
▦
Gallery View
Best for: visual content, portfolio items, inspiration boards
Shows each database entry as a card with a large cover image. Perfect for visual content — design assets, product photos, client mood boards, or any database where the visual identity of each entry matters. You can choose which property appears as the card subtitle.
▮
Board (Kanban) View
Best for: project management, workflow tracking, status pipelines
Groups database entries into columns based on a Select property — typically a Status field. Drag cards between columns to update their status. Perfect for project management, content pipelines, client CRMs, and any workflow with distinct stages. This is the most-used view by freelancers.
📅
Calendar View
Best for: content calendars, deadline tracking, event planning
Displays entries on a monthly calendar based on a Date property. Drag entries between dates to reschedule them. Multiple entries on the same date stack and are individually clickable. Ideal for content calendars, project deadline views, and any time-sensitive database.
▬
Timeline View
Best for: project planning, multi-phase work, Gantt-style views
Shows entries as horizontal bars across a timeline, spanning from a start date to an end date. This is Notion's Gantt chart equivalent. Perfect for project planning with overlapping phases, multi-deliverable client engagements, or quarterly content campaigns with defined windows.
☰
List View
Best for: reading lists, reference databases, compact task views
A compact, single-column view showing each entry as a line item with its title and one or two selected properties. Loads fast and takes up minimal space. Best for reference databases you need to scan quickly — a reading list, a contacts directory, a resource library.

Notion Database Properties Explained

Properties are the columns in your database — the structured fields that every entry shares. Choosing the right property types is the most important decision when designing a Notion database.

Text
Free-form text. Use for notes, descriptions, addresses, or any unstructured value.
Number
Numeric values. Supports formatting as currency, percentage, or plain number. Used for prices, hours, quantities.
Select
A single-choice dropdown with colour-coded options. Perfect for Status, Priority, Category, or Platform fields.
Multi-select
Multiple tags from a predefined list. Ideal for Skills, Topics, or any field where an entry can belong to multiple categories.
Date
A date picker with optional time and end date (for date ranges). Powers Calendar and Timeline views.
Person
Assign a Notion workspace member to a database entry. Essential for team task trackers and collaborative databases.
Checkbox
A true/false toggle. Use for "Done," "Invoiced," "Reviewed" flags. Can be filtered to show only unchecked items.
URL
A clickable web link. Use for portfolio links, reference URLs, client websites, or source documentation.
Formula
Calculated values based on other properties. Can compute days until a deadline, concatenate strings, or calculate total value.
Relation
Links entries to another database. Connects Projects to Tasks, Clients to Invoices, or Articles to Topics.
Rollup
Aggregates data from a related database — sum, count, average, min, max. Used to total invoice amounts per client.
Files & Media
Upload or link files directly on a database entry. Attach contracts, design assets, or reference images to project rows.

How to Create Your First Notion Database

  1. Create a new page. In your Notion sidebar, click the + button to add a new page. Give it a descriptive name like "Client Projects" or "Content Calendar."
  2. Add a database block. On the new page, type /database and select "Table — Full page" for a standalone database, or "Table — Inline" to embed the database within a page alongside other content.
  3. Rename the default Name column. The title property (always the first column, displayed in bold) will be the main identifier. Rename it to match your content: "Project Name," "Client," "Article Title," etc.
  4. Add your properties. Click the + button at the right end of the column headers to add a new property. Choose the property type that matches your data. Start with 4 to 6 essential properties — you can always add more later.
  5. Add your first entries. Click "+ New" at the bottom of the table to add rows. Fill in each property. Click a row title to open the full page and add notes, tasks, or any rich content inside that entry.
  6. Add a second view. Click "+ Add a view" at the top left. Add a Board view grouped by your Status property. Now your database has both a spreadsheet view and a kanban pipeline view — same data, two perspectives.

Filters and Sorts

Filters and sorts are how you slice a Notion database to show exactly the data you need. They are especially powerful in multi-purpose databases where a single database serves several different use cases.

Filters

Add a filter to a view to show only entries that match specific criteria. Click "Filter" in the database toolbar and add a condition: Status = "In Progress," Due Date = "This week," Platform = "LinkedIn." You can stack multiple filters with AND/OR logic. Saved filters stay with the view — so your "Active Projects" view and your "Completed Projects" view are actually the same database filtered differently.

Sorts

Sorts reorder entries based on a property value. Sort by Due Date ascending to see what is due soonest. Sort by Revenue descending to see your highest-value projects first. You can stack multiple sorts — first by Priority (High to Low), then by Due Date within each priority group.

Linked Databases: One Database, Multiple Places

A linked database is a view of an existing database placed on a different Notion page. It shows the same data as the original database, with its own independent filters, sorts, and views. Changes made in a linked database update the original instantly.

This is one of Notion's most powerful features. You can have a master Tasks database containing every task across all projects, then embed a filtered linked database view on each individual Project page showing only that project's tasks. Same data, contextualised in the right place.

To create a linked database, type /create linked view of database on any Notion page and select the source database. Add filters to narrow the view to the relevant subset of data.

Notion Database Formula Basics

Formulas let you create calculated properties based on other property values. You do not need to be a programmer to use them — a handful of functions covers 90% of use cases.

// Days until deadline
dateBetween(prop("Due Date"), now(), "days")
// Is this overdue? (returns true/false)
prop("Due Date") < now() and not(prop("Done"))
// Combine first and last name
prop("First Name") + " " + prop("Last Name")
// Calculate hourly rate from project value and hours
prop("Project Value") / prop("Hours Logged")

Formulas update in real-time. A "Days Until Deadline" formula shows a different number every day without any manual input. This makes your Notion databases genuinely dynamic rather than static tracking documents.

Frequently Asked Questions

What is a Notion database?
A Notion database is a structured collection of pages where every page shares the same set of properties (columns). Unlike a regular Notion page, a database page can have dates, tags, checkboxes, relations to other databases, and calculated fields. Every database can be viewed as a table, gallery, kanban board, calendar, timeline, or list without changing the underlying data.
What is the difference between a Notion table and a database?
In Notion, a table view IS a database. When you create a Notion table, you are creating a database with the table view selected by default. The database is the underlying data structure; the table is just one way to look at it. You can add a calendar view, kanban view, or gallery view to the exact same database at any time.
What are Notion database relations and rollups?
A Relation property links two Notion databases together, so a row in one database can reference a row in another. For example, a Projects database and a Tasks database can be linked so each task knows which project it belongs to. A Rollup property then lets you calculate a value from the related database — such as counting the number of tasks per project, or summing the revenue from all invoices linked to a client.
