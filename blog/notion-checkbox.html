Notion Checkboxes: Complete Guide to Task Lists and Progress Tracking

8 min read • Updated July 2026 • FreelanceOS Team

Everything you need to know about checkboxes in Notion — from simple to-do lists to database properties, smart filters, completion formulas, and building a full task management system.

Two Types of Checkboxes in Notion

Notion has two distinct checkbox features that serve very different purposes. Understanding the difference is the foundation of using them effectively:

Type 1
To-Do Block

A checkbox item inside a Notion page — like a line on a handwritten checklist. Lives in page content. No database required. Check it off and the text gets a strikethrough. Best for: quick checklists, project notes, meeting agendas, SOPs.

Type 2
Checkbox Property

A true/false column in a Notion database. Every row has this property set to checked or unchecked. Filterable, sortable, formula-usable. Best for: task databases, tracking completion status on projects, contracts, and deliverables.

Most Notion beginners start with To-Do blocks because they're instant and simple. Power users migrate to Checkbox properties in databases because they're filterable, linkable, and formula-capable. This guide covers both.

Creating Checkbox To-Do Blocks in Pages

To-Do blocks are the quickest way to create an interactive checklist inside any Notion page — no database needed.

Three Ways to Create a To-Do Block

MethodHow to Do It
Markdown shortcutType [] (left bracket, right bracket — no space between) then press Space on a blank line
Slash commandType /todo or /checkbox and press Enter
Slash command (alt)Type /task — also opens the to-do block

Once you have a to-do item, press Enter to create the next item in the same list. The next line automatically becomes another to-do block. Press Enter twice (on a blank to-do line) to exit the list and return to a regular text block.

What Happens When You Check a To-Do Item

Click the checkbox to mark it done. The item's text immediately gets a strikethrough and the checkbox turns purple. Click again to uncheck it. Checked items stay in the list in place — they don't move or disappear.

Nesting To-Do Items

Press Tab on a to-do block to indent it (making it a sub-item under the previous item). Press Shift+Tab to unindent. Sub-items can be checked independently of their parent — checking the parent doesn't auto-check sub-items.

Nesting is useful for: breaking big tasks into sub-tasks, adding notes below a checklist item without creating a new item, grouping related items with a header item.

Limitation of To-Do blocks: You can't filter, sort, or query the checked/unchecked state of To-Do blocks the way you can with database Checkbox properties. If you need to filter tasks by completion, use a database with a Checkbox property instead.

Checkbox Database Properties

A Checkbox property is a true/false column in a Notion database. It's the property type you add to a database when you want to track binary status (done/not done, signed/unsigned, verified/unverified, active/inactive) on every entry.

How to Add a Checkbox Property to a Database

  1. Open any Notion database in Table view
  2. Click the + button to the right of the last column header
  3. Select Checkbox from the property type list
  4. Name the property (e.g., "Done", "Signed", "Paid", "Reviewed")
  5. Press Enter — the property appears as a checkbox column on every row
  6. Click any row's checkbox to toggle it between checked and unchecked

Checkbox Property vs. Select "Done/Not Done"

A question that comes up often: why use a Checkbox property instead of a Select property with options "Done" and "Not Done"? Use Checkbox when the state is strictly binary (true/false, yes/no, done/not done) — it's one click to toggle and simpler to filter. Use Select when there are more than two states (e.g., Not Started / In Progress / In Review / Done) — that's a workflow, not a binary flag.

Filtering by Checkbox

Filtering a database by a Checkbox property is one of the most useful things you can do with it — it lets you instantly show only incomplete items or only completed items.

Show Only Unchecked (Incomplete) Tasks

  1. Click Filter at the top of the database
  2. Click Add filter
  3. Select your Checkbox property (e.g., "Done")
  4. Choose condition: Is unchecked

The database now shows only tasks where "Done" is unchecked — your active task list. Completed items (where Done is checked) disappear from view. When you check off a task, it vanishes from this filtered view automatically.

Show Only Checked (Completed) Tasks

Same process, but choose condition: Is checked. Use this to build a "Completed" view of your database — a running log of everything you've finished.

Saving Checkbox Filters as Views

Create two named views for any database with a Done/Completed checkbox property:

  • Active — filter: Done is unchecked. This is your working view.
  • Completed — filter: Done is checked. This is your accomplishment log.

Switch between them with one click. Never manually hunt through a long list to find what's still outstanding.

Combined filter example: Done is unchecked AND Due Date within the next 7 days AND Assignee = Me → shows only your incomplete tasks due this week. A ready-made weekly focus view.

Checkbox Formulas: Calculate Completion Percentage

One of the most impressive Notion database setups is a completion percentage tracker — showing what percentage of a project's tasks are done. This requires a Rollup property and optionally a Formula property.

Method 1: Rollup — Percent Checked

If you have a parent-child relationship between two databases (e.g., Projects → Tasks, with a Relation property), you can use a Rollup to track how many tasks are done:

  1. In your Projects database, add a Rollup property
  2. Set Relation to your Tasks relation property
  3. Set Property to the "Done" checkbox in Tasks
  4. Set Calculate to Percent checked

This shows the percentage of related tasks with "Done" checked — e.g., "67%" means 4 of 6 tasks are complete. No formula needed.

Method 2: Formula — Completion Status

You can use a formula to convert a checkbox's true/false value into something more descriptive:

if(prop("Done"), "✅ Complete", "🔄 In Progress")

Or use a checkbox rollup value (returned as a number 0–1) to create a human-readable label:

if(prop("Task Completion") == 1, "✅ All Done", if(prop("Task Completion") >= 0.5, "🔄 More than half done", "⚠️ Just getting started"))

Visual Progress Bar via Formula

This formula turns a completion percentage into a visual progress bar using emoji blocks:

let(p, round(prop("Task Completion") * 10), replaceAll(replaceAll("##########", "#", "▓", p), "#", "░") + " " + format(round(prop("Task Completion") * 100)) + "%")

Common Checkbox Use Cases and Best Practices

Client Onboarding Checklist (To-Do Blocks)

Create a template page with all your onboarding steps as to-do items. Duplicate this page for each new client. Work through it step by step — checking items as you go gives visible progress and nothing gets missed.

Weekly Review Checklist (To-Do Blocks)

Build a recurring weekly review template with checkboxes for each review step: check financials, review active projects, update task priorities, send outstanding invoices, schedule next week's focus time. Duplicate each week and check off as you go.

Contract Signed Tracker (Database Property)

Add a "Contract Signed" checkbox property to your Clients database. Filter to "Is unchecked" to see clients who still need a contract. Filter to "Is checked" to see all confirmed clients. Simple, binary, and filterable.

Invoice Paid Tracker (Database Property)

Add "Paid" checkbox to your Invoices database. Filter to "Is unchecked" for your outstanding invoices view — the one you check before chasing payments. Filter to "Is checked" for your income log.

Content Publishing Status (Database Property)

Add "Published" checkbox to your content calendar database. Filter to "Is unchecked" for your drafts and scheduled content — what still needs to go live. Filter to "Is checked" for a published content archive.

Project Completion with Rollup (Advanced)

Use a Rollup set to "Percent checked" to show project completion percentage based on related task checkboxes. At a glance, see which projects are 20% done vs. 90% done — without counting tasks manually.

Frequently Asked Questions

How do I create a checkbox in Notion?
Type [] (left and right bracket, no space) followed by a space on a blank line — it automatically converts to a checkbox to-do item. You can also type /todo and press Enter. For a database checkbox property, click the + button in the database header, choose Checkbox as the property type, and name it (e.g., "Done" or "Signed").
How do I filter a Notion database by checkbox?
Click Filter at the top of the database, click Add filter, select your Checkbox property, and choose either Is checked (shows only ticked items) or Is unchecked (shows only unticked items). "Is unchecked" is particularly useful for building a live "active tasks" view — completed items automatically drop out when their checkbox is ticked.
Can I calculate a completion percentage from checkboxes in Notion?
Yes. If you have a parent database (e.g., Projects) related to a child database (e.g., Tasks) that has a Checkbox property, add a Rollup property to the parent. Set it to roll up the Checkbox property using Percent checked. This shows the percentage of related tasks that are checked — your project completion percentage, updated automatically as you check off tasks.
What is the keyboard shortcut for a checkbox in Notion?
Type [] followed by a space at the start of a line to create a new checkbox to-do block. To check or uncheck an existing checkbox block, press Cmd+Enter (Mac) or Ctrl+Enter (Windows) when the text cursor is inside that block. You can also simply click the checkbox square directly.
How do I use a checkbox property in a Notion database?
In any database view, click the + icon to add a new property and select Checkbox as the type. Name it something descriptive ("Done", "Paid", "Published"). The property appears as a checkbox column in every row — click any checkbox to toggle it. You can then filter by it (Is checked / Is unchecked), sort by it, and use it in formulas and rollups.
