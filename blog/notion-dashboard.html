Step-by-step guide to building a Notion dashboard that actually helps you work — personal, freelancer, and team dashboard setups with free template.

A Notion dashboard is your command center — the page you open every morning, scan in 30 seconds, and immediately know what to work on. Done well, it replaces scattered sticky notes, overflowing inboxes, and the constant mental overhead of "what do I actually need to do today?" Done poorly, it becomes a beautiful graveyard of outdated widgets you stopped checking after two weeks.

The difference between a dashboard that sticks and one that gets abandoned isn't design skill — it's knowing what to surface and what to leave in the database. This guide covers exactly that, with specific setups for three types of users and a free template to get you started instantly.

What Makes a Great Notion Dashboard (vs a Cluttered One)

The single most important rule of Notion dashboards: your dashboard should contain only information you act on daily. Reference material, archives, and project documentation belong in databases — your dashboard is the filtered lens on top of all of it.

Great dashboards share four characteristics:

1
Immediate orientation
You know what to work on within 10 seconds of opening the page. Today's tasks, top priority, and any overdue items are visible above the fold.
2
Filtered, not full
Every linked view shows only the relevant slice of a database. A client view filtered to Status = Active shows 4–8 entries, not 200 historical clients.
3
Low maintenance
Data updates automatically from underlying databases. You shouldn't have to manually update the dashboard — if you do, it won't stay current.
4
Predictable structure
The same sections are in the same places every day. Your brain learns to scan it quickly. Constant redesigns reset this muscle memory and create friction.

3 Types of Notion Dashboards

The right dashboard depends on what you're managing. Here are the three main types and what each should prioritize:

👤
Personal Dashboard
Focus areas: habits, goals, journaling, reading list, personal projects. Best views: calendar (tasks), gallery (areas of life), simple checklist (daily routine).
💼
Freelancer Business Dashboard
Focus areas: active clients, open invoices, project deadlines, weekly revenue. Best views: filtered tables (clients, invoices), timeline (deadlines), number callouts (revenue).
👥
Team Dashboard
Focus areas: sprint tasks, team member assignments, project status, upcoming meetings. Best views: board (kanban by status), filtered by assignee, timeline (project roadmap).

Building a Freelancer Business Dashboard: The Core Setup

The freelancer dashboard is the most high-value Notion dashboard you can build — it replaces spreadsheets, email threads, and sticky notes with a single source of truth for your business. Here's how to build each section:

Section 1: Today's Focus

Widget Type
Linked View — Tasks Database, filtered by Due Date = Today
Create a linked view of your Tasks database. Apply filter: Due Date is Today. Sort by Priority (high first). Set view type to List for compact display. Limit to 10 entries. This becomes your morning orientation — the first thing you read, the last thing you check before closing.

Section 2: Active Clients Widget

Widget Type
Linked View — Clients Database, filtered by Status = Active
A Gallery or Table view filtered to Status = Active. Include properties: Client Name, Next Action Date, Project Count, Monthly Value. This shows you your active roster at a glance — who you're working with, what's due, and the rough value of each relationship. Typically 2–8 entries for most freelancers.

Section 3: Outstanding Invoices

Widget Type
Linked View — Invoices Database, filtered by Status = Sent or Overdue
Table view showing: Client, Amount, Due Date, Status. Add a formula property that calculates days overdue for any unpaid invoices. The total outstanding amount (via Notion's Calculate sum at the bottom of the Amount column) gives you instant cash flow visibility. Never lose track of what you're owed again.

Section 4: Upcoming Deadlines

Widget Type
Calendar View — Projects Database, filtered by Due Date within 14 days
A Calendar view showing project deadlines in the next two weeks. Unlike the task list, this gives you a temporal overview — you can see when crunch periods are coming before they arrive. Filter by Status != Completed so finished projects don't clutter the view.

Section 5: Quick Capture Input

Widget Type
New Entry Button — Tasks Database
Add a "New Task" button (using Notion's button block or simply a linked view with "+ New" visible) so you can capture new tasks without leaving the dashboard or navigating away. The goal is zero friction between "I just thought of something" and "it's captured."

Section 6: This Week's Revenue vs Goal

Widget Type
Callout Block + Linked View — Invoices Database, filtered by Paid Date = This Week
A linked view summing paid invoice amounts for the current week, alongside a manually-set weekly revenue goal. Use a Callout block (the yellow or purple box in Notion) to display the goal prominently. At a glance, you know if you're on track. Some freelancers use a formula to calculate percentage: (thisWeek / goal) * 100 formatted as a number with a % symbol.

Building With Notion's Layout Options

The layout tools that make dashboards readable:

  • Columns: Drag any block into a column layout using the six-dot handle. Ideal for putting Today's Tasks and Active Clients side by side. 2-column layouts work well; 3+ columns get cramped on smaller screens.
  • Callout blocks: Use for key metrics, important reminders, or revenue numbers. The colored background makes them visually distinct from surrounding content.
  • Linked database views: The backbone of any dashboard. Create them with /linked view, select your database, apply filters and sorts, choose the right view type for the data.
  • Dividers and spacing: Use /divider to separate sections. Empty lines between sections add visual breathing room. Don't underestimate whitespace — it's what separates a "pro" dashboard from a crowded one.
  • Headings: H2 headings (##) for major sections, H3 for subsections. Keep them short and action-oriented: "Today's Priorities" not "Task Management System Overview."

Dashboard Design Principles

Good dashboard design is about hierarchy, not decoration:

1
Hierarchy: most important information at the top
Today's tasks should be above the fold. Revenue metrics and client overview mid-page. Reference widgets (reading list, goals) at the bottom. Information priority should match visual position.
2
Whitespace over density
An empty line between sections makes scanning easier. Resist the urge to fill every pixel. The eye needs rest points to move efficiently across the page.
3
Color sparingly, for signal not decoration
Callout block color should mean something: red = urgent, yellow = warning, green = on track. Don't use all six Notion colors for aesthetic variety — it creates noise rather than signal.

Essential Notion Dashboard Blocks

Linked Database Views (filtered/sorted)

The most powerful dashboard block. One database can power dozens of different views across your workspace. A Tasks database can show Today's Tasks on the dashboard, All Tasks in a project page, Overdue Tasks in a weekly review template, and Team Tasks in a shared workspace — all from the same data source, never duplicated.

Gallery Views for Visual Project Cards

Gallery views work well for projects and clients because you can see a "card" with the project name, status, and cover image at a glance. Set the card size to Small for dashboard use — Large cards take too much vertical space.

Timeline View for Deadline Planning

Timeline view (similar to a Gantt chart) shows projects and tasks across a horizontal time axis. For dashboards, a 2-week timeline filtered to active projects gives you a clear picture of upcoming crunch. Set the timeline scale to "Week" for the best balance of detail and overview.

Callout Blocks for Key Metrics

Use callout blocks to highlight numbers that matter: monthly revenue, client count, days until next project deadline. Keep each callout to a single number or short phrase. Pair with an emoji icon for fast visual parsing.

Mobile Dashboard Tips

Notion's mobile app handles some view types better than others. Design your dashboard with mobile in mind from the start:

  • Best on mobile: List views, simple table views (few columns), callout blocks, heading + paragraph text
  • Acceptable on mobile: Gallery views (medium card size), board views (one column visible at a time)
  • Avoid on mobile: Wide multi-column layouts (scroll horizontally), timeline views (too small to read), tables with 6+ columns
  • Best practice: Keep your most-used mobile section (Today's Tasks) at the very top, before any multi-column layouts. iPhone and Android users can scroll past layout complexity to find what they need.

Frequently Asked Questions

What should a Notion dashboard include?
A great Notion dashboard includes: a Today's Focus section (linked view of tasks due today), an Active Clients or Active Projects widget, an Outstanding Invoices or Key Metrics callout, an Upcoming Deadlines calendar view, and a Quick Capture input. For freelancers, add a This Week's Revenue vs Goal tracker. The key is surfacing actionable information, not storing everything.
How do I create a linked database view in Notion?
To create a linked database view: type /linked view on any page, select the database you want to link, then choose your view type (table, board, calendar, gallery, list, or timeline). After creating the view, click the filter icon to add filters (e.g., Status = Active, Due Date = This Week). This lets you show a filtered slice of any database anywhere in your workspace without duplicating data.
How do I stop my Notion dashboard from getting cluttered?
Three rules prevent dashboard clutter: (1) Only surface information you act on daily — reference material belongs in a separate database, not your dashboard. (2) Use filters aggressively on linked views so you only see relevant entries. (3) Limit dashboard sections to 5–7 maximum. When something stops serving you daily, remove it from the dashboard but keep it in the underlying database.
Can I use Notion as a dashboard on mobile?
Yes, but you need to design for mobile. List views and simple table views work best on small screens. Gallery views with medium-sized cards are readable. Avoid wide multi-column layouts — they scroll horizontally on phones. The best mobile dashboard uses a List view for Today's Tasks, a Gallery view for Active Projects, and callout blocks for key numbers.
