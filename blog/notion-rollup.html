Notion Rollup: The Complete Guide to Rollup Properties in Notion

Notion rollups let you aggregate data across related databases — count tasks, sum hours, find earliest deadlines, and build dashboards that update automatically. Here is how they work.

8 min read  ·  July 2026  ·  FreelanceOS Team
Home›Blog›Notion Rollup

Once you start using Notion's database relations to connect pages across multiple databases, you quickly hit the next question: how do you see aggregate data across those relationships? How many tasks does each project have? What is the total estimated time for all the tasks in this project? What is the earliest due date across all related items?

That is exactly what Notion rollups are for. A rollup reads data from related pages and calculates or aggregates it — counts, sums, averages, dates, and more. They are one of the most powerful features in Notion's database toolkit, and one of the most underused. This guide covers how rollups work, how to create one, every available rollup function, filtering rollups, and practical examples you can implement today.

Prerequisite: Rollups require a Relation property to already exist in your database. You cannot create a rollup without a relation — the rollup uses the relation as its data source. If you have not set up a relation between your databases yet, do that first, then come back to create rollups on top of it.

What Is a Notion Rollup?

A Notion rollup is a database property that reads data from pages in a related database and performs a calculation on that data. It then displays the result in the current database as a property on each row.

The key conceptual model: imagine you have a Projects database and a Tasks database, linked by a relation (each project is related to several tasks). Without a rollup, you can see which tasks are linked to each project — but to know how many tasks there are, you have to count them manually each time you look. A rollup does that counting automatically and shows the result as a column in your Projects database. Every time a task is added to a project, the rollup count updates automatically.

Common rollup use cases:

  • Count tasks per project — how many tasks does each project have total
  • Count incomplete tasks per project — how many tasks are still Not Done (with a filter)
  • Sum estimated hours — total estimated time across all tasks in a project
  • Earliest due date — the nearest deadline across all related tasks
  • Average invoice value per client — across all invoices linked to each client
  • Sum revenue per project — total value of all related invoice line items

Relation vs. Rollup — What Is the Difference?

These two properties are often confused because they work together, but they do different things:

PropertyWhat It DoesWhat It Shows
RelationLinks pages between two databasesThe linked page names (clickable links)
RollupAggregates data from linked pagesA calculated value (count, sum, date, etc.)

A relation is the connection. A rollup is what you do with the data on the other end of that connection. You need the relation to exist before you can create a rollup. You can have multiple rollup properties on a single relation, each calculating something different — for example, a relation to Tasks could have three rollup properties: Total Tasks (count), Estimated Hours (sum), and Next Due Date (earliest date).

How to Create a Rollup in Notion

Here is the step-by-step process to add a rollup property to any database:

1

Confirm You Have a Relation Property

Open the database where you want to add the rollup. Make sure there is already a Relation property that links to another database. If not, add one first: click the + icon to add a property, select "Relation," choose the target database, and configure the relation settings.

2

Add a Rollup Property

Click the + icon to add a new property to your database. In the property type list, scroll down to find Rollup and select it. A rollup property appears in your database with a configuration panel.

3

Configure the Relation

In the rollup configuration panel, click Relation and select which relation property to use as the data source. If your database has multiple relations (linking to different databases), choose the one you want this rollup to pull data from.

4

Choose the Property to Roll Up

Click Property and select which property from the related database to calculate on. For a task count rollup, you might select the task's Name property. For a sum of hours, select the estimated hours number property. For earliest date, select the due date property.

5

Set the Calculate Function

Click Calculate and choose the aggregation function. For a count of tasks, choose "Count All." For a total of hours, choose "Sum." For the nearest deadline, choose "Earliest Date." The rollup property now shows the calculated result for each row in your database, updating automatically as related pages change.

6

Name Your Rollup Property

Give the rollup a descriptive name that makes clear what it shows. Good names: "Task Count," "Total Hours," "Next Deadline," "Open Tasks," "Revenue." Poor names: "Rollup," "Property 4." Clear names make your database immediately readable without having to inspect the property settings.

The available rollup functions depend on the type of property you are rolling up. Here are all the options and what they calculate:

Filtering Rollups

One of the most powerful — and underused — rollup features is the filter. By default, a rollup aggregates all related pages. But you can add filter conditions to limit which related pages are included in the calculation. This lets you create much more specific rollup properties.

To add a filter to a rollup: click on the rollup property settings, then click Filter. A filter configuration panel appears. Add filter conditions based on any property of the related pages. Only pages that match the filter will be included in the rollup calculation.

Practical Filter Examples

  • "Open Tasks" rollup — Count All, filtered to Status does not equal "Done." Shows only incomplete tasks per project.
  • "Completed Tasks" rollup — Count All, filtered to Status equals "Done." Shows completed task count.
  • "High Priority Hours" rollup — Sum of Estimated Hours, filtered to Priority equals "High." Total hours just for high-priority tasks.
    • "This Week's Deadlines" rollup — Earliest Date on Due Date, filtered to Due Date is within the next 7 days. Shows the soonest upcoming deadline this week.
  • "Blocked Tasks" rollup — Count All, filtered to Status equals "Blocked." Surfaces blocked tasks directly in the project view.
Filtered vs. unfiltered rollups on the same relation: You can have multiple rollup properties on the same relation with different filters. For example: "Total Tasks" (no filter), "Open Tasks" (filter: status not done), "Completed Tasks" (filter: status done). These three rollups together give you a task completion picture directly in your Projects database without opening each project.

Using Rollups Inside Formulas

Rollup properties can be referenced inside Notion formula properties — this is where things get genuinely powerful. Once you have a rollup that calculates a value, you can use that value in a formula to derive further calculations or conditional logic.

Example: you have a Project database with these properties:

  • Total Tasks (rollup — Count All of related tasks)
  • Completed Tasks (rollup — Count All, filtered to Status = Done)
  • Completion % (formula — calculate from the two rollups)

The formula for Completion %:

if(prop("Total Tasks") > 0, round((prop("Completed Tasks") / prop("Total Tasks")) * 100), 0)

This gives you a real-time completion percentage for each project, calculated automatically from your task data. Build a progress bar visualization in Notion using this number property to get a visual completion indicator on each project row.

Common Rollup Problems and Solutions

Rollup shows "Calculate" but no value

This usually means no related pages exist for that row, or the selected property has no values on the related pages. Check that: (1) the row actually has related pages in the relation property, and (2) the property you are rolling up has values on those related pages.

Rollup shows all values instead of a calculated result

You likely selected "Show Original" or "Show Unique Values" as the calculate function. Change the Calculate setting to Count All, Sum, Earliest Date, or whichever aggregation you intended.

Sum rollup shows 0 even though related pages have numbers

The property you selected in the rollup configuration might be a formula property that displays a number but is technically a formula type. Rollups on formula properties work differently — try selecting the underlying number property instead, or use a Formula property that references the rollup after you confirm the data is visible.

Rollup filter is not excluding the right pages

Double-check that the filter condition is referencing the correct property from the related database (not the current database). Rollup filters operate on the properties of the linked pages — check the property names and values in your related database to confirm they match what you are filtering on.

Frequently Asked Questions

What is a rollup in Notion?
A Notion rollup is a database property that reads data from pages in a related database and calculates or displays that data in the current database. It requires a Relation property to exist first — the rollup uses the relation as its data source. For example: if your Projects database has a relation to a Tasks database, a rollup property can count how many tasks each project has, sum the estimated hours, or show the earliest due date across all related tasks. Rollup values update automatically as related pages change.
How do I create a rollup in Notion?
To create a Notion rollup: (1) Make sure you have a Relation property in your database linking to another database. (2) Click the + icon to add a new property. (3) Select Rollup as the property type. (4) In the rollup configuration: set Relation to your existing relation property, set Property to the field in the related database you want to calculate on, set Calculate to your desired function (Count All, Sum, Average, Earliest Date, etc.). The rollup property now displays the calculated data from your related pages, updating automatically.
What is the difference between a Notion relation and a rollup?
A Notion Relation property creates a link between two databases — it shows which pages are connected to each row. A Rollup property reads data from those connected pages and calculates or aggregates it. You need a Relation before you can create a Rollup. Think of it this way: the Relation is the link; the Rollup is what you calculate with the data on the other end of that link. You can have multiple rollup properties on a single relation, each calculating something different.
Can I filter a Notion rollup?
Yes. Notion rollups support filters that limit which related pages are included in the calculation. When configuring a rollup, click "Filter" in the rollup settings to add filter conditions based on properties of the related pages. For example: a rollup that counts tasks can be filtered to count only tasks with Status not equal to "Done" — giving you an open task count rather than a total task count. You can add multiple rollups on the same relation with different filters to see multiple slices of the same data.
What can rollup properties calculate in Notion?
Notion rollup calculation options include: Count All (total number of related pages), Count Values, Count Unique Values, Count Empty, Count Not Empty, Percent Empty, Percent Not Empty, Show Original, Show Unique Values. For number properties additionally: Sum, Average, Median, Min, Max, Range. For date properties additionally: Earliest Date, Latest Date, Date Range. The available calculations depend on which property type you are rolling up in the related database.
