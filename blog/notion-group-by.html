Notion Group By: Organize Your Database Views for Better Clarity

9 min read • Updated July 2026 • FreelanceOS Team

Learn how Notion's Group By feature transforms flat database lists into organized, scannable views — grouped by status, date, person, client, or any other property you choose.

What Is Notion Group By?

Group By is a Notion database feature that organizes your entries into labeled sections based on a property value — turning a flat list of items into structured, scannable groups without hiding any data.

The classic example: a Tasks database grouped by Status automatically sorts your tasks into sections like "Not Started", "In Progress", "In Review", and "Done" — each section collapsible, each showing a count of entries inside.

This is different from filtering (which hides non-matching entries) and sorting (which reorders entries within a flat list). Group By organizes entries into named buckets while keeping everything visible.

Group By vs. Filter vs. Sort — When to Use Each

FeatureWhat It DoesUse When
Group ByOrganizes entries into labeled sections by property valueYou want to see all data organized by category, status, or owner
FilterHides entries that don't match the filter conditionYou only want to see a subset of your data
SortReorders entries within each group (or within the flat list)You want items in a specific order (by date, priority, alphabetical)

You can use all three together: filter to show only active tasks, group by client, sort by due date within each client group.

How to Enable Group By on Any Database View

  1. Open any Notion database in Table, Board, Gallery, or List view
  2. Click the Group button in the view toolbar (or click ... → View options → Group by)
  3. Select the property you want to group by from the dropdown
  4. Entries immediately reorganize into labeled group sections
  5. Each group section shows the group name and an entry count in parentheses
  6. Groups are collapsible — click the arrow next to a group header to collapse it

To remove grouping: click Group → select No grouping from the top of the property list. The database returns to a flat list.

Board view note: Board view always uses Group By as its core structure — you can't turn off grouping in Board view, only change which property groups the columns. The default is usually Status.

Grouping by Select Property (Most Common)

Grouping by a Select property is the most common use — and the most useful for day-to-day project management. Each unique option in the Select property becomes a group.

Grouping by Status

Your Tasks or Projects database likely has a Status property with options like Not Started, In Progress, In Review, Done, Blocked. Grouping by Status gives you a Kanban-style view of your workflow:

Not Started (3)
Write proposal draft
Research competitor pricing
Set up invoice template
In Progress (2)
Client homepage redesign
Monthly report — Acme
Done (5)
Logo revision v3
Contract signed — Acme
Onboarding call notes

This is exactly what Board view uses by default. You can replicate it in Table or List view by enabling Group By → Status.

Grouping by Priority

If you have a Priority property (High / Medium / Low), grouping by it creates urgency sections — instantly showing you what needs attention first at the top of your view.

Grouping by Client or Project (Relation)

If you have a Relation property connecting tasks to a Clients database, you can group tasks by client. Each client becomes a group header — all their tasks underneath. This is ideal for client work management and weekly status reviews.

📁 Acme Corp (4 tasks)
📁 BrightLine Agency (2 tasks)
📁 Solo Project (6 tasks)
📁 No Client (1 task)

Grouping by Date Property

When you group by a date property, Notion offers granularity options — choose how to bucket the dates:

Date GroupingResultBest For
DayOne group per calendar dayContent calendars, daily task view
WeekOne group per week (Mon–Sun)Weekly sprint planning
MonthOne group per monthMonthly invoicing, content scheduling
QuarterQ1/Q2/Q3/Q4 groupsQuarterly goal and project tracking
YearOne group per yearAnnual archives, long-term planning

Date Grouping in Practice

Content calendar by week: Group your content database by Publish Date → Week. You see each week as a collapsible section with all content due that week — how many pieces, what they are, their status.

Monthly invoice tracker: Group invoices by Invoice Date → Month. Quickly see totals due each month without needing a rollup formula view.

Project timeline by quarter: Group a projects database by Start Date → Quarter. Visualize your pipeline across Q1, Q2, Q3, Q4.

Pro tip: Combine date grouping with a sort (sort by Due Date within each group — earliest first). This gives you chronologically ordered tasks within each week or month section.

Grouping by Person / Assignee

If you use a Person property to assign tasks to team members (or to yourself), grouping by it creates per-person task sections:

👤 Mark (8 tasks)
👤 Sarah (5 tasks)
👤 Jordan (3 tasks)
👤 Unassigned (2 tasks)

Use cases:

  • Workload review: see at a glance if one person has too much on their plate
  • Daily standup: pull up the board grouped by person to discuss each member's tasks
  • Sprint planning: balance task assignment across the team before kicking off a sprint

Grouping by Person works best in Table or List view. In Board view, you'd typically group by Status, then use Person as a secondary filter or sort.

Grouping by Checkbox

Grouping by a checkbox property creates exactly two groups: Checked and Unchecked. This is a simple but powerful pattern:

☑ Completed (12 items)
☐ Not Done (5 items)

Use cases:

  • To-do list database: group by a "Done?" checkbox to separate completed and pending tasks
  • Onboarding checklist: group by a "Verified?" checkbox to see what's been confirmed vs. outstanding
  • Contract tracker: group by "Signed?" to see signed vs. unsigned contracts at a glance

Sub-Grouping: Group By + Sub-Group By

Notion supports a second level of grouping — a sub-group within each primary group. This is available in Table and List views (not Board view).

How to Add Sub-Grouping

  1. Enable Group By for your primary grouping (e.g., Status)
  2. Click Group again → look for the Sub-group by option
  3. Select a secondary property (e.g., Priority)

Now each Status group is further subdivided by Priority:

📂 In Progress
   🔴 High (2 tasks)
   🟡 Medium (3 tasks)
   🟢 Low (1 task)
📂 Not Started
   🔴 High (1 task)
   🟡 Medium (4 tasks)

Sub-grouping is powerful for complex databases where you need two-dimensional organization — for example, tasks grouped by Phase (primary) then Priority (secondary), or content grouped by Month (primary) then Status (secondary).

Hiding Empty Groups

By default, Notion shows all groups — including empty ones (groups with no entries). This can be cluttered if you have a Status property with 8 options but most entries only use 3.

How to Hide Empty Groups

  1. Click the Group button in the toolbar
  2. Toggle off Show empty groups

Empty groups disappear immediately. When a new entry is added with a value matching a previously hidden group, that group automatically reappears.

When to keep empty groups visible: If you're using Board view as a Kanban board for a team workflow, keeping empty groups visible (like a "Done" column even when nothing is done yet) serves as a visual reminder of workflow stages and makes it easy to drag items into any column.

Group By Across Different Notion Views

Board View
Group By is the core mechanism — columns ARE groups. Default group is Status. Change it to any Select property to create a different Kanban board. Can't disable grouping in Board view.
Table View
Most flexible. Group By adds collapsible section headers between rows. Supports sub-grouping. All properties visible per entry. Best for dense data views.
Gallery View
Group By adds section headers above card grids. Useful for visual asset databases grouped by category or status. Each section shows a card grid.
List View
Simplest view — minimal columns. Group By adds section headers. Good for reading lists, link databases, reference lists where you want category sections.
Calendar View
Grouping by date is inherent to Calendar — entries appear on their date. An additional "Group by" option lets you create multiple rows per date property within the calendar.
Timeline View
Group By creates multiple rows in the timeline, one per group value. Group by Client or Project to see each client's/project's timeline as a separate horizontal row.

Combining Group By with Filters and Sorts

Group By, Filter, and Sort work independently and combine freely — enabling highly specific views:

Example 1: Client Workload This Week

Filter: Due Date within next 7 days + Status ≠ Done. Group By: Client. Sort (within groups): Due Date ascending.

Result: Only this week's active tasks, organized by client, earliest due date first within each client section.

Example 2: Content Calendar by Status

Filter: Publish Date within next 30 days. Group By: Status. Sort: Publish Date ascending.

Result: Next 30 days of content, grouped into "Draft", "In Review", "Scheduled", "Published" sections — instantly shows what's stuck and what's ready.

Example 3: Overdue Tasks by Priority

Filter: Due Date before today + Status ≠ Done. Group By: Priority (High/Medium/Low). Sort: Due Date ascending within each group.

Result: All overdue tasks, organized by urgency, most overdue item in each priority level shown first.

Group By Best Practices

Choose Your Primary Group Property Carefully

Group By works best with Select properties that have 3–8 distinct values. Too few groups (2 options) is barely useful. Too many groups (20+ options like tags) creates a cluttered view. Status, Priority, Client, Category, and Phase are ideal grouping properties.

Create Separate Views for Different Groupings

Don't switch Group By settings in a single view — create multiple named views instead. "By Status" grouped by Status, "By Client" grouped by Client, "By Priority" grouped by Priority. Switch between them with one click.

Use Group By Counts as Progress Tracking

The entry count shown in each group header doubles as a progress indicator. A "Done (0)" vs "Done (12)" comparison tells you immediately how much has been completed without needing a separate tracker.

Reorder Group Columns in Board View

In Board view, you can drag and drop column headers to reorder your groups. Arrange them to match your actual workflow — e.g., "Not Started → In Progress → In Review → Done" from left to right for a natural Kanban flow.

Add New Entries Directly Into Groups

In any grouped view, click the + New button inside a specific group section — the new entry is automatically assigned that group's property value. In a Status-grouped view, clicking + inside "In Progress" creates a task already set to "In Progress".

Frequently Asked Questions

How do I group a Notion database?
Open any database view, click Group in the view toolbar (or go to View options → Group by), then select the property you want to group by. Entries immediately reorganize into collapsible sections labeled with each property value. To remove grouping, click Group → select No grouping.
What is the difference between Group By and Filter in Notion?
Group By organizes all visible entries into labeled sections without hiding any data. Filter hides entries that don't match the filter condition — only matching entries are shown. You can use both together: filter to narrow what's visible, then group to organize what remains. For example, filter to "active tasks only" then group by client.
Can I group by a date in Notion?
Yes. When you group by a date property, Notion offers grouping granularity: by Day, Week, Month, Quarter, or Year. Choose the level that makes sense for your use case — Week is great for sprint planning, Month for content calendars and invoicing, Quarter for project and goal tracking.
How do I hide empty groups in Notion?
Click the Group button in the toolbar, then toggle off Show empty groups. Sections with no entries disappear immediately. They automatically reappear when an entry with that group's value is added. This keeps your view clean when many Select options exist but only a few are currently in use.
What views support Group By in Notion?
Group By works in Table, Board, Gallery, and List views. Board view requires grouping — you can change the property but can't disable it. Table and List views support sub-grouping (a secondary Group By). Calendar view groups by date inherently. Timeline view supports Group By to create multiple rows per property value.
