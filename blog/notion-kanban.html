What Is a Kanban Board and Why It Works

Kanban originated at Toyota in the 1940s as a visual scheduling system for manufacturing. The insight was simple: visualizing work in progress reveals bottlenecks that are invisible in a task list. Work gets stuck not because people are lazy but because handoffs fail, review queues pile up, or too many things are in progress simultaneously.

A kanban board has three core elements:

  • Columns representing stages in a workflow (e.g., To Do, In Progress, Done).
  • Cards representing individual tasks or items, one per row in the underlying database.
  • Movement — cards move left to right through columns as work progresses.

The visual format makes problems visible immediately. If your In Progress column has 12 cards and Done has 2, something is wrong. Kanban lets you spot that in seconds rather than digging through status reports.

Creating a Kanban View in Notion (Step by Step)

  1. Open or create a Notion database. Any existing database works — tasks, projects, content, clients.
  2. Click the + Add a view button at the top-left of the database header (or the view selector dropdown).
  3. Choose Board from the view type options. Notion creates the kanban view immediately.
  4. Notion automatically groups by the Status or first Select property it finds. If you need a different grouping property, click Group in the view options and select your preferred property.
  5. Name the view (e.g., "Kanban", "Project Board", "Pipeline") by clicking the view tab and renaming it.
  6. Click Properties in the view options to choose which properties show on each card (due date, assignee, priority, etc.).

Your kanban board is live. Cards appear in columns based on their Status value. Drag any card to a different column to update its status instantly.

Customizing Kanban Columns

Notion kanban columns are driven by a Select or Status property. To customize columns:

  1. Open a record in your database and click the Status or grouping property.
  2. Edit, add, or delete options. Each option becomes a column on the board.
  3. You can also click the three-dot menu on any column header to hide it from the view, rename it, or change its color.
  4. Drag column headers to reorder them.

The Status property type (introduced in 2023) has special behavior — it includes "Not Started," "In Progress," and "Done" categories that Notion treats slightly differently for filtering and rollups. Use Status for workflow stages; use regular Select for categorical groupings like Priority or Type.

Card Properties to Display on Kanban Cards

By default, Notion kanban cards show only the title. Click Properties in the view options to add visible properties. The most useful for project management:

  • Due Date — appears on the card with a color warning as the deadline approaches.
  • Priority — a Select property (High / Medium / Low) shown as a colored badge.
  • Assignee — person property showing avatar(s). Useful for team boards.
  • Project — a Relation to a parent Projects database, showing which project the task belongs to.
  • Estimated Hours — a Number property for capacity planning.

Avoid displaying too many properties on cards — more than 3-4 makes the board visually cluttered. Use card-level properties for the information you need at a glance; everything else lives inside the card itself.

4 Ways to Use Notion Kanban

1. Project Management
Track tasks across a project lifecycle — To Do, In Progress, Review, Done. Link tasks to a parent Projects database. Filter the board by project to see only the tasks for the current engagement. Add a Due Date property to spot upcoming deadlines.
2. Client Pipeline / CRM
Group contacts or deals by pipeline stage — Lead, Discovery, Proposal, Active, Past. Drag cards as leads progress. Add Value and Close Date properties to see deal size on each card. This replaces basic CRM tools like Trello for pipeline management.
3. Content Calendar
Group content pieces by production stage — Idea, Outline, Draft, Review, Scheduled, Published. See all your content at different stages simultaneously. Add a Publish Date property and filter to the current month. Switch between kanban and calendar views on the same database.
4. Task Management
A simple personal task board — Today, This Week, This Month, Someday. Or: Backlog, In Progress, Done. Use filters to show only tasks assigned to you or due this week. The kanban view turns a flat task list into a visual daily planning tool.

Advanced Kanban Tricks

Filter by assignee
In team Notion workspaces, use the Filter option to show only cards assigned to a specific person. Save the filtered view as a named view (e.g., "Mark's Board") so each team member has a personal kanban from the shared database.
Color-code by priority
Add a Priority select property (High / Medium / Low) with distinct colors. Make Priority visible on cards. Your board immediately communicates urgency visually — red cards demand attention, gray cards can wait.
Limit work in progress (WIP limits)
Kanban's original power came from WIP limits — capping how many items can be in progress simultaneously. In Notion, use a filtered Count rollup or simply establish a personal rule: no more than 3 cards in "In Progress" at once. When the column is full, finish before starting.

  • Backlog — Scoped work not yet started. Every project that has been confirmed and scoped lives here until you begin.
  • In Progress — Active work. Keep this to 2-3 items maximum for focus.
  • Review — Work delivered to client or awaiting your own quality review before sending.
  • Delivered — Completed and delivered but not yet invoiced. The gap between Delivered and Invoiced is where revenue gets stuck.
  • Invoiced — Invoice sent. Move to a separate archive database when payment clears.

Frequently Asked Questions

Does Notion have a kanban board?
Yes. Notion has a built-in kanban view that you can add to any database. Select a database, click Add a View, choose Board, and group by any Select or Status property. Cards drag between columns automatically as you update the property, and you can customize which properties appear on each card.
Is Notion kanban as good as Trello?
Notion kanban is more powerful than Trello for most use cases because it lives inside a full workspace — your kanban tasks are linked to projects, notes, documents, and databases. Trello is simpler and faster to set up from scratch, but Notion replaces Trello entirely for most freelancers who already use Notion for other purposes.
How do I add a kanban view in Notion?
Open any Notion database. Click the + Add a view button at the top left of the database. Select Board from the view type options. Choose the property to group your board by — typically a Status or Select property with your stage names. Your kanban board is ready instantly.
What is the best kanban setup for freelancers in Notion?
The best freelance kanban in Notion uses five columns: Backlog (ideas and future work), In Progress (active tasks), Review (awaiting client or self-review), Delivered (completed and delivered to client), and Invoiced (billed). This covers the full freelance project lifecycle and makes it easy to see where every project stands at a glance.
