-- Revenue by acquisition cohort, monthly
-- Table: orders | Updated: 2025-10-01 | Owner: @you
SELECT
  DATE_TRUNC('month', u.created_at) AS cohort_month,

  COUNT(DISTINCT o.user_id) AS paying_users,
  SUM(o.revenue) AS total_revenue
FROM orders o
JOIN users u ON u.id = o.user_id
GROUP BY 1, 2

Data Documentation

A Data Dictionary That Actually Gets Used

Stop relitigating metric definitions in every meeting. One source of truth for every table, metric, and owner in your data stack.

What the Data Dictionary Covers

Metric Entries

  • Business definition — in plain English
  • Calculation logic — exact formula or query pattern
  • Source table(s) — linked to Table entries
  • Owner — who maintains this definition
  • Update frequency — real-time, daily, weekly
  • Downstream dashboards — every report using this metric
  • Known caveats — exclusions, historical gaps

Table Entries

  • Table name & schema — exact database location
  • Description — what this table contains
  • Key fields — data types and business meaning
  • Upstream source — where data comes from
  • Downstream tables — what depends on this table
  • Owner — who to contact for questions
  • Known issues — nulls, duplicates, lag warnings

Frequently Asked Questions

Everything analysts ask before getting started.

Can Notion replace Confluence for data documentation? ▾
For most analytics teams, yes. Notion offers faster page creation, better cross-linking, real-time collaboration, and a much lower learning curve than Confluence. Teams under 50 consistently find Notion more practical for data dictionaries, runbooks, and analysis notes. FreelanceOS Pro's structured database approach makes it more powerful than a blank Notion setup.
How do I build a SQL snippet library in Notion? ▾
Create a Notion database with properties for table name, use-case tag, last tested date, complexity, and a relation to your Dashboard Inventory. Each row gets a full code block with SQL syntax highlighting. Use filtered views to find snippets by table or use-case in seconds. FreelanceOS Pro ships this pre-built with example entries.
What is a data dictionary in Notion and how do I structure it? ▾
A Notion data dictionary is a database where each row is a metric or table. Metric entries include business definition, calculation logic, source table, owner, and downstream dashboards. Table entries include schema location, key fields, upstream sources, and known issues. It becomes the single reference your team checks before every meeting.
Is Notion useful for data scientists, not just analysts? ▾
Absolutely. Data scientists use Notion for experiment tracking, hypothesis logging, model documentation, and ML project management. Code blocks support Python and Markdown, making it a lightweight lab notebook alongside GitHub links and Jupyter references. The Learning Tracker is especially useful for papers, courses, and conference talks.
How do I track stakeholder analytics requests in Notion? ▾
Set up a request queue database with requester, business question, priority, effort estimate, status, due date, and final analysis link. Share a filtered status view with stakeholders so they see progress without pinging you on Slack. FreelanceOS Pro includes this queue pre-configured with an intake template.
Can I use Notion as a BI analyst workspace alongside Tableau or Power BI? ▾
Yes — Notion complements BI tools. Your dashboards live in Tableau or Power BI; Notion holds the context: what the dashboard measures, which tables it queries, who owns it, when last validated, and the SQL behind each metric. The Dashboard Inventory is built for BI analysts managing large portfolios who need to track dependencies, freshness, and owners.
