Database Deep Dive

The four databases, explained

Clients Database: Your Freelance CRM

The Clients database is the anchor of the entire system. Every other database links back to it. Where most freelancers keep client notes scattered across emails, DMs, and document folders, the Clients database centralizes everything — contact details, relationship status, contract start date, and a live view of every project and invoice associated with that client.

The Status property uses a Select field with options like Prospect, Active, Paused, and Past Client. This lets you filter your client list instantly to see only the clients you're currently working with, or only the prospects you should be following up with. The Industry field helps you track where your work is concentrated — useful when you want to double down on a niche or diversify.

The real power is in the Rollup fields. Because Invoices links to Clients, you can add a Rollup property that sums up every invoice amount for a given client — giving you a lifetime revenue figure per client without ever running a calculation manually. This is the kind of insight that takes hours to surface in a spreadsheet but is instant in a properly built relational Notion database.

Projects Database: From Kickoff to Delivery

The Projects database tracks every engagement, proposal, and retainer. Each project record links to a Client, which means the client's name, email, and relationship details are always visible from the project without navigating away. The Phase property (Discovery, In Progress, Review, Completed) lets you group your board view by phase — so at any given moment you can see exactly where every active project sits in your workflow.

The Days Left formula calculates the number of days remaining to the Deadline property automatically. When a project goes into the red, it changes visually — no more mentally calculating whether you have enough runway. The Budget field combined with a rollup of Invoice amounts means you can track actual vs. planned revenue at the project level.

One underrated property in the Projects database is the linked Tasks field. Because every task in the Tasks database links back to a project, the Projects database automatically shows a list of all associated tasks. The board view inside a project record gives you a mini Kanban for just that project's deliverables — without creating a separate board for every client.

Invoices Database: Your Income Tracker

The Invoices database does one job and does it exceptionally well: it tells you what you're owed and what you've been paid. The Payment Status Select property has five options — Draft, Sent, Paid, Partial, and Overdue. A filtered view called "Unpaid Invoices" surfaces every invoice that isn't marked Paid, sorted by Due Date. This is the view you open every Monday morning.

The Days Overdue formula is one of the most valuable additions in the system. It calculates the number of days since the Due Date for any invoice that is not Paid — so you always know at a glance which invoices need a follow-up and how urgently. Paired with the client's email address pulled in via the Client relation, sending a payment reminder is a two-click operation.

Because Invoices links to both Clients and Projects, every invoice record inherits context from both. You can filter your invoice view by client to see all billing history for a single client, or filter by project to see exactly what was invoiced for a specific engagement — useful at tax time or when clients ask for billing breakdowns.

Tasks Database: Deliverables That Know Their Context

The Tasks database is not a general to-do list. It is a structured database of deliverables, each of which belongs to a specific project. Because every task links to a project, and every project links to a client, the Tasks database can surface the client name for any task via a Rollup — without duplicating data. This is what a properly normalized database looks like in Notion.

The Priority Select field (Urgent, High, Normal, Low) combined with the Due Date property powers a "Today's Focus" filtered view — tasks due within the next three days, sorted by Priority. The Kanban view groups tasks by Status (Not Started, In Progress, Review, Done) and is the most-used view among FreelanceOS users.

The Blocked By relation is a self-referential field that links tasks within the same database — meaning you can flag that Task B cannot start until Task A is complete. For multi-step projects with dependencies, this prevents missed steps and keeps delivery timelines realistic.

Can I customize the database properties?

Absolutely. Every property in every database is fully editable. You can rename fields, change property types, add new select options, create additional formula properties, and build new views. What is the difference between a database and a regular Notion page?
+
A regular Notion page is like a document — it holds text, images, and blocks. A Notion database is a structured collection of pages where every entry shares the same set of properties (fields). This means you can filter, sort, group, and link entries across databases. For example, a database of projects can be filtered to show only active projects, sorted by deadline, and linked to your client database so each project displays the client's contact details automatically. Databases unlock capabilities that regular Notion pages simply cannot replicate.
Can I import my existing data?
+
Yes. Notion supports CSV import into any database. If you have existing client or project data in a spreadsheet, you can export it as a CSV and import it directly into the corresponding FreelanceOS database. Relation fields will need to be linked manually after import, but the process is straightforward and is covered step-by-step in the included setup guide. For most freelancers with under 50 client records, manual entry is often faster than a CSV import — the setup guide helps you decide which approach makes sense.
How do the databases connect to each other?
+
FreelanceOS uses Notion's built-in Relation and Rollup properties to create a fully connected system. The Clients database links to Projects, so each client record shows all associated projects. The Projects database links to both Clients and Invoices. The Invoices database links back to both Projects and Clients. The Tasks database links to Projects, so every task is tied to a specific deliverable — and the client name is surfaced via a Rollup from the linked project. All relations are bidirectional: editing a linked record in one database automatically updates the connected record in every other database.
