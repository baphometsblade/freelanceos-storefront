Frequently Asked Questions

Can I use this to manage docs in multiple formats — API refs, user guides, tutorials?
Yes. The Documentation Project Tracker includes a Doc Type field so you can categorize and filter by format — API reference, user guide, tutorial, release notes, README, and more. You can manage all formats simultaneously and filter the tracker to see only the document type you're focused on at any given moment.
How do I track SME review cycles?
The Review and Feedback Tracker captures each review round with reviewer name, review type, date sent, date received, status, and key feedback notes. You can see at a glance which documents are waiting on a reviewer, which are in active revision, and which have cleared all gates — so no review goes cold and no deadline slips through.
Does it include a style guide template?
Yes. The Style Guide Database is pre-structured for technical writing conventions — terminology decisions, capitalization rules, code formatting standards, approved phrasing patterns, and brand-specific preferences. You can add entries as you make decisions on a project and build a comprehensive, searchable style guide incrementally over the course of an engagement.
Can I use it alongside tools like Confluence or GitBook?
Yes. FreelanceOS Pro in Notion is a project management and planning layer — not a replacement for your publishing platform. Link each Documentation Project Tracker entry to the corresponding Confluence space, GitBook section, or GitHub repo URL. Use Notion for planning, tracking, and style decisions; use your publishing platform for the actual doc content.
How do I track time and bill for technical writing projects?
The Time Tracker lets you log hours per document or milestone with a project relation, and the Invoice Tracker captures all invoices with project, amount, status, and payment date. Filter the Time Tracker by project to calculate billable hours for each invoice period and keep all financial records in the same workspace as your documentation projects.
